深度学习报告：Ozkaya高速光通信自适应接收机¶

1. 论文基本信息¶

1 figures from original paper:

Figure 1.1 Figure 1.1: CDF of link utilization [8]

🔍 深度说明：这是数据中心链路的链路利用率累积分布函数（CDF）图，横轴是链路利用率（0%~100%），纵轴是CDF值（对应某个利用率出现的概率或以下的比例）。这是Ozkaya论文提出自适应接收机的motivation——数据中心的光链路平均利用率<10%，意味着90%的时间链路空闲但仍消耗全功耗。从图中可以读出：绝大多数链路的利用率低于10%，说明按峰值功率设计的收发模块存在巨大的功耗浪费空间。CDF曲线越陡峭（集中在低利用率区），功耗浪费越严重。这是为什么需要自适应功耗技术——在低利用率时降低功耗，在高利用率时恢复全速。这个数据是整个论文研究问题定义的出发点，我们现在做数据中心光模块设计时，也是基于同样的低平均利用率假设来优化功耗/性能动态切换机制。

标题: Adaptive Receiver Design for High Speed Optical Communication

作者: Ilter Özkaya

机构: 瑞士洛桑联邦理工学院（EPFL）微电子系统实验室

页数: 约135页（博士论文，2018年）

关键词: 自适应接收机；高速光通信；NRZ调制；突发模式；时钟数据恢复（CDR）；决策反馈均衡（DFE）；功率可扩展；FinFET

2. 研究背景与问题定义¶

9 figures from original paper:

Figure 2.1 Figure 2.1: CTLE example: (a)Frequency response (b) Pulse response (c)Eye diagram of the bandwidth limited signal (d)Eye

🔍 深度说明：这是连续时间线性均衡器（CTLE）的综合示例图，包含(a)频率响应、(b)脉冲响应、(c)带宽受限原始信号眼图、(d)CTLE均衡后眼图。CTLE是Serdes接收机前端的核心均衡模块，通过在频率域抬升高频增益来补偿信道的高频衰减（因为光纤和PCB的损耗都是频率相关的——高频损耗更大）。(a)频率响应展示了CTLE的零点(z)和极点(p)配置——零点用于提升高频，极点用于限制带宽避免放大噪声。(b)脉冲响应展示了CTLE对单位脉冲的时域响应——可以看到过冲(overshoot)和振铃(ringing)特性，这是高频提升的代价。(c)和(d)的眼图对比直接展示CTLE的效果——均衡前眼图闭合，均衡后眼图张开。我们现在做56G/112G Serdes时，CTLE是标准配置，一般用5~7阶零点/极点配置，峰值增益3~9dB，具体参数根据信道特性调优。

Figure 2.2 Figure 2.2: CTLE circuit example: (a)Passive (b)Active

🔍 深度说明：这是CTLE的两种电路实现方案：无源（Passive）CTLE和有源（Active）CTLE。无源CTLE使用电阻电容网络（RC退化放大器或源极电感）实现频率响应，不需要静态功耗，但高频增益受限（约0~6dB），噪声性能好，适合低功耗应用；有源CTLE使用主动放大器（gm-C或电阻反馈放大器）实现，可以提供更大的高频增益（6~15dB），但有静态功耗，噪声性能较差。图中可见两种方案的结构差异——无源方案用LC谐振回路或有源方案用gm-cell实现频率整形。Ozkaya论文选择哪种方案取决于具体应用——数据中心短距（<100m）优先用无源CTLE（低功耗），长距或高灵敏度需求用有源CTLE（更高增益）。我们现在做Serdes IC设计时，一般会同时流片两种方案进行对比。

Figure 2.2 Figure 2.2: CTLE circuit example: (a)Passive (b)Active

🔍 深度说明：这是第二张CTLE电路示例图，可能是Passive CTLE的另一种实现（差分版本）或不同工艺节点（FinFET vs CMOS）的实现。差分Passive CTLE相比单端版本有更好的共模噪声抑制能力，但电路面积更大（需要2倍的无源元件）。图中元件的布局反映了差分电路的对称性要求——两个分支的电阻电容必须精确匹配，否则共模噪声会转化为差模噪声。我们现在做Serdes IC的CTLE时，一般用差分结构以确保共模抑制能力，特别是在数据中心这种噪声环境复杂（开关电源、串扰）的场景。

Figure 2.3 Figure 2.3: N-tap DFE conceptual block diagram

🔍 深度说明：这是N抽头判决反馈均衡器（DFE）的概念框图。DFE的核心思想是利用已判决的历史符号（z^-1延迟块表示）经对应抽头系数（k1, k2, ..., kN）加权后反馈叠加，抵消信道引入的后向码间串扰（post-ISI）。DFE相比FFE的优势是：可以处理非因果的ISI（后向干扰），而FFE只能处理前向干扰（pre-ISI）；同时DFE不放大噪声（因为反馈路径不加权到噪声上）。N抽头的DFE可以抵消N个符号的后向ISI，但每个抽头都会增加硬件复杂度（约每抽头1个乘法器+1个加法器）和错误传播风险——如果当前判决错误，错误会通过反馈路径影响后续符号的判决。Ozkaya论文的DFE设计目标是支持>56Gb/s，同时保持低功耗（<100mW），这是当时CMOS Serdes的挑战。

Figure 2.4 Figure 2.4: Eye diagrams after DFE

🔍 深度说明：这是DFE均衡后的信号眼图。眼图是评估Serdes系统性能的经典工具——眼高（eye height）对应噪声容限，眼宽（eye width）对应时序容限。从这张图可以评估DFE对ISI的抑制效果——好的DFE应该让眼图尽可能张开（眼高>100mVUI，眼宽>0.3UI）。实际中，DFE后的眼图通常不是完全对称的——因为后向ISI的分布不对称（取决于信道的频率响应特性）。这张图也间接说明了信道特性——如果均衡前眼图是高度闭合的（带宽严重受限），DFE能恢复的眼图也有限，必须先用CTLE/FFE处理高频衰减。Ozkaya做DFE眼图测试时，用的是28nm FinFET工艺，目标是56Gb/s NRZ，测试眼图应该在0.3UI眼宽附近。

Figure 2.5 Figure 2.5: 1-tap speculative DFE

🔍 深度说明：这是1-tap speculative DFE的结构图。Speculative DFE（推测式DFE）是为了解决DFE关键路径延迟（critical path delay）问题而提出的架构——传统DFE需要在当前符号被判决后才能计算反馈信号，这限制了工作速度；speculative DFE在时钟周期内同时计算两个可能结果（z=+1和z=-1的反馈值），然后在判决结果出来后再选择正确的分支。这种"先猜后选"的架构可以把DFE的工作速度提升30%~50%。1-tap意味着只有一个反馈抽头，适用于轻度ISI的短距信道（如数据中心内部的100m MMF）。Speculative DFE的代价是：硬件面积翻倍（需要两套加法器），功耗增加约20%，但换来了更高的工作频率。Ozkaya选择speculative DFE是因为它能支持>56Gb/s的目标。

Figure 2.6 Figure 2.6: N-tap FFE block diagram

🔍 深度说明：这是N抽头前向反馈均衡器（FFE）的实现框图。FFE的核心思想是将输入信号经多级延迟链路（T-delay）生成多路延迟副本，各副本乘以预设加权系数（w0, w1, w2, ..., wN）后求和输出。FFE相比DFE的优势是：没有错误传播问题（因为不依赖判决结果），可以处理前向和后向ISI；但FFE会放大噪声（因为每个分支的加权求和都会把噪声一起放大）。N抽头的FFE可以补偿N个符号的ISI，每增加1个抽头约增加3~5mW功耗和0.1mm²面积（28nm工艺）。FFE的抽头系数一般通过离线训练获得（用已知训练序列），或通过自适应算法（LMS）实时更新。Ozkaya在论文中用FFE作为DFE的补充——先用FFE做粗均衡（抵消主要ISI），再用DFE做精均衡（抵消残留后向ISI）。

2.1 研究背景¶

传统I/O链路的功耗与系统带宽需求变化无关。数据中心链路平均利用率低于10%，大部分时间空闲但仍消耗功率。这导致大量能源浪费。

功耗分布（数据中心）: - 服务器: ~40% - 存储: ~37% - 网络设备: ~23%

本地通信（服务器与内存间I/O链路）约占芯片总功耗10%。

2.2 核心问题¶

  1. 能效问题: 峰值功率设计导致低利用率时大量能耗浪费
  2. 响应速度: 传统系统级休眠/聚合/速率适配技术反应慢、复杂
  3. 高速需求: 目标>56 Gb/s，是当时最先进的CMOS光接收机2倍
  4. 快速唤醒: 目标<10 ns上电时间，适应突发数据
  5. 高灵敏度: 目标<-9 dBm OMA灵敏度

2.3 研究目标¶

设计实现自适应光学接收机，具备快速上电/下电功能，数据率60 Gb/s，14 nm FinFET技术，突发模式运行能力。

3.2 均衡技术¶

3.2.1 连续时间线性均衡（CTLE）¶

功能: 通过高通滤波器恢复带宽

无源电路传递函数（式2.4）:

TF_PASS = (R₂/R₁)·(1+sC₁R₁)/(1+s(C₁+C₂)R₁R₂/(R₁+R₂))

有源电路传递函数（式2.6）:

TF_ACT = (2g_mR_L/(2+g_mR_S))·(1+sC_SR_S)/(1+sC_Sg_m/(2+g_mR_S))/(1+sR_LC_L)

简化形式（式2.7）:

TF_ACT ≈ (2R_L/R_S)·(1+sC_SR_S)/(1+sC_Sg_m/(2+g_mR_S))/(1+sR_LC_L)

局限: - 开关引入寄生效应 - 高频噪声增强

3.2.2 决策反馈均衡（DFE）¶

N抽头DFE概念（图2.3）:

VFB = R(V_IN·G_IN + ΣG[k]·D[k], k=1 to N)

时序约束（式2.9-2.11）:

t_cq + t_setup + t_G[1] < UI  （第一抽头关键路径）
t_cq + t_setup + t_sq,MUX < UI （推测架构）

优缺点: - 不增加噪声 - 错误传播效应

4.1 自适应协议设计¶

数据协议: - 协议确定突发开始和结束 - 减少CDR相位锁定时间

自主运行: - 接收机自主检测数据需求 - 无需外部使能/禁止信号 - 上电/下电模式无需控制器

4.2 功率优化技术¶

  1. 电源和偏置缩放: 根据数据率调整
  2. 4:1时间交织: 降低最大时钟速度
  3. 推测DFE: 放宽时序约束

4.3 电路创新¶

  1. SFR TIA选择: 低噪声高带宽
  2. 低延迟CDR: 定制逻辑实现快速锁定
  3. 突发模式CDR: 解决元稳定性问题
