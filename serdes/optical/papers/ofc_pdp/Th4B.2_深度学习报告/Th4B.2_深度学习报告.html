Th4B.2 深度学习报告：TFLN光DAC实现无驱动器448G PAM4和1.2T 16-QAM传输¶

原始论文：McGill University + HyperLight Corporation — "Driver-less 448 Gbps PAM4 and 1.2 Tbps 16-QAM IMDD/Coherent-lite transmission using TFLN optical DACs" 整理日期: 2026/04/15 研究深度: 深度解读（结合TFLN调制器、光学DAC、CMOS驱动、IM-DD与相干检测）

一、论文核心贡献¶

1.1 技术突破¶

本文展示了世界首款无驱动器光学DAC (oDAC)，使用CMOS逻辑门直接驱动TFLN调制器实现超高数据率传输。

核心成就： | 指标 | 数值 | 意义 | |------|------|------| | IM-DD PAM4 | 448 Gbps @ 2km | DR8-2兼容 | | 相干-lite 16-QAM | 1.2 Tbps @ 10km | 业界领先 | | 调制格式 | PAM4/6/8, 16/32/64-QAM | 多阶调制 | | oDAC带宽 | 110 GHz | CMOS极限 | | 驱动器电压 | 600 mV差分 | CMOS直接输出 | | 传输距离 | 2-20 km | 覆盖DCI |

1.2 为什么要无驱动器¶

传统方案的问题：

传统发射机架构：
DSP → eDAC → Driver → MZM
         ↑          ↑
      功耗大     成本高

Driver挑战：
- 功耗: 500mW - 2W
- 带宽限制: ~60 GHz
- 延迟: ~10 ns
- 成本: $50-200

CMOS直接驱动优势：
- 功耗极低
- 带宽极高 (>100 GHz)
- 亚纳秒延迟
- 无额外成本

二、光学DAC (oDAC) 原理¶

2.1 二进制加权MZM (BW-MZM)¶

架构设计：

V₁ (MSB) ──┬──▶│         │
            │   │  MZM₁  │───┐
V₂ (LSB) ──┤──▶│         │   │  ──▶ 输出
            │   └─────────┘   │
            │                  │
            └──────────────────┘
                 光功率合成

MSB: 所有7个bit全开 = 1
LSB: 仅LSB 6bit = 1

二进制加权：
P(out) = Σ Vᵢ·2ⁱ·Gᵢ

其中Gᵢ是第i个光功率权重

工作原理：

电域 → 光域转换

电信号（0/1）→ 直接调制光功率
- 1: 全功率
- 0: 零功率

光域叠加：
P_total = P₁ + P₂ + ... + P₇

实现PAM4/8等多电平

2.2 二进制加权IQM (BW-IQM)¶

用于相干调制：

IQM结构：
         I-branch              Q-branch
    ┌─────────────┐      ┌─────────────┐
V_I₁─▶│   MZM_I   │──┬──▶│             │
V_I₂─▶│   (BW)    │  │   │   90° Hybrid│──▶ 输出
    └─────────────┘  │   │             │
                     │   └─────────────┘
V_Q₁─▶│   MZM_Q   │──┘
V_Q₂─▶│   (BW)    │
    └─────────────┘

四路二进制加权 → 16-QAM甚至64-QAM

相位调制特性：

push-pull配置：
- I-branch: +φ
- Q-branch: -φ
- 90°混合后: I+jQ

无啁啾条件：
- 相同调制深度
- 完美相位平衡

2.3 TFLN优势¶

为什么选择TFLN？ | 特性 | TFLN | SiPh | LiNbO₃ | |------|------|------|--------| | 半波电压 | 低 (~2-4V) | 高 (~8V) | 中 (5V) | | 带宽 | >100GHz | ~60GHz | ~40GHz | | 线性度 | 极好 | 一般 | 好 | | 功耗 | 低 | 中 | 中 |

三、CMOS逻辑门直接驱动¶

3.1 3nm CMOS特性¶

先进节点优势：

3nm CMOS规格：
- 逻辑门切换速度: >100 GHz
- 输出电压摆幅: ~0.7 V
- 功耗: < 10 fJ/bit

对比eDAC：
- eDAC带宽: ~60 GHz (商用)
- eDAC功耗: ~1 pJ/bit
- eDAC延迟: ~10 ns

逻辑门输出等效：

一个逻辑门等效于：
- 1-bit分辨率eDAC
- 带宽: 110 GHz (6dB)
- 输出: 600 mV差分

使用7个逻辑门实现7-bit分辨率

3.2 预加重与均衡¶

传统TX DSP功能：

传统TX DSP功能：
1. 数字预加重 (Pre-emphasis)
2. 脉冲整形 (Pulse shaping)
3. 重采样 (Resampling)

逻辑门驱动的限制：
- 无法数字预加重
- 无法复杂脉冲整形

解决方案：
- 模拟滤波器替代
- 光学均衡 (oDAC特性)

3.3 多级调制实现¶

PAM4产生：

2个逻辑门 → 4电平

电平映射：
- 00: 0 mW
- 01: P/3
- 10: 2P/3
- 11: P

眼图质量：
- 眼高均匀
- 线性度好

16-QAM产生：

4个逻辑门 (I×2 + Q×2) → 16星座点

I: 4电平 (-3, -1, +1, +3)
Q: 4电平 (-3, -1, +1, +3)

16-QAM = 16个点

四、实验配置¶

4.1 IM-DD实验¶

PAM4配置：

发射端：
- 3nm CMOS eDAC (1-bit模式)
- 2个逻辑门输出
- 差分600mV驱动
- TFLN BW-MZM

光源：1310nm DFB
光纤：2km SMF

接收端：
- 110 GHz PD
- 113 GHz RTO
- DSP (FFE 51tap + DFE 11tap)
- 1-tap MLSE

PAM6/8配置：

类似PAM4，更多电平：
- PAM6: 3个逻辑门
- PAM8: 4个逻辑门

4.2 相干-lite实验¶

16-QAM配置：

发射端：
- 4个逻辑门 (I×2, Q×2)
- TFLN BW-IQM
- 偏振控制
- DP (双偏振)

传输：
- 10km SMF
- 20km SMF

接收端：
- 相干检测
- 4×4 MIMO均衡
- 载波恢复

4.3 DSP链路¶

接收端DSP：

IM-DD DSP链：
1. 同步
2. FFE (51 taps)
3. DFE (11 taps)
4. 1-tap MLSE
5. BER计算

相干-lite DSP链：
1. 偏振解交织
2. 90°光学混合
3. 4×4 MIMO均衡
4. 载波相位恢复
5.判决 + BER

五、性能结果¶

5.1 PAM4结果¶

448 Gbps @ 2km： | 参数 | 数值 | |------|------| | 符号率 | 224 GBd | | 调制 | PAM4 | | 距离 | 2 km | | BER | 1.51×10⁻³ | | ER | 3.76 dB | | OMA | 7.1 dBm | | TDECQ | <4 dB |

对比传统方案：

oDAC vs 传统IMDD：
- oDAC @ 2km: 448G
- 传统IMDD: 通常<200G

优势来源：
- 110GHz带宽
- 无驱动器瓶颈
- TFLN低Vπ

5.2 16-QAM结果¶

1.2 Tbps配置：

126 Gbaud × 32 QAM × 2 pol = 1.2 Tbps

各阶调制性能：
| 格式 | 速率 | 距离 | BER |
|------|------|------|-----|
| 16-QAM | 800G | 10km | 3.49×10⁻³ |
| 32-QAM | 1T | 10km | 4.5×10⁻² |
| 64-QAM | 1.2T | 500m | 4.9×10⁻² |

EVM分析：

EVM (Error Vector Magnitude)：
- 16-QAM: EVM = 18.3%
- 32-QAM: EVM = 29.1%
- 64-QAM: EVM = 23.1%

EVM vs BER关系：
EVM² ≈ 1/(2×SNR)

5.3 频谱效率¶

oDAC vs 传统IQM：

64-QAM @ 125 GBaud：
- oDAC 500m: EVM = 23.1%
- Normal Coherent 500m: 基准

OSNR代价：
- SD-FEC 20%: 0.34 dB penalty
- SD-FEC 25%: 稍好

结论：oDAC性能接近传统相干

六、无驱动器架构优势¶

6.1 功耗分析¶

功耗对比： | 组件 | 传统方案 | oDAC方案 | |------|---------|----------| | DAC | ~500 mW | ~100 mW | | Driver | ~1 W | 0 | | 调制器 | ~100 mW | ~100 mW | | 总计 | ~1.6 W | ~200 mW |

功耗节省：~1.4 W/ch

6.2 延迟分析¶

延迟来源：

传统方案延迟：
- DAC延迟: ~5 ns
- Driver延迟: ~3 ns
- 调制器: ~1 ns
总计: ~9 ns

oDAC方案：
- 逻辑门延迟: <100 ps
- TFLN调制: ~10 ps
总计: <200 ps

节省: ~45×

6.3 带宽优势¶

带宽瓶颈对比：

传统eDAC瓶颈：
- 商用eDAC: ~60 GHz
- 频率交织eDAC: ~100 GHz (复杂)

oDAC优势：
- 3nm CMOS逻辑门: >100 GHz
- 简单架构
- 无外部复用器

八、技术深度¶

8.1 二进制加权原理¶

功率合成计算：

第i个MZM输出：
Pᵢ = P₀/2ⁱ

总功率：
P_total = Σ Pᵢ = P₀ × (1 - 2⁻ⁿ)/(1 - 1/2)
        = P₀ × (1 - 2⁻ⁿ)

n=7时：
P_total ≈ P₀ × 0.992

最大误差：<1%

8.2 光学合成损耗¶

插入损耗分析：

合成器损耗：~3 dB
MZM固有损耗：~2 dB
光纤耦合：~2 dB
总IL：~7 dB

这是可以接受的
