IEEE P802.3dj/D2.4 深度技术解读报告¶

IEEE Draft Standard for Ethernet — Amendment 12
Media Access Control Parameters for 1.6 Tb/s and Physical Layers and Management Parameters for 200 Gb/s, 400 Gb/s, 800 Gb/s, and 1.6 Tb/s Operation

文档版本: IEEE P802.3dj/D2.4, 27 January 2026
报告日期: 2026/04/17
研究员: Bert (with Kimi)
阅读对象: 高速 SerDes/光通信设计工程师、标准研究人员、系统架构师

执行摘要¶

IEEE P802.3dj 是 802.3 标准家族中首次同时覆盖 200G/400G/800G/1.6T 四代速率的修正案，定义了从芯片电接口到光纤链路的完整物理层规范。D2.4 草案（2026 年 1 月）是当前最新的工作版本，新增了 Clause 174 ~ Clause 187 以及 Annex 174A ~ Annex 186A，共计约 954 页。

本报告基于 D2.4 草案原文，结合 802.3ck (100G/lane 电接口)、802.3df (224G/lane) 等前代标准的演进脉络，对 802.3dj 的系统架构、PCS/PMA 设计、FEC 方案、电接口规格（KR/CR）、光接口规格（DR/SR/FR/LR/ER/ZR）以及管理接口进行系统性、逐层深入的解读。

第一章 802.3dj 在以太网演进中的战略定位¶

1.1 从 400G 到 1.6T：速率跃升背后的产业驱动力¶

时间 标准 关键里程碑 驱动力
2017 802.3bs 200G/400G SR16/DR4 云计算数据中心
2018 802.3cd 50G/100G/200G 5G 前传、服务器接入
2020 802.3cm 400G SR8 (MMF) AI 训练集群
2022 802.3ck 100G/lane 电接口 CPO/LPO 预研
2024 802.3df 800G/1.6T 框架确立 800G 光模块商用
2025~2026 802.3dj 200G~1.6T 全规格 AI 超算、3.2T 交换机

802.3dj 的发布时间点恰逢AI 大模型训练集群对互连带宽需求呈指数级增长的窗口期： - NVIDIA H100/B200 集群单节点已需要 400G/800G 多网卡配置 - 下一代 3.2T 交换芯片（如 Broadcom Tomahawk 6）需要 1.6T 光模块支持 - 1.6T = 16 × 100G 或 8 × 200G 成为两种主流实现路径

1.2 802.3dj 与前代标准的关系矩阵¶

802.3dj 并非独立标准，而是对 IEEE Std 802.3-2022 及其多个修正案（802.3dd, 802.3cs, 802.3db, 802.3ck, 802.3de, 802.3cx, 802.3cz, 802.3cy, 802.3df, 802.3-2022/Cor 1-2024, 802.3da, 802.3dk）的再次修正。

802.3-2022 (基础标准)
    ├── 802.3ck (100G/lane 电接口)
    ├── 802.3df (800G/1.6T 框架)
    └── 802.3dj (200G~1.6T 完整 PHY 规格) ← 本次焦点

关键继承关系： - 电接口：直接继承并扩展 802.3ck 的 100GBASE-CR1/KR1 规范，扩展到 8 lane 的 1.6TBASE-KR8/CR8 - 光接口：继承 802.3df 的 800G 光模块框架，新增 1.6TBASE-DR8/DR8-2/SR8/FR8/LR8/ER8 - PCS：基于 800GBASE-R PCS (Clause 162) 扩展为 1.6TBASE-R PCS (Clause 175)

1.3 802.3dj 的核心技术挑战¶

  1. ** lane 数倍增带来的 skew/延迟挑战**：1.6T 需要 8 lane × 200Gbps 或 16 lane × 100Gbps
  2. 功耗爆炸：1.6T 光模块功耗目标需控制在 20-25W（目前 800G 约 15W）
  3. FEC 级联复杂度：Inner FEC (KP4/RS) + Outer FEC (HFEC/SC FEC) 的延迟需满足 AI 网络要求
  4. 信号完整性：100G/lane 电信号在 PCB/铜缆上的损耗预算极紧
  5. 光器件集成：8 lane 激光器阵列、8 lane 探测器阵列的封装密度要求

第三章 1.6TMII / 800GMII 与 Extender 子层设计¶

3.1 为什么需要 1.6TMII？¶

随着 lane 数增加，传统的 MII 接口宽度需要扩展： - 100G 使用 64-bit 宽度的 XGMII-like 接口 - 400G 使用 64-bit 宽度的 400GMII - 800G 引入了 800GMII (Clause 167) - 1.6T 必须引入 1.6TMII (Clause 170) 以支持 8 lane × 200G 的数据流

3.2 1.6TMII 的核心参数¶

参数 400GMII 800GMII 1.6TMII
数据宽度 64-bit 64-bit 64-bit
控制信号 8-bit 8-bit 8-bit
时钟频率 ~6.25 GHz ~12.5 GHz ~25 GHz
每时钟周期传输 64-bit 64-bit 64-bit
等效位宽扩展 无 2× 400G 2× 800G
关键差异 4 lane PMA 8 lane PMA 16 lane PMA 或 8 lane 200G PMA

注意：1.6TMII 的数据宽度保持 64-bit，但时钟频率翻倍。然而在实际芯片实现中，这么高的时钟频率（25 GHz）不现实，因此需要通过 1.6TXS (Extender Sublayer) 将 1.6TMII 拆分为多个低速的 1.6TAUI-n 接口。

3.3 800GXS / 1.6TXS：解决芯片引脚瓶颈的关键¶

800GXS (800GMII Extender Sublayer, Clause 171) 和 1.6TXS (1.6TMII Extender Sublayer) 的功能是将高速但窄带宽的 MII 接口转换为低速但宽带宽的 chip-to-chip 接口。

1.6TXS 的两种工作模式¶

模式 A: DTE 1.6TXS (位于 MAC/RS 侧)

MAC/RS ←→ 1.6TMII ←→ DTE 1.6TXS ←→ 1.6TAUI-n ←→ PHY 1.6TXS ←→ PCS/PMA

模式 B: PHY 1.6TXS (位于 PHY 侧)

PHY 1.6TXS 将 1.6TAUI-n 重新组装为 1.6TMII 接口给 PCS

1.6TAUI-n 的 lane 配置¶

接口 lane 数 每 lane 速率 总带宽 应用场景
1.6TAUI-16 16 ~106 Gb/s 1.6 Tb/s CPO/共封装光学

工程意义： - 1.6TAUI-16 对应 CPO (Co-Packaged Optics) 场景，每 lane 106Gbps 与 800G 光引擎兼容 - 1.6TAUI-8 对应下一代 200G/lane 电接口，减少 lane 数但提高单 lane 速率

3.4 Skew 与 Delay 约束¶

第四章 PCS 设计：从 800GBASE-R 到 1.6TBASE-R¶

4.1 800GBASE-R PCS (Clause 172) 回顾¶

802.3dj 中的 800GBASE-R PCS 是对 802.3df 中 Clause 162 的继承和细化。

核心流程：

1.6TMII/800GMII 数据流
    ↓
64B/66B 编码 (每 64-bit payload + 2-bit sync)
    ↓
RS-FEC (KP4, RS(544,514)) 或 BCH 内码
    ↓
Alignment Marker (AM) 插入
    ↓
Scrambling (x^58 + x^39 + 1)
    ↓
Lane Distribution & Bit Interleaving
    ↓
PMA (8:32 或 16:64)

4.2 1.6TBASE-R PCS (Clause 175) 的创新点¶

核心挑战：1.6T 的数据量是 800G 的两倍，但 PCS 的处理时钟不能无限提升。

解决方案： 1. 并行度翻倍：800G PCS 使用 8 lane 并行处理，1.6T PCS 使用 16 lane 并行 2. 双 800GBASE-R PCS 拼接：1.6TBASE-R PCS 在概念上可以视为两个 800GBASE-R PCS 的协同工作 3. 统一的 AM 锁定机制：16 lane 的 Alignment Marker 需要同步锁定，否则 skew 会导致帧失步

1.6TBASE-R PCS 的架构框图¶

                    ┌─────────────────────────────────────────┐
                    │         1.6TBASE-R PCS                  │
                    │                                         │
 1.6TMII ───────────┤  RS ├─ 64B/66B Encoder ─┬─ Lane 0~7  ─┤─── PMA (8 lanes)
                    │       │                   ├─ Lane 8~15 ─┤─── PMA (8 lanes)
                    │       └─ RS-FEC Encoder ──┴─ AM Insert  │
                    │                                         │
                    └─────────────────────────────────────────┘

4.3 FEC 子层设计：Inner FEC + Outer FEC¶

802.3dj 引入了 两级 FEC 架构 (Clause 177)：Inner FEC + Outer FEC。

Inner FEC (内码)¶

参数 800G (802.3df) 1.6T (802.3dj)
码型 RS(544,514) KP4 RS(544,514) KP4
编码率 514/544 ≈ 94.5% 514/544 ≈ 94.5%
开销 5.5% 5.5%
纠错能力 t=15 t=15
处理粒度 每 lane 独立 每 lane 独立

为什么保持 KP4 不变？ - KP4 在 100G/lane 和 200G/lane 上已经被验证成熟 - 对于 1.6T (8 lane × 200G)，每 lane 独立进行 KP4 编解码可以重用 800G 的 IP

Outer FEC (外码)¶

对于长距离光传输（LR/ER/ZR），802.3dj 引入了 Outer FEC 以进一步提升净编码增益 (NCG)。

外码类型 应用场景 特性
Hamming Product Code (HPC) 早期草案 低延迟
Staircase Code (SC FEC) 相干光通信 高增益 (~10.5 dB)
Concatenated RS-BCH 超长距 极高增益

两级 FEC 的延迟预算： - Inner FEC (KP4): ~100 ns - Outer FEC (SC): ~2-5 μs (取决于交织深度) - 总延迟: 需满足 AI 网络 < 10 μs 的要求

4.4 Auto-Negotiation (AN) 与 APSU¶

802.3dj 扩展了 Clause 73 的 Auto-Negotiation 机制，支持 1.6TBASE 的速率协商。同时引入了 APSU (Autonomous Path Startup)，允许链路在无需主机 CPU 干预的情况下自动完成： 1. 信号检测 2. 对齐锁定 3. FEC 同步 4. 链路 up

APSU 的时序要求： - 从 PMD signal detect 到 link up: < 10 ms (电气), < 100 ms (光学)

第五章 PMA 子层：数据通路的"交通枢纽"¶

5.1 PMA 的功能定位¶

PMA (Physical Medium Attachment) 子层负责： 1. 串并转换 (SerDes)：将 PCS 的并行数据转换为 PMD 的高速串行流 2. 时钟恢复 (CDR)：从接收信号中恢复时钟 3. Lane Mapping：管理 lane 到物理通道的映射 4. Skew 补偿：吸收通道间的 skew 差异

5.2 802.3dj 的 PMA 类型¶

8:32 PMA¶

用于 800GBASE-R 和 1.6TBASE-R 的部分配置： - 输入：8 路来自 PCS 的数据流 - 输出：32 路 25G/lane 或 16 路 50G/lane - 实际上在 802.3dj 中，800G 更多使用 8:8 PMA (每 lane 100G)

16:64 PMA¶

这是 1.6TBASE-R 的核心 PMA 架构： - 输入：16 路数据流 (来自 PCS) - 输出：64 路物理 lane - 或者配置为 8:8 PMA 用于 200G/lane 场景

200G/lane PMA¶

对于 200GBASE-R (Clause 120 的扩展) 和 400GBASE-R，802.3dj 定义了 1:1 PMA，即每 lane 200Gbps 直接对应一个物理 SerDes。

5.3 200G/lane SerDes 的技术指标¶

802.3dj 定义的电接口速率有两种模式： 1. 100G/lane 模式：用于 1.6TAUI-16 (16 lane × 100G) 2. 200G/lane 模式：用于 1.6TAUI-8 (8 lane × 200G)

参数 100G/lane 200G/lane
调制格式 PAM4 PAM4
波特率 53.125 GBaud 106.25 GBaud
Nyquist 频率 26.5625 GHz 53.125 GHz
每 UI 时间 18.8 ps 9.4 ps
信道损耗预算 (PCB) ~28 dB @ 13.3 GHz ~20 dB @ 26.6 GHz
CDR 抖动容限 ~0.25 UI ~0.20 UI
FEC 开销 RS(544,514) RS(544,514)

工程难点分析： - 200G/lane 的 Nyquist 达 53 GHz，已经接近现有 CMOS/SiGe 工艺的极限 - 目前只有最先进的 3nm 工艺才能支持 200G/lane 的 SerDes - 因此 1.6T 的早期实现（2025-2026）更可能采用 16 lane × 100G 的方案

第六章 电气 PMD 规范：KR8 与 CR8¶

6.1 1.6TBASE-KR8：背板电接口 (Clause 178)¶

1.6TBASE-KR8 是 802.3dj 中最激进的电接口规范，将 8 lane × 200Gbps 的电信号驱动到背板/连接器上。

关键技术指标¶

发射机均衡 (FFE)¶

802.3dj 要求 KR8 发射机支持 4-tap FFE（含 1 个 pre-cursor, 1 个 main cursor, 2 个 post-cursor）：

H(z) = c_{-1}·z^{+1} + c_0 + c_1·z^{-1} + c_2·z^{-2}

c_2 (post2) -0.2 ~ 0 0.01

均衡的意义： - 背板的插入损耗在 26.6 GHz 处可达 28 dB - 仅靠 CTLE 无法充分补偿，需要发射端 FFE + 接收端 DFE 联合均衡 - 4-tap FFE 可以预补偿信道的频率响应，降低接收端复杂度

第七章 光学 PMD 规范：DR8 / SR8 / FR8 / LR8 / ER8 / ZR8¶

7.1 1.6T 光模块的多 lane 架构¶

1.6T 光模块采用 8 lane × 200Gbps 的并行架构： - 发射端 (Tx)：8 个激光器，每波长 200Gbps PAM4 - 接收端 (Rx)：8 个探测器，每通道 200Gbps - 波长分配： - DR/SR/FR/LR/ER：通常采用 CWDM4 或 LWDM 扩展（8 波长） - ZR：采用相干检测，单载波 1.6Tbps

7.2 1.6TBASE-DR8 / DR8-2：数据中心主流方案 (Clause 180 / 182)¶

DR8 是 1.6T 光模块中最具商业价值的类型，主要应用于数据中心内部的 Leaf-Spine 互联。

DR8 光口规格¶

参数 1.6TBASE-DR8 1.6TBASE-DR8-2

lane 数 8 8

波长范围 ~1310 nm ~1310 nm
发射光功率 (OMAouter) -2.4 ~ +4 dBm -2.4 ~ +4 dBm

TDECQ (发射机眼图闭合) < 3.4 dB < 3.4 dB

TDECQ 解读： - TDECQ (Transmitter and Dispersion Eye Closure for PAM4) 是 PAM4 系统的核心发射机指标 - 3.4 dB 意味着发射机眼图由于自身非线性和色散导致的闭合不能超过 3.4 dB - 对于 200G/lane PAM4，TDECQ 的挑战在于： - EML 激光器的带宽需 > 50 GHz - 驱动器的线性度要求极高 - 光纤色散在 2km 处可达 ~34 ps/nm (1310nm)，对 53 GBaud PAM4 影响显著

DR8-2 与 DR8 的差异¶

DR8-2 (2km) 相比 DR8 (500m) 的主要增强： 1. 更强的 FEC：可能需要 Outer FEC 补偿额外的 OSNR 损失 2. 更严格的色散容限：需要 pre-compensation 或 DSP 均衡 3. 更高的发射光功率：补偿 2km 光纤损耗 (~0.7 dB)

SR8 用于数据中心机柜内/相邻机柜的互联。

7.5 1.6TBASE-ZR8：超长距相干检测 (Clause 186)¶

ZR8 是 802.3dj 中最具突破性的规范，首次在以太网标准中定义了 80km 1.6T 相干光接口。

ZR8 的技术特征¶

参数 1.6TBASE-ZR8
距离 ≥ 80 km
调制 16QAM 或 PCS-16QAM
波特率 ~67-80 GBaud
载波 单载波 1.6T

为什么 ZR8 必须用相干检测？ - 80km SMF 的损耗约为 20 dB (0.25 dB/km) - 色散约为 1360 ps/nm，直接检测 (IM/DD) 的 PAM4 完全无法补偿 - 相干检测配合 DSP 可以： - 补偿 > 50,000 ps/nm 的色散 - 支持偏振模色散 (PMD) 补偿 - 利用本地振荡器 (LO) 提升接收灵敏度 ~15-20 dB

ZR8 与 OIF 400ZR/800ZR 的关系： - 802.3dj ZR8 直接借鉴了 OIF 400ZR 的 DSP 架构 - 主要差异：802.3dj 强调以太网帧格式和 MAC 速率，OIF ZR 更侧重光层互通 - 预计 ZR8 光模块将重用现有的 7nm/5nm 相干 DSP 芯片

第十章 总结与展望¶

10.1 802.3dj 的核心价值¶

IEEE P802.3dj 是以太网历史上最重要的修正案之一，其意义不仅在于将速率提升到 1.6T，更在于：

  1. 统一了 200G/400G/800G/1.6T 的完整规范家族
  2. 首次在以太网标准中引入 80km 相干检测 (ZR8)
  3. 为 CPO/LPO 等下一代封装形态提供了电接口标准基础
  4. 通过 1.6TXS/800GXS 解决了高 lane 数下的芯片 I/O 瓶颈

10.2 技术演进路线图¶

2024-2025: 800G 光模块规模商用 (802.3df)
    ↓
2025-2026: 1.6T 光模块样品 (802.3dj D2.4 → 正式发布)
    ↓

    ↓

10.3 对设计工程师的建议¶

  1. 短期 (2025-2026)：聚焦 100G/lane 技术，16 lane 方案是 1.6T 的主流实现路径
  2. 中期 (2026-2027)：关注 200G/lane SerDes 的成熟，8 lane 方案将降低光模块复杂度
  3. 长期：CPO + 相干 DSP 的融合将是 3.2T 时代的关键技术
