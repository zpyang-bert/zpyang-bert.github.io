对高带宽数据传输的不断增长的需求，推动了高性能通信系统领域的研究努力。人们已经探索了单芯片性能的研究，例如更快的多核处理器和更高的系统内存容量。为了进一步提升系统性能，研究重点放在了通过高速串行链路实现芯片间通信的数据传输带宽优化上。人们已经提出了许多解决方案来克服由非理想因素造成的瓶颈，例如连接两个链路设备的带宽受限的电气通道以及通信系统中各种不希望出现的噪声。然而，这些解决方案在低成本印刷电路板（PCB）材料上以受限功耗预算运行的多Gbps高速接口中，在时序裕量方面遇到了限制。因此，设计用于高速通信系统的物理层（PHY）链路的挑战在于实现低功耗、高可靠性和高成本效益。在此背景下，本论文旨在聚焦于 PHY 链路在高速通信系统中的架构设计、系统级和电路级验证，以及在功耗、可靠性和适应性方面的系统性能优化。

PHY 主要由时钟数据恢复（CDR）、均衡器（EQ）和高速 I/O 驱动器组成。PHY 链路的对称结构通常在两个链路设备中复制，以实现双向数据传输。通过在高速通信系统中引入训练机制，一个链路设备中的时序能够自适应地对准另一个链路设备中指定的时序条件，尽管由于各个链路中的工艺、电压和温度（PVT）变化会导致不同的偏斜或感应抖动。通过提供接口信号之间可靠的时序关系，系统总带宽得到了显著提高。另一方面，接口训练为复用提供了高灵活性，无需对高成本组件进行进一步研究。

在训练模式下，CDR 模块对于重建传输的比特流以获得最佳数据眼图以及在异步系统或源同步系统中检测数据流的边沿至关重要。通常，CDR 作为一个反馈控制系统工作，将其输出时钟与接收数据的中心对齐。在包含多个数据链路的系统中，整体 CDR 功耗随着链路数量的增加而线性增加，因为每个链路需要一个 CDR。因此，在具有并行链路的高功耗系统中，低功耗 CDR 起着重要作用。此外，高性能 CDR 要求在高输入抖动的情况下产生低抖动。为了最小化功耗和 CDR 抖动之间的权衡，提出了一种利用比例积分（PI）控制器和三次采样方案的新型 CDR 架构。

同时，当数据速率超过几Gbps 时，信号完整性（SI）变得至关重要。由于系统中的非理想因素导致的信号失真可能会严重降低信号质量，并在最坏情况下导致不可容忍的传输误差，从而影响系统的有效带宽。因此，为了信号完整性目的而进行的额外训练（如发射器（Tx）和接收器（Rx）均衡器训练）被加入到接口训练中。此外，提出并分析了一种在单个链路设备中非对称放置自适应 Rx 和 Tx 均衡器的简化系统架构，使用不同的系数自适应算法。该架构能够通过训练减少大量均衡器，特别是在并行链路的情况下。同时节省了相当大的功耗和芯片面积。

最后，讨论了针对 PVT 变化的高速 I/O 驱动器。关键问题（如过冲和下冲干扰数据）主要由 I/O 驱动器与其传输通道之间的阻抗失配引起。通过应用 PVT 补偿技术，I/O 驱动器阻抗可以有效地校准到接近目标值。实现了针对 PVT 变化的不同数字阻抗校准算法，并就比较快速校准和低功耗要求进行了比较。 \f摘要（德文）

对高带宽数据传输的不断增长的需求，推动了高性能通信系统领域的研究。人们已经对单个芯片的性能提升进行了深入研究，例如通过更快的多核处理器或更高的存储容量。为了进一步提升系统性能，研究重点放在了通过高速串行链路实现芯片间通信的数据传输带宽优化上。许多解决方案被提出，以防止由非理想因素引起的通信组件之间的瓶颈。这类非理想因素例如是电气通道的有限带宽或各种不希望出现的电气噪声。然而，这些解决方案通常对环境条件（如板材质量）有很高的要求，并且需要大量的能源消耗。在廉价板材或降低能源消耗的情况下，这些解决方案会导致高速接口传输速率的限制。因此，高速通信系统的 PHY 链路设计挑战在于实现节能、可靠和低成本。在此背景下，本论文聚焦于 PHY 链路的架构设计、系统级和电路级验证，以及在高速通信系统中关于能源消耗、可靠性和适应性的系统性能优化。

PHY 主要由时钟数据恢复（CDR）、均衡器（EQ）和高速 I/O 驱动器组成。PHY 链路的对称结构通常在两个通信组件中复制，以实现双向数据传输。通过使用训练机制，一个通信组件的时序可以尽管存在各种时序偏差或由制造工艺、电压和温度波动（PVT）引起的抖动，自适应地适应另一个通信组件的时序条件。通过接口信号之间可靠的时序关系，系统总带宽得到了显著提高。另一方面，接口训练为复用提供了高灵活性，无需对高成本组件进行进一步的研究。

在训练模式下，需要 CDR 模块来重建传输的比特流以获得最佳数据眼图，并确定异步或源同步系统中数据流的信号边沿。通常，CDR 作为一个闭环控制器工作，将其输出的时钟信号与接收到的数据居中对齐。在包含多个数据连接且每个连接需要独立的 CDR 的系统中，CDR 单元的总能耗随连接数量线性增长。因此，CDR 的能效在这种并行连接系统中起着重要作用。此外，高性能 CDR 必须在高输入抖动的情况下仅产生非常低的输出抖动。为了最小化能耗和 CDR 抖动之间的权衡，提出了使用比例积分（PI）控制器和三次采样的新型 CDR 架构。

当数据速率达到几Gbps 时，数据的完整性也变得非常重要。系统中的非理想因素导致的失真数据可能会严重降低信号质量，并在最坏情况下导致无法纠正的传输错误，从而负面影响系统的有效带宽。因此，为了优化信号完整性，发射器（Tx）和接收器（Rx）均衡器训练被作为接口训练额外使用。此外，提出并分析了一种使用不同系数自适应算法的单链路组件中自适应 Rx 和 Tx 均衡器的非对称排列的简化系统架构。该架构使得，特别是在并行连接的情况下，能够通过训练减少所需的大量均衡器。同时，这显著节省了硅面积并最小化了能源消耗。

最后，分析了高速 I/O 驱动器在不同 PVT 波动下的特性。关键问题，如以过冲和下冲形式影响数据的问题，主要由 I/O 驱动器与相应传输通道之间的阻抗失配引起。通过应用 PVT 补偿技术，I/O 驱动器的阻抗可以有效地匹配到预定的目标值。此外，实现并比较了针对 PVT 波动的不同数字阻抗校准算法，就其速度和能源消耗进行了比较。 \f目录

1.1 背景 随着多媒体点播和云计算使用的兴起，数据流量显著增长的趋势推动了对支持高带宽数据传输的高性能通信系统的持续需求。例如，9 年前以 5 Gbps [1] 速度运行的全集成背板收发器现在已达到 28 Gbps [2] 的速度。为了跟上摩尔定律的步伐，通信系统中的微处理器已转向多核甚至众核架构，因为单核处理器通过不断提高时钟速度已达到功耗墙。

PHY 链路，顾名思义，是从一个链路设备中的 PHY 到另一个链路设备中的 PHY 的连接。到目前为止，大多数针对高数据带宽的 PHY 链路设计研究仅按照其特定应用的标准协议进行，例如用于扩展卡的 PCI Express、用于无线基站的 RapidIO Serial。尽管通过使用先进的互连技术来最小化数据错误并满足协议要求已取得了许多改进，但在低成本印刷电路板（PCB）材料上，数据已面临时序裕量的限制。这些限制阻碍了现代通信系统中更高速的数据传输。

在工业中，通常针对不同需求进行定制设计，以确保在高数据速率下进行稳健和可靠的信号传输。然而，重新设计在成本和上市时间方面变得很成问题。在最坏情况下，可能导致随机时序抖动的工艺、电压和温度（PVT）变化以及随机噪声等不希望出现的问题可能会干扰系统。

\f4 第 1 章 引言与概述

最后但同样重要的是，额外先进互连电路的复杂性使 PHY 链路达到其总功耗的极限，从而降低了功率效率。PHY 链路中消耗的功率与 Tx 和 Rx 方向上的链路数量成正比，这意味着要增加系统带宽，但由于受限功耗预算，链路数量却受到约束。

因此，高速通信系统的 PHY 链路设计挑战在于实现自适应、可靠、低功耗和高成本效益。应用于 PHY 链路设计和优化的思路和动机如下。

• 高速通信系统 PHY 链路设计的低成本和适应性：构成 PHY 链路的组件（如 Tx EQ 和 Rx EQ）由于 PCB 板上的损耗通道，在各种高速通信系统中经常使用。为了节省成本，不考虑优化或新设计 PCB 板。因此，未来高速通信系统需要一种复杂的自适应机制，尽管有各种低成本 PCB 板，仍能实现更高速的数据传输。

• 高速通信系统 PHY 链路设计的可靠性：当数据速率增加到几Gbps 时，主要由采样时钟中的抖动引起的时序抖动会显著影响信号质量和数据传输速率。因此，可靠的高速数据传输取决于低抖动采样时钟的生成。

• 高速通信系统 PHY 链路的功耗感知设计：PHY 链路的设计和架构主要侧重于低功耗散发。单个 PHY 链路上功耗的降低为给定功耗预算下增加链路数量提供了可能性，从而增加整个系统带宽。因此，提出了一种基于低功耗的 PHY 链路优化和结构开发的思路。

• 优化 PHY 链路中 CDR 系统在高速数据传输的抖动产生和功耗之间的权衡[12]。提出在相位检测器（PD）中使用三次采样（3TS）方案来减少恢复采样时钟中的抖动。

3.1 现有技术 论文中提出了多种 CDR。[24] 中的作者建议使用基于模数转换器（ADC）前端的数字 CDR。接收的模拟信号首先通过连续时间模拟 EQ 进行均衡，然后其输出通过 flash ADC 转换为 5 位数字数据。在此架构中，通过应用过零 PD 调整有效数据数量而非采样时钟的相位。当数据更快时，插入一个额外数据。否则移除一个重复数据。[25] 中的作者在应用环路展开判决反馈 EQ（DFE）时提出了双阈值眼图跟踪 CDR。由于后 DFE 信号的边沿和眼图中心之间的时间差不等于 0.5UI，因此需要两个不同的阈值分别用于边沿和数据信号。此架构的缺点是其显著的电路复杂性。在[26] 中，作者建议使用动态抖动传输带宽控制的相位跟踪 CDR 以增加抖动容限。在[27] 中，相位跟踪 CDR 通过预检测器进行优化以减少闭环延迟。提供预检测器是为了改善由有限状态机（FSM）中频率检测器和累加器所需的更新周期引起的固有抖动。在预检测器中比较 FSM 中输入和输出的符号。[28] 中的作者在相位插值器设计中提出使用全差分方法的子速率相位跟踪 CDR，避免了时钟切换，从而产生更好的时钟抖动。

通常，CDR 被分为几类[29]，例如基于 PLL 的 CDR [30]、基于延迟锁定环（DLL）的 CDR [31] 和基于相位插值器的 CDR [32]。由于在相位和频率方面的双跟踪环，基于 PLL 的 CDR 提供了良好的输入抖动抑制。但与此同时，它受到高阶 PLL 的抖动累积和稳定性问题的影响。相反，基于 DLL 的 CDR 使用电压控制延迟线在稳定性分析中显示出其优点。然而，基于 DLL 的 CDR 具有有限的相位捕获范围，可以通过用相位插值器替换电压控制延迟线来解决。其他 CDR 如双边沿注入锁定 CDR [33]、波特率时序恢复[34]、具有睡眠模式的相位跟踪 CDR [35] 和使用符号率 PD 的 CDR [36] 也针对快速锁定或低功耗和面积消耗提出。

在不同类型的 CDR 结构中，基于相位插值器的 CDR 由于其许多优势而越来越受欢迎。基于相位插值器的 CDR 的主要优点是单个时钟源可被多个基于相位插值器的 CDR 电路共享。此功能降低了系统功耗、芯片尺寸开销以及相邻通道之间的串扰[37]。此外，基于相位插值器的 CDR 在相位跟踪期间提供短重新锁定时间、更好的噪声抑制和由于器件失配导致的偏移的简单补偿方案[38]。

3.4.1 小结

在本节中，我们展示了一种用于读/写训练的新型基于 PI 控制器的相位插值器 CDR。通过结合改进型和自适应模型的优点，在提出 CDR 中实现了功耗和相位抖动之间的良好权衡。自适应模型在所有情况下总体表现良好，除了功耗消耗加倍（由于需要四个插值器而非两个）。与自适应模型相比，改进型模型的缺点是不可能跟随由抖动引起的快速相位变化。PI2 CDR 结合了自适应和改进型模型的优点，可以使用与改进型模型几乎相同的功耗跟随快速相位变化。但提出 CDR 的缺点是在相位跟踪方面比自适应 CDR 慢一些，并且总体具有高抖动。为了解决提出 CDR 中的高抖动问题，建议使用带有改进 Alexander PD 的 3TS CDR。在 3TS CDR 的帮助下，系统能够检测恢复时钟是否与数据中间对齐以及读/写训练状态。 \f32 第 3 章 基于相位插值器的时钟数据恢复 \f第 4 章

4.5.4.1 仿真环境

图 4.40a 说明了传统的混合信号自上而下设计流程。该流程主要包括七个设计级别。它从系统概念的数学验证开始。一旦系统数学建模被验证，将执行基于行为模型的块级仿真。该级别使设计人员能够验证块级子系统是否满足系统级要求。在成功的块级仿真之后，系统被划分为数字和模拟部分。数字子流程包括综合、时序仿真、布局和布线等多个级别，而模拟子流程则遵循原理图设计和布局。最后，混合信号的设计布局在 ASIC 中集成和实现。

对于系统级仿真，Matlab/Simulink 框架由于其可用工具集和对算法实现的易用性已被广泛采用多年。许多努力已致力于将 Matlab 集成到不同应用领域的流行 EDA 工具流程中，例如 RFID [57]、电力电子[58]。B. Gestner 等人[59]开发了用于 RTL 调试和验证的 Modelsim-Matlab 接口。

对于行为级仿真，Verilog-AMS 已应用于混合信号设计环境。作为 Verilog-HDL 和 Verilog-A 的扩展，Verilog-AMS 在单个仿真器中提供准确的行为建模。P. Frey 等人[60]解释了 Verilog-AMS 在模拟和数字上下文之间的连接模块语义。

然而，传统设计流程的一个主要问题是它需要从 Matlab/Simulink 到 Verilog-AMS 的更高级别细化。尽管一些现有工具，如 Mathworks 的 Simulink HDL Coder 或 Synopsys 的 Synphony HLS，允许从 Matlab/Simulink 环境直接生成可综合的 HDL 代码，但由于其支持的块的限制，生成的 RTL 通常不适合高效的 FPGA 或 ASIC 实现。因此，"全定制"Verilog-AMS 行为建模通常仍具竞争力。为了受益于 Matlab/Simulink 和 Verilog-AMS 两者并最大限度地减少设计时间，本文提出了 Matlab/Simulink-HDL 共仿真。

共仿真是混合信号设计中不可或缺的。它允许各个组件通过同时运行的不同仿真工具以协作方式交换信息。此外，可以使用共仿真平台仿真不同抽象级别的不同组件。通过这种方式，可以实现设计加速和用系统级模型验证低级设计。

P. Daglio 等人[61]提出了一种 VHDL-AMS 方法，通过集成不同的 EDA 工具来探索不同抽象级别的混合信号领域。基于设计流程，嵌入式闪存宏单元被示例如作为测试案例。Verilog-AMS（混合信号设计的另一种 HDL）与 VHDL-AMS 进行了比较，如语言方面、结构表达、信号流语义等，其中安全气囊系统应用被演示作为案例研究。然而，两者都接近实现级别。它们相对较慢的用于系统验证的仿真器使它们对于复杂的片上系统应用效率低下。

5.2.2.1 混合线性搜索 一种混合阻抗校准的方法是应用线性搜索和二元搜索。如果角落的驱动器输出电压在模式电路中指定的电压窗口内，则应用线性搜索。对于线性搜索，使用三个具有恒定步长的额外分支。每个晶体管的宽度为 0.5W，与每个分支的电阻 4R 串联连接。线性步长设置得足够小，以校准那些接近 ±1%VREF 变化的角落。线性搜索从中心值开始，然后基于比较器的结果在 ±1%VREF 变化的方向上顺序遍历。在线性搜索期间，如果 PFET 输出电压小于 Reflow 或 NFET 输出电压大于 Refup，参考晶体管导通，所有校准晶体管关闭。对于其他情况，如果 PFET 输出电压大于 Refup 或 NFET 输出电压小于 Reflow，参考晶体管关闭，三个最高有效位晶体管同时导通作为参考。使用线性步长晶体管执行校准。

5.2.2.2 混合减量二元搜索 另一种混合阻抗校准是使用减量二元（RB）和二元搜索。RB 方法重用相同的校准晶体管。当模式信号为高时，校准逻辑仅使用最后三个校准晶体管来校准阻抗。在 RB 搜索期间，如果 PFET 输出电压小于 Reflow 或 NFET 输出电压大于 Refup，参考晶体管导通。三个最高有效位晶体管关闭。使用三个最低有效位晶体管执行校准。如果 PFET 输出电压大于 Refup 或 NFET 输出电压小于 Reflow，参考晶体管关闭。在这种情况下，三个最高有效位校准晶体管同时导通。这些晶体管作为参考，而剩余三个校准晶体管将电压拉回匹配电路范围内。 \f5.3 仿真结果 79
