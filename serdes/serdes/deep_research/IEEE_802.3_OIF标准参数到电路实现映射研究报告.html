IEEE 802.3/OIF 标准参数到电路实现映射深度研究报告¶

Phase: 3 (Engineering & Standards) — Task 3.1 研究范围: 标准参数到电路指标的翻译方法、TX/RX 性能容限的工程化表达、互操作性测试要求与验证方法、从草案到产品的标准化流程 参考来源: IEEE P802.3dj/D2.4、OIF CEI Specification

1. 研究背景¶

1.1 为什么标准参数到电路实现的映射是核心工程能力¶

IEEE 802.3 和 OIF 标准定义了交换芯片与光模块、电缆之间的"数字契约"——它们规定了接口处必须满足的电气/光学特性，但并不规定如何用电路实现这些特性。这种设计给了芯片架构师充分的灵活性，但同时也意味着：能否准确将标准规范转化为电路设计指标，直接决定了芯片能否通过标准合规测试。

一个典型的映射失败案例：224G SerDes 设计中，标准规定接收机灵敏度 > 50 mVpp（经过信道衰减后），但工程师没有意识到这个指标是在"考虑了均衡后"的，需要在 CTLE + DFE 联合作用下的等效灵敏度，而非 CTLE 输入端的要求。如果直接把 50 mVpp 当作 CTLE 输入灵敏度设计，接收机必然过设计（功耗浪费）或欠设计（性能不达标）。

1.2 本报告的核心问题¶

本报告试图回答三个工程问题： 1. 标准说的"接收机灵敏度 > X mV"到底对应 ADC 输入端的多少 mV？ 2. 标准规定"TDECQ < 3.4 dB"，TX 设计团队应该把这个分配到哪些 TX 子模块？ 3. OIF 的 TP1/TP4 测试点规范如何在芯片级 BIST 中实现自动化验证？

2. IEEE 802.3dj 规范体系与实现映射框架¶

2.1 802.3dj 规范层次结构¶

IEEE 802.3dj 规范从系统到电路分为四个层次，每一层都有明确的规范边界：

Layer 1: 系统架构层 (Clause 174)
  定义: 1.6T 交换机 → 1.6TMII → 1.6TXS → PCS/PMA → PMD → 光纤
  规范对象: 端口配置、lane 速率、拓扑结构
  电路映射: 无直接映射 — 这是系统架构决策

Layer 2: PCS/PMA 层 (Clause 172/175)
  定义: 64B/66B 编码、多lane分发、时钟同步
  规范对象: 误码率 (BER < 1e-13 after FEC)、skew、delay
  电路映射: FEC 编码器/译码器规格、clock domain crossing 设计

Layer 3: PMA/电接口层 (Clause 176/179)
  定义: 1.6TAUI-16、1.6TAUI-8 电接口电气规格
  规范对象: 眼图模板、插入损耗、回波损耗、抖动容限
  电路映射: TX driver FFE 规格、RX CTLE/DFE 规格 ← 核心映射区

Layer 4: PMD/光接口层 (Clause 180-186)
  定义: DR8/SR8/FR8/LR8/ER8/ZR8 光模块规格
  规范对象: OMA、灵敏度、TDECQ、色散容限
  电路映射: 光模块内部 DSP/TIA 规格（对 SerDes ASIC 间接影响）

关键认知：对于 SerDes ASIC 工程师而言，Layer 3（PMA/电接口层）是最直接相关的——这是芯片与光模块或背板之间的接口规范。

2.2 标准参数到电路指标的映射流程¶

将标准规范转化为电路设计指标的标准流程：

Step 1: 明确规范的对象（谁负责满足这个规范？）
  → TX 侧规范: 芯片 TX 端负责（1.6TAUI-n 输出）
  → RX 侧规范: 芯片 RX 端负责（1.6TAUI-n 输入到 PMA 输出）
  → 端到端规范: 需要 TX + 信道 + RX 联合满足

Step 2: 确定测试点位置
  → IEEE 802.3 的测试点定义:
    - TP1: 发射机输出（芯片引脚）
    - TP2: 信道输入
    - TP3: 信道输出
    - TP4: 接收机输入（芯片引脚）

Step 3: 分配规范预算（Budgeting）
  → 将端到端规范分配到 TX + 信道 + RX 三个子系统
  → 典型分配原则: TX 占 30%、信道占 40%、RX 占 30%

Step 4: 提取电路设计指标
  → 从分配的预算推算出具体的电路子模块指标

3. 核心 TX 指标的标准参数映射¶

3.1 眼图模板 (Eye Mask) 的工程解读¶

802.3dj 对 1.6TAUI-16 (106 Gb/s/lane) 规定的眼图模板定义了 TX 输出眼图的合法区域：

眼图模板（简化表示）:
  眼高 (Eye Height): > 50 mVpp (典型值，具体见规范)
  眼宽 (Eye Width): > 0.2 UI
  模板边界: 规定眼图中"不得进入"的区域

工程解读:
  - TX 输出眼图必须满足眼图模板，说明 TX driver + FFE 联合设计必须使
    输出眼图打开到一定程度
  - 眼高 50 mVpp 是"通过/不通过"的硬性指标
  - 对于 112 GBd PAM4 (UI = 8.93 ps)，眼宽 0.2 UI = 1.79 ps

TX 设计团队的眼图模板映射：

标准规范 TX 子模块指标 设计目标
眼高 > 50 mVpp TX driver 差分输出摆幅 > 100 mVpp (考虑到封装损耗)
眼宽 > 0.2 UI CDR 带宽/抖动跟踪 满足 Jitter Transfer 要求
眼图模板边界 TX FFE 预加重强度 主抽头 + pre/post tap 联合优化
上升/下降时间 Driver 带宽 > 0.7 × Baud Rate

3.2 TDECQ (发射机眼图色散闭合) — 光学 PMD 的 TX 指标¶

TDECQ (Transmitter and Dispersion Eye Closure for PAM4) 是 802.3dj 规定的光发射机眼图质量指标，专用于 PAM4 调制格式。

TDECQ 的物理含义： - 模拟发射机眼图经过指定光纤色散后的闭合程度 - TDECQ = 3.4 dB 意味着：PAM4 眼图经过标准光纤后，由于色散和非线性导致的眼图闭合，等效于 3.4 dB 的功率损失

对 SerDes TX 设计的直接映射：

TDECQ 规范 TX 设计指标 说明
TDECQ < 3.4 dB TX FFE 预加重 必须补偿光纤的频率选择性衰减
TDECQ < 3.4 dB EML/调制器线性度 驱动器的 THD 必须 < -30 dB
TDECQ < 3.4 dB 波长选择 1310 nm O-band 色散最小（0 ps/nm·km）
TDECQ 测试条件 2 km SSMF (DR8-2) TX 设计必须在此信道条件下验证

重要映射关系：TDECQ 是光模块侧的指标，但对 SerDes ASIC 的预均衡设计有直接影响——SerDes TX 的预加重必须与光模块的 TDECQ 能力匹配。设计时需要确认：光模块厂商给出的 TDECQ 预算中，有多少是留给主机侧 SerDes 预均衡来补偿的。

3.3 抖动规范与 TX PLL 设计¶

802.3dj 对 TX 输出的抖动有明确规定（Jitter Tolerance/Jitter Transfer）：

抖动分类： - 随机抖动 (RJ)：来自 PLL/时钟电路的热噪声，无界 - 确定性抖动 (DJ)：来自电路非理想性，有界（通常 < 0.1 UI） - 周期性抖动 (PJ)**：来自电源噪声或串扰，通常以正弦波形式出现

抖动传递函数 (Jitter Transfer)：

Jitter Transfer = 20*log10(fin/fout)
  - 低端 (< 1 MHz): 增益约 0 dB（PLL 跟踪）
  - 中端 (1-10 MHz): 可能出现峰值 (+/- 1 dB)
  - 高端 (> 10 MHz): 衰减 (> -20 dB/decade)

TX PLL 设计的关键参数映射：

标准抖动要求 PLL 电路指标 说明
Jitter Generation < 0.1 UI PLL RMS 抖动 < 1 ps RMS @ 106 Gb/s
Jitter Transfer 3 dB带宽 PLL 环路带宽 典型值 1-5 MHz
Jitter Tolerance (UI) CDR 跟踪能力 必须 > 0.4 UI @ 10 MHz

4. 核心 RX 指标的标准参数映射¶

4.1 接收机灵敏度 (Receiver Sensitivity)¶

802.3dj DR8 规范： - 接收灵敏度 (OMAouter) < -5.5 dBm（DR8 @ 212.5 Gbps PAM4） - 经过 500 m SSMF 光纤后，在 KP4 FEC 前 BER < 2.2e-4

这个灵敏度指标如何映射到 SerDes RX 电路设计？

映射链条：

目标: 2 km 光纤后，光功率 -5.5 dBm → 等效电信号 SNR > X dB

Step 1: 计算接收光功率对应的电 SNR
  - 2 km SSMF @ 212.5 Gbps 损耗约 1 dB
  - 探测器响应度 ~0.8 A/W
  - 光功率 -5.5 dBm = 0.28 mW → 光电流 ~0.22 mA
  - TIA 跨阻增益设计: 50 kΩ → 11 mVpp 输出
  - 放大器噪声等效: SNR ~15 dB

Step 2: FEC 前 BER < 2.2e-4 对应的 Q-factor
  - Q = 5.3 (对应 BER 2.2e-4)
  - 所需 CNR ~12 dB

Step 3: 映射到 ADC 设计指标
  - ADC 输入 SNR > 12 dB (考虑量化噪声)
  - ADC ENOB > 6.5 bit
  - ADC 采样率 > 212.5 GS/s (实际用 256+ GS/s)

4.2 COM (Channel Operating Margin) — 背板/铜缆的 RX 指标¶

COM 是 802.3dj 用于评估背板/铜缆信道质量的综合指标，定义在 KR/CR 规范中：

COM 的定义： $$COM = 10 \log_{10}\left(\frac{\text{峰值信号功率}}{\text{噪声 + 串扰功率}}\right)$$

802.3dj 规定 COM > 3 dB 是 KR8/CR8 系统的最低要求（在最大 ISI + 串扰条件下）。

COM > 3 dB 如何映射到 RX 均衡设计？

COM 要求 RX 均衡指标 说明
COM > 3 dB CTLE 增益 @ Nyquist 必须补偿信道在 26.6 GHz 处的损耗
COM > 3 dB DFE 抽头数 至少 1-tap DFE（处理残余 ISI）
COM 测试条件 28 dB 损耗 @ 26.6 GHz RX 设计必须在此信道条件下验证
COM 含串扰 RX 对串扰的抑制 需要 EQ 降低串扰影响

工程实现提示：COM 是一个系统级指标，涵盖了信道 + TX + RX 的联合效果。在设计 RX 均衡器时，工程师需要将"3 dB COM 剩余裕量"合理分配： - 如果信道损耗比标准信道更深，需要更强的 CTLE - 如果 TX 预均衡较弱，RX DFE 需要补更多的 post-cursor

4.3 抖动容限 (Jitter Tolerance) 与 CDR 设计¶

802.3dj 对接收机的抖动容限要求（KR8 为例）： - 必须能容忍 SJ > 0.1 UI @ 1 MHz 的正弦抖动 - 在 0.1 UI 的随机抖动下，BER 不得恶化超过 10×

抖动容限到 CDR 设计的映射：

抖动要求 CDR 环路参数 说明
容忍 0.1 UI @ 1 MHz CDR 环路带宽 必须 > 1 MHz 才能跟踪
跟踪正弦抖动 CDR 相位检测增益 决定了跟踪斜率
抖动容限测试 CDR 鉴相器类型 Hogge / Alexander / Half-Rate

224G 系统 (112 GBd PAM4) 的特殊问题： - UI = 8.93 ps - 0.1 UI = 0.893 ps（非常小） - 1 UI 的正弦抖动在 1 MHz 处对应的相位偏移：Δφ = 2π × 1 MHz × 8.93 ps ≈ 0.056 rad - CDR 必须在 1 MHz 处跟踪 0.056 rad 的相位偏移

5. OIF CEI-Linear 测试点与芯片级验证¶

5.1 CEI-Linear TP1/TP4 测试点体系¶

OIF CEI-Linear 定义了 Linear Drive (非重定时) 光模块的测试点体系，是芯片-光模块互联验证的核心参考：

主机 ASIC (TX)              光模块              主机 ASIC (RX)
      │                      │                      │
      │  TP1 (电气输出)       │  TP2 (光输入)        │
      ▼                      ▼                      ▼
  ┌──────────────────────────────────────────────┐
  │  CEI-Linear TX spec    │   Optical spec      │  ← 责任分界线
  └──────────────────────────────────────────────┘
      ▲                      ▲                      ▲
      │  TP4 (电气输入)       │  TP3 (光输出)        │
      │                      │                      │

各测试点的规范对象：

测试点 位置 规范方 关键规范参数
TP1 主机 TX 输出（芯片引脚） SerDes ASIC 眼图模板、插入损耗、回波损耗、抖动
TP2 光模块光输入 光模块 光功率承受范围、OMA、灵敏度
TP3 光模块光输出 光模块 消光比、线性度、眼图
TP4 主机 RX 输入（芯片引脚） SerDes ASIC 灵敏度、抖动容限、CTLE 增益

TP1/TP4 对芯片级 BIST 的直接指导意义： - TP1 规范定义了芯片 TX 输出必须满足的电气特性 → TX BIST 必须验证眼图模板合规 - TP4 规范定义了芯片 RX 输入必须能处理的信号质量 → RX BIST 必须注入指定幅度的衰减/抖动信号并验证 BER

5.2 芯片级 BIST (Built-In Self-Test) 的 TP1/TP4 验证实现¶

基于 TP1/TP4 规范，SerDes 芯片应实现以下 BIST 功能：

TX BIST (TP1 验证)：

PRBS 生成器 → TX → 测量眼图 → 与眼图模板比对 → PASS/FAIL

关键 BIST 参数:
- PRBS 类型: PRBS31Q (PAM4)
- 测量: 高速 ADC 采样 TX 输出眼图
- 眼图模板比对: 眼高 > 阈值, 眼宽 > 阈值
- 自动化: 上电自检, 无需外部仪器

RX BIST (TP4 验证)：

PRBS 注入 → 衰减信道模型 → RX → BER 计数 → PASS/FAIL

关键 BIST 参数:
- 注入信号: PRBS31Q (PAM4)
- 信道模型: 28 dB 损耗 + 反射 + 串扰 (模拟 KR8/CR8 信道)
- 衰减范围: 0 ~ 35 dB (验证灵敏度曲线)
- 通过标准: BER < 2.2e-4 (KP4 FEC 阈值)

5.3 互操作性测试的实际执行流程¶

互操作性测试是产品化的关键环节，需要在多个厂商设备之间验证兼容性：

多厂商互操作性测试矩阵：

测试配置 (以 800G DR4 为例):
  交换芯片厂商 A (1.6TAUI-16) ↔ 光模块厂商 B (800G DR4)
  交换芯片厂商 A (1.6TAUI-16) ↔ 光模块厂商 C (800G DR4)
  交换芯片厂商 D (1.6TAUI-16) ↔ 光模块厂商 B (800G DR4)
  交换芯片厂商 D (1.6TAUI-16) ↔ 光模块厂商 C (800G DR4)

OIF 800G-LR 测试要求： - 必须在规定的光纤类型 (SMF) 和距离 (10 km) 下验证 - 必须通过 BER 测试（KP4 FEC 前 < 2.2e-4） - 必须验证温度范围 (-40~85℃) 下的互通性

6. 从草案到产品的标准化流程¶

6.1 IEEE 802.3 标准的时间节点¶

IEEE 802.3 标准从草案到发布通常经历以下阶段：

Working Group Draft → Committee Draft → Sponsor Ballot → Final Standard
      (内部审议)          (投票)           (公开投票)         (发布)
         ↓                   ↓                 ↓              ↓
      12-18 月            6-12 月           3-6 月          1-3 月

对于 SerDes 设计团队的实际意义： - 标准草案（如 D2.4）在正式发布前可能仍有变更 - 设计必须留有足够的裕量应对规范微调 - 密切关注 IEEE 802.3 email list 获取最新修订信息

6.2 工程设计如何应对标准不确定性¶

策略 1: 保守设计 (Recommended) - 在所有关键指标上留 20% 的裕量 - 即使标准草案中的某个参数变化 ±10%，设计仍可满足 - 适用于量产芯片（不容忍 respin）

策略 2: 快速迭代 (适用于早期研究) - 先按当前草案实现 - 标准正式发布后快速 respin - 适用于非量产原型或预研项目

当前 802.3dj D2.4 的状态： - D2.4 是 2026 年 1 月的草案 - 预计 2026 年中会有更新的草案版本 - 设计团队应关注: 1.6TAUI-16 的具体眼图模板参数可能微调

6.3 OIF 实现 agreements (MSAs)¶

OIF 的多源协议 (MSA) 是比 IEEE 更快速的标准实现参考：

常见 OIF MSAs： - 800G-ZR: 面向 DCI 的 800G 相干光模块（已商用） - 800G-OR: 面向 OpenROADM 的 800G PCS-16QAM（OFC 2026 互通验证） - CEI-224G: 正在制定的 224 Gbps 电气接口规范

OIF MSAs 对设计的指导价值： - MSAs 通常定义了比 IEEE 更详细的实现细节 - 加入 OIF MSAs 可以提前获取规范信息（在公开发布之前） - 主流 SerDes 供应商（Broadcom、Marvell、Intel）通常积极参与 OIF

7. 224G 系统实现的特殊映射挑战¶

7.1 112 GBd PAM4 的关键映射差异¶

224G/lane (112 GBd PAM4) 相比 112G/lane (56 GBd PAM4) 的映射挑战：

参数 56 GBd PAM4 (112G) 112 GBd PAM4 (224G) 变化影响
UI 17.86 ps 8.93 ps 时序预算减半
Nyquist 频率 28 GHz 56 GHz 封装带宽要求翻倍
信道损耗 ~25 dB ~35 dB 均衡深度增加
眼高 (最差眼) ~50 mV ~25 mV ADC 分辨率要求更高
TDECQ ~2.5 dB ~3.4 dB TX EOM 线性度更严苛

7.2 112 GBd PAM4 的 COM 映射示例¶

具体计算: KR8 @ 112 GBd 信道

信道损耗 @ 56 GHz: 35 dB
允许的 TX 输出: 1 Vpp (差分)
TX 预加重后经过信道: 约 35 mVpp 到达 RX 输入

CTLE 需要提供: ~20 dB 增益
DFE 处理残余 ISI: ~15 dB 等效
ADC 输入 SNR: > 12 dB

→ CTLE 设计指标:
  - 增益范围: 0-25 dB
  - 带宽: > 60 GHz
  - 噪声谱密度: < 10 nV/√Hz

→ ADC 设计指标:
  - 采样率: > 112 GS/s (实际 128-256 GS/s)
  - ENOB: > 6.5 bit
  - 输入带宽: > 56 GHz

8. 核心结论汇总¶

维度 关键结论
规范层次 IEEE 802.3dj 从系统到电路分四层，SerDes 工程师直接相关的是 PMA/电接口层 (Clause 176/179)
TX 眼图模板 眼高 > 50 mVpp、眼宽 > 0.2 UI 是硬性通过条件，映射到 TX FFE 预加重和 Driver 带宽设计
TDECQ 光模块指标，但对 SerDes TX 预加重设计有直接指导——两者必须联合优化
COM > 3 dB 背板/铜缆系统级指标，映射到 CTLE 增益 > 20 dB @ Nyquist 和至少 1-tap DFE
TP1/TP4 OIF 定义的测试点体系是芯片-光模块互联验证的核心参考，芯片 BIST 必须覆盖 TP1/TP4 规范
224G 特殊性 UI 8.93 ps 使时序预算减半、ADC ENOB > 6.5 bit 是刚需、封装带宽必须 > 56 GHz
标准不确定性 D2.4 仍是草案，设计必须留 20% 裕量应对规范微调

🔍 深度说明：标准参数到电路指标的映射，是 SerDes 工程师最核心但也最容易出错的工程能力。常见陷阱有两种：第一种是"过度设计"——把标准规范的余量全部吃光，不给自己留任何裕量，结果功耗/面积超标；第二种是"欠设计"——没有正确理解标准规范的边界条件，在某些 corner case 下无法满足要求。正确的做法是：先严格理解标准规范的测试条件（在什么信道下测？用什么 FEC？温度范围？），再确认设计是否在所有 PVT 角下都有足够裕量。对于 224G 系统，建议建立完整的"标准-电路指标-验证方法"三对照表，在设计初期就明确每一项电路指标的唯一来源。
