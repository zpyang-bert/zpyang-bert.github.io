CDR (时钟数据恢复) 技术详解¶

创建日期: 2026/04/15 更新日期: 2026/04/15 分类: SerDes 基础 / CDR 标签: #serdes #cdr #jitter #pll

1. CDR 基本原理¶

1.1 什么是时钟数据恢复¶

CDR (Clock Data Recovery) 是高速串行通信中的核心技术，用于从高速数据流中恢复出发送端的时钟信息，而无需传输独立的时钟信号。

┌─────────────────────────────────────────────────────────────────┐
│                      CDR 在 SerDes 中的位置                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   发送端                                        接收端             │
│  ┌─────────┐                                  ┌─────────┐        │
│  │   TX    │─────── 串行数据流 ────────▶      │   RX    │        │
│  └─────────┘                                  └────┬────┘        │
│                                                  │              │
│                    ┌─────────────────────┐       │              │
│                    │      CDR 电路        │◀──────┘              │
│                    │  (时钟数据恢复)       │                      │
│                    └─────────────────────┘                       │
│                            │                                     │
│                            ▼                                     │
│                    恢复出时钟 + 数据                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

1.2 CDR 的必要性¶

传统方式 CDR 方式
独立时钟线传输 无独立时钟，利用数据编码
时钟偏移问题 自适应锁定
多根走线 节省信道资源
高速下难以实现 适用于 10Gbps+ 速率

1.3 典型的 NRZ 数据编码¶

原始数据:    1  0  1  1  0  0  1  0
NRZ 编码:   ┌─┐   ┌─────┐       ┌─┐   ┌─┐
            │ │   │     │       │ │   │ │
            │ │   │     │       │ │   │ │
            └─┘   └─────┘       └─┘   └─┘
                    时间 ──────────────────▶

数据中隐含时钟信息 (跳变沿可用于时钟恢复)

2. CDR 架构分类¶

2.1 按架构类型分类¶

CDR 架构
├── 线性 CDR (Linear CDR)
│   ├── 半速率 (Half-rate)
│   └── 全速率 (Full-rate)
├── 突发模式 CDR (Burst-mode CDR)
└── 数字 CDR (Digital CDR)
    ├── 基于 PLL 的数字 CDR
    └── 无 PLL 的数字 CDR

2.2 常见 CDR 架构¶

2.2.1 Bang-Bang CDR (非线性 CDR)¶

特点: - 最常用的架构 - 使用相位检测器 (Phase Detector, PD) - 根据数据跳变沿调整时钟相位 - "Bang-bang" 描述其控制特性：加速/减速

架构图:

数据输入 ──▶ 数据采样器 ──▶ DFE/均衡 ──▶ 输出
              ▲
              │
        ┌─────┴─────┐
        │ 相位检测器 │
        │ (PD)      │
        └───┬─────┘
            │
            ▼
      ┌──────────┐
      │   VCO    │
      │ (压控振荡)│
      └──────────┘
            │
            ▼
      恢复时钟输出

2.2.2 Hogge CDR (线性 CDR)¶

特点: - 线性相位检测 - 更精确的相位跟踪 - 用于高阶应用

相位误差计算:

相位误差 = 数据边沿位置 - 采样时钟边沿
误差方向: 超前 / 滞后
误差大小: 线性量化

2.2.3 亚历山大 CDR (Alexander CDR)¶

特点: - 使用三个采样点 - 广泛应用于 USB、HDMI 等

三个采样点: 1. 数据采样 (Data): 采样数据眼图中心 2. 过渡采样 (Transition): 采样数据边沿 3. 早/晚判定: 比较数据与过渡采样

3. CDR 关键电路模块¶

3.1 相位检测器 (Phase Detector, PD)¶

Bang-Bang PD 工作原理:

数据边沿到来
    │
    ▼
比较恢复时钟与数据边沿
    │
    ├── 时钟边沿在数据边沿之前 ──▶ "过早" (UP)
    ├── 时钟边沿在数据边沿之后 ──▶ "过晚" (DOWN)
    └── 时钟边沿对准数据边沿   ──▶ "锁定"

输出信号: - UP: 时钟需要超前 - DOWN: 时钟需要滞后

3.2 电荷泵 (Charge Pump)¶

作用: 将 PD 的数字输出转换为 VCO 的模拟控制电压

UP ──┐
     ├───▶ 充放电电流 ──▶ 控制电压
DOWN ─┘

关键参数: - 充放电电流匹配 - 注入/抽取瞬态响应 - 泄漏电流抑制

3.3 VCO (压控振荡器)¶

类型: - LC-VCO: 高频低噪声 - Ring-VCO: 面积小，可调范围宽

调谐曲线:

频率
  ▲
  │         /
  │        /
  │       /
  │      /
  │     /
  │    /
  │   /
  │  /
  └──┼──────────────────▶ 控制电压

3.4 分频器 (Divider)¶

作用: 将 VCO 输出分频，得到需要的时钟频率

典型配置: - 输入: VCO 频率 (如 10GHz) - 输出: 数据率的 1/2 或 1/4

4. CDR 性能指标¶

4.1 抖动传递函数 (Jitter Transfer)¶

定义: 输出抖动与输入抖动的比值，描述 CDR 对抖动的放大/抑制能力

关键指标: - 抖动带宽 (Jitter Bandwidth): CDR 跟踪输入抖动的-3dB 带宽 - 抖动峰值 (Jitter Peaking): 传递函数的最大增益

           增益 (dB)
             ▲
             │    ┌──┐
             │    │  │  抖动峰值
             │    │  │
   0 dB ─────┼────┤  ├───────────────
             │    │  │
             │    └──┘
             │
             └────────────────────────▶ 频率
                    ↑
              抖动带宽点

4.2 抖动产生 (Jitter Generation)¶

定义: CDR 本身产生的抖动（内部噪声源）

来源: - VCO 相位噪声 - 电荷泵噪声 - PD 非线性 - 电源噪声耦合

4.3 抖动容限 (Jitter Tolerance)¶

定义: CDR 能接受的最大输入抖动，而不会产生比特错误

要求: - 高于规范规定的最小值 - 典型值: 0.2 UI (Unit Interval) @ 低频

4.4 锁定时间 (Lock-in Time)¶

定义: CDR 从初始状态到锁定所需的时间

影响因素: - 环路带宽 - 初始频率误差 - 环路增益

5. CDR 设计挑战¶

5.1 高速率挑战¶

速率 主要挑战
10 Gbps 基本架构可用
28 Gbps PD 速度、功耗
56 Gbps 抖动容限、CTLE 配合
112 Gbps 极限设计、PAM4 CDR

5.2 PAM4 CDR 特殊考虑¶

PAM4 信号特点: - 三个眼图（1.5 UI） - 两个判决电平 - 过渡沿更密集

PAM4 CDR 设计要点: - 多级判决 - 符号间干扰 (ISI) 处理 - 载波同步

5.3 抖动性能优化¶

技术手段: 1. 低噪声 VCO: 使用 LC-VCO 或优化 Ring-VCO 2. 高阶环路: 增强高频跟踪能力 3. 校准电路: 补偿 PVT 变化 4. 数字辅助: DSP 辅助 CDR

6. CDR 与 PLL 的关系¶

6.1 相似性¶

  • 都是锁相环路
  • 都基于反馈控制
  • 都使用 VCO 和分频器

6.2 差异¶

方面 CDR PLL
输入 数据流（含边沿） 独立参考时钟
目标 跟踪数据边沿 锁定参考时钟
鉴相器 边沿检测型 PD 传统相位/频率检测
应用 串行接收机 时钟产生/分配

7. 实际应用¶

7.1 PCIe CDR¶

架构: 半速率 Bang-Bang CDR

关键参数: - 128b/130b 编码 - 8GT/s (PCIe 3.0) - 抖动预算: < 1ps RMS

7.2 Ethernet CDR¶

架构: 完整速率 CDR

标准要求: - IEEE 802.3 以太网 - 25G/50G/100G/200G/400G/800G - RS-FEC 后 CDR

7.3 USB CDR¶

架构: 亚历山大 CDR

特点: - 高速: 5Gbps / 10Gbps - 低功耗 - 集成在 PHY 中

8. CDR 电路设计要点¶

8.1 设计流程¶

1. 确定规格
   ├── 速率/协议
   ├── 抖动预算
   └── 功耗限制

2. 架构选择
   ├── CDR 类型
   ├── 环路带宽
   └── 功耗预算

3. 模块设计
   ├── PD 设计

8.2 关键注意事项¶

  1. PVT 鲁棒性: 确保全工艺角、温度、电压下正常工作
  2. 功耗 vs 性能: 权衡抖动带宽与功耗
  3. EMI/RFI: 注意射频干扰对 CDR 的影响
  4. 测试性: 设计 DFT 便于生产测试
