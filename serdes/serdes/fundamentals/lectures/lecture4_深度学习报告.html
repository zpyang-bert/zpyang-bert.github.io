ECEN720 Lecture 4: Channel Pulse Model & Modulation Schemes — 深度学习报告¶

课程: ECEN720: High-Speed Links Circuits and Systems 授课人: Sam Palermo, Texas A&M University 整理日期: 2026/04/16 研究深度: 深度解读（结合信道脉冲响应模型、峰值失真分析、NRZ/PAM-4调制对比、眼图仿真）

一、信道脉冲响应与ISI¶

1.1 NRZ数据建模¶

NRZ信号数学表示：

NRZ数据流建模为孤立"1"和"0"的叠加

比特"1"的单位脉冲:
c_k(t) ≡ u(t - kT) - u(t - (k+1)T)

比特"0":
c_k(t) = -c_k^{(1)}(t)

其中 u(t) 为单位阶跃函数:
u(t) = 1, t ≥ 0
u(t) = 0, t < 0

完整数据流:
V(t) = Σ d_k · c_k(t - kT)
       k=-∞
其中 d_k ∈ {+1, -1} 表示第k个比特

信道脉冲响应：

输入脉冲 → 信道 → 输出脉冲响应

数学表示:
y_k(t) = c_k^{(1)}(t) ⊛ h(t)

其中:
• c_k^{(1)}(t): 输入单位脉冲
• h(t): 信道冲激响应
• y_k(t): 输出脉冲响应

信道输出为所有比特脉冲响应的叠加:
V_out(t) = Σ d_k · y_k(t - kT)
           k=-∞

1.2 脉冲响应结构¶

Pre-cursor / Cursor / Post-cursor ISI：

🔍 深度说明： 【研究背景】该图展示了信道脉冲响应的采样结构，是分析ISI（码间干扰）的核心工具。脉冲响应的主峰（cursor）表示当前比特，而前标（pre-cursor）和后标（post-cursor）表示相邻比特对当前比特的干扰。 【核心结论】y^(1)(t)在t=0采样值为0.540（cursor），k=-1处为0.036（pre-cursor ISI），k=+1处为0.165（post-cursor ISI）；这些采样值直接决定了FFE/DFE的tap设计。 【工程价值】通过脉冲响应可以计算峰值失真（peak distortion），预测最坏情况眼图；这是高速链路设计验证的标准流程。 【落地注意】实际信道脉冲响应需要通过VNA测量S参数后做IFFT得到；测量带宽决定了时间分辨率（15GHz带宽→~33ps分辨率）。

1.3 信道"FIR"模型¶

离散时间等效模型：

脉冲响应采样值构成FIR滤波器系数:

y^{(1)}(t) 采样得到:
a = [..., a_{-2}, a_{-1}, a_0, a_1, a_2, a_3, ...]
  = [..., 0.003, 0.036, 0.540, 0.165, 0.065, 0.033, ...]
  k=        -2     -1      0      +1     +2     +3

FIR滤波表示:
V_out(t) = Σ d_k · a_{m-k} · p(t - mT)
           m,k=-∞

等效于:
• 抽头延迟线
• 每比特周期采样一次
• 抽头系数 = 脉冲响应采样值

二、峰值失真分析¶

2.1 最坏情况眼高估计¶

理论框架：

峰值失真分析(Peak Distortion Analysis):
→ 从脉冲响应预测最坏情况眼图

最坏情况"1":
s_1(t) = y^{(1)}(t) + Σ y^{(1)}(t - kT)
         k=-∞, k≠0
         y(t-kT)<0

(1脉冲) + (所有负后标贡献)

最坏情况"0":
s_0(t) = y^{(0)}(t) + Σ y^{(0)}(t - kT)
         k=-∞, k≠0
         y(t-kT)>0

(-1脉冲) + (所有正后标贡献)

眼高计算：

最坏情况眼高:
Eye_Height(t) = s_1(t) - s_0(t)

代入 y^{(0)}(t) = -y^{(1)}(t):

Eye_Height(t) = y^{(1)}(t) - (-y^{(1)}(t))
               + Σ_{y<0} y^{(1)}(t-kT) - Σ_{y>0} (-y^{(1)}(t-kT))
               + Σ_{y<0} y^{(1)}(t-kT) - Σ_{y>0} y^{(1)}(t-kT)

简化:
= 2 × y^{(1)}(t)
  + 2 × Σ_{y<0} y^{(1)}(t-kT)
  - 2 × Σ_{y>0} y^{(1)}(t-kT)

= 2 × [ y^{(1)}(t)
       + Σ_{y<0} y^{(1)}(t-kT)
       - Σ_{y>0} y^{(1)}(t-kT) ]

其中:
• 第一项: cursor位置的主脉冲
• 第二项: 所有负post-cursor的叠加
• 第三项: 所有正post-cursor的相减

2.2 峰值失真示例1¶

案例1分析：

🔍 深度说明： 【研究背景】该图展示了峰值失真分析的完整推导过程。通过脉冲响应采样值可以预测最坏情况眼高，无需遍历所有可能的数据模式。 【核心结论】当cursor=0.540，Σ负后标=-0.007，Σ正后标=0.389时，最坏情况眼高=2×[0.540+(-0.007)-0.389]=0.288，归一化值仍为正，眼图张开。 【工程价值】峰值失真分析可以在实际硬件测试前预测信道性能，是SerDes设计验证的重要分析工具。 【落地注意】该分析方法假设线性时不变系统，适用于NRZ调制；对于PAM-4等高阶调制，分析复杂度显著增加。

最坏情况比特模式：

从脉冲响应推导最坏情况模式:

脉冲系数: [..., a_{-2}, a_{-1}, a_0, a_1, a_2, a_3, ...]
              0.003   0.036   0.540  0.165  0.065  ...

最坏情况模式:
• 在cursor位置保持"1"
• 翻转所有正抽头位置为"0"
• 翻转所有负抽头位置为"0"(实际上正负都需考虑)

最坏模式 = -sign(a_{-k}) 当 k≠0, 符号翻转

结果: 产生最大ISI叠加

2.3 峰值失真示例2¶

案例2分析(更差情况)：

脉冲响应采样值:
y^{(1)}(t=0) = 0.426

所有负采样值的和(不包括k=0):
Σ_{y<0} y^{(1)}(t-kT) = -0.053

所有正采样值的和(不包括k=0):
Σ_{y>0} y^{(1)}(t-kT) = 0.542

最坏情况眼高:
s(t) = 2 × [0.426 + (-0.053) - 0.542]
     = 2 × (-0.169)
     = -0.338

眼高为负值 → 眼图完全闭合!
→ 这个信道无法支持可靠的NRZ传输

2.4 峰值失真分析的意义¶

设计工具价值:
• 从脉冲响应预测最坏情况性能
• 不需要遍历所有可能数据模式
• 快速评估信道是否满足BER要求

关键发现:
• 正后标(post-cursor)会减小眼高
• 负后标会增大眼高(在某些情况下)
• 最坏情况取决于脉冲响应符号分布
• 当Σ正 - Σ负 > cursor/2时，眼图闭合

三、NRZ(PAM-2) vs PAM-4调制¶

3.1 调制格式对比¶

NRZ/PAM-2：

🔍 深度说明： 【研究背景】该图对比了NRZ和PAM-4两种调制格式的信号电平。NRZ为2电平（1bit/symbol），PAM-4为4电平（2bit/symbol），是当前高速链路的两大主要调制方案。 【核心结论】NRZ的优点是电压裕量大（眼高=2A）、接收机简单（1个比较器）；PAM-4的优点是带宽效率提升2倍，但眼高仅为2A/3，SNR代价约9.5dB。 【工程价值】在损耗斜率>9.54dB/octave的高损耗信道中，PAM-4可能优于NRZ；在低损耗信道中，NRZ的电压裕量优势更明显。 【落地注意】PAM-4接收机需要3个比较器（检测±A/3和0三个阈值），且阈值需根据信道特性自适应调整，复杂度显著高于NRZ。

PAM-4：

特点:
• 4电平调制
• 2 bits/symbol
• 谱效率提高2倍
• 适用于高阶调制需求

信号电平:
• "11": +A
• "01": +A/3
• "00": -A/3
• "11": -A

带宽效率:
• Nyquist频率 = R_s/4 = R_b/4
• 带宽需求减半
• 但: SNR代价约9.5dB

3.2 频谱效率对比¶

NRZ带宽需求:
• 10Gb/s → 5GHz Nyquist
• 信号主瓣在1/T到2/T之间

PAM-4带宽需求:
• 20Gb/s (等效) → 2.5GHz Nyquist
• 带宽需求减半

频谱图对比:
NRZ: 主要功率在1GHz带宽内
PAM-4: 主要功率在0.5GHz带宽内

适用场景:
• 损耗斜率 > 20dB/octave → 考虑PAM-4
• 通道插入损耗 > -9.54dB/octave → PAM-4可能更优

3.3 NRZ vs PAM-4决策指南¶

损耗斜率准则：

损耗斜率 > -9.54dB/octave (每倍频程)?

├─ 否:
│   → NRZ/PAM-2 足够
│   → NRZ有更好的电压裕量
│
└─ 是:
    → PAM-4 值得考虑
    → 但需要评估以下因素:
       • PAM-4接收机复杂度增加
       • 3x比较器数量
       • 阈值生成依赖信道
       • CDR抖动增加
       • 串扰敏感性更高

3.4 NRZ vs PAM-4案例对比¶

Desktop通道(低损耗)：

损耗特性:
• @2.5GHz: -4.8dB
• @5GHz: -7.5dB
• 倍频程损耗: ~2.7dB

结果:
• 眼图用4-tap TX FIR均衡
• NRZ电压裕量优于PAM-4
• 建议: NRZ/PAM-2

原因: 损耗斜率不陡峭，NRZ电压裕量更大

T20服务器通道(高损耗)：

损耗特性:
• @2.5GHz: -11.1dB
• @5GHz: -26.9dB
• 倍频程损耗: ~15.8dB

结果:
• 眼图用4-tap TX FIR均衡
• PAM-4"可能"是更好选择

原因: 损耗斜率超过9.54dB/octave

四、PAM-4接收机与峰值失真¶

4.1 PAM-4接收机架构¶

PAM-4接收机复杂度:

比较器数量:
• NRZ: 1个比较器 (差分放大器)
• PAM-4: 3个比较器 (阈值检测)

原因:
• 需要检测4个电平
• 3个阈值电平: ±A/3, 0
• 等效于2-bit ADC

阈值电平生成:
• 不能简单固定在0V
• 必须根据信道损耗和TX均衡自适应
• 信道损耗越大，阈值偏移越复杂

4.2 PAM-4眼图¶

NRZ vs PAM-4眼图对比：

🔍 深度说明： 【研究背景】该图对比了NRZ和PAM-4的眼图结构。NRZ有1个眼（眼高=2A），PAM-4有3个眼（眼高=2A/3），PAM-4的眼高只有NRZ的1/3。 【核心结论】PAM-4的三个眼分别对应4个电平的三个判决区域，眼高公式为2×(A/3 - 损耗)；在相同信道损耗下，PAM-4的电压裕量约为NRZ的1/3。 【工程价值】PAM-4的较低眼高意味着对噪声更敏感，需要更好的信噪比；在设计PAM-4 SerDes时，均衡器和CDR的性能要求比NRZ更严格。 【落地注意】PAM-4眼图的三个眼可能不完全对称，取决于信道损耗的非线性；实际测试时需要验证所有三个眼的张开度。

4.3 PAM-4峰值失真分析¶

PAM-4脉冲响应分析:

最坏情况电平确定:
• 需要考虑所有可能符号组合
• 16种符号对(4×4)
• 比NRZ复杂得多

最坏眼高:
• 多个判决边界可能闭合
• 串扰影响更显著
• 需要考虑maximum transition pattern

关键发现:
• PAM-4调制对残余ISI更敏感
• 同样的脉冲响应，PAM-4眼图更易闭合

五、多阶PAM挑战¶

5.1 接收机复杂度¶

PAM-4 vs NRZ接收机对比:
┌──────────────────┬─────────┬─────────┐
│ 指标              │ NRZ    │ PAM-4  │
├──────────────────┼─────────┼─────────┤
│ 比较器数量        │ 1       │ 3       │
│ ADC精度          │ 1-bit   │ 2-bit   │
│ 阈值生成         │ 简单    │ 复杂    │
│ CDR抖动          │ 基准    │ 更大    │
│ 串扰敏感性        │ 低      │ 高      │
└──────────────────┴─────────┴─────────┘

5.2 阈值生成挑战¶

自参考 vs 固定阈值:

NRZ:
• 差分信号自参考在0V
• 比较器天然检测过零点
• 简单可靠

PAM-4:
• 需要生成3个阈值电平
• 阈值依赖:
   • 信道损耗量
   • TX预加重程度
   • 共模噪声
• 任何阈值误差直接导致判决错误

5.3 CDR抖动增加¶

NRZ时钟恢复:
• 每个比特一个边沿
• 边沿密度高
• CDR跟踪容易

PAM-4时钟恢复:
• 某些电平转换可能无边沿(如01→00)
• 有效边沿密度降低
• CDR跟踪更困难
• 额外抖动来源

5.4 串扰敏感性¶

PAM-4串扰问题:

串扰影响:
• 串扰电压叠加在信号上
• NRZ: 串扰可能推过判决阈值
• PAM-4: 串扰更容易推过阈值(眼高更小)

最大转换模式(Maximum Transition):
• 邻近通道同时转换 → 最大串扰
• PAM-4更易触发
• FEXT/NEXT影响更严重

5.5 高级均衡的补偿能力¶

NRZ + 高级均衡的潜力:

关键发现:
• 先进DFE可以补偿 >9.5dB/octave损耗
• NRZ + MLSE可能优于PAM-4

均衡复杂度权衡:
• NRZ需要更复杂的均衡器(MLSE)
• PAM-4需要更复杂的接收机
• 取决于工艺和设计成熟度

选择依据:
• 片上时钟速度限制 → PAM-4
• 均衡器设计成熟度 → NRZ+DFE
• 功耗预算 → 详细分析

六、调制格式决策框架¶

6.1 损耗斜率指南¶

6.1 损耗斜率指南¶

决策树：

🔍 深度说明： 【研究背景】该决策树是ECEN720课程提出的NRZ vs PAM-4选择准则，基于信道损耗斜率（dB/octave）进行判断，是工程实践中常用的简化工具。 【核心结论】当损耗斜率 > -9.54dB/octave时，PAM-4可能优于NRZ；当损耗斜率 < -9.54dB/octave时，NRZ是更好的选择（电压裕量更大）。 【工程价值】该准则考虑了PAM-4的~9.5dB SNR代价；在实际系统设计中，可以先通过仿真或测量获取信道损耗曲线，再据此选择调制格式。 【落地注意】该准则是一个简化指南，实际选择还需考虑功耗、芯片工艺、CDR复杂度等因素；对于超短距离（chip-to-chip），NRZ往往仍然是首选。

6.2 综合评估因素¶

选择调制方案需考虑:

1. 通道特性
   • 损耗斜率
   • 插入损耗绝对值
   • 阻抗不连续程度

2. 电路能力
   • 片上时钟最高频率
   • TX驱动能力
   • RX设计成熟度

3. 系统级因素
   • 功耗预算
   • 互操作性需求
   • 标准化要求

4. 性能指标
   • 目标BER
   • 温度/电压工作范围
   • 老化效应

6.3 链接分析工具要求¶

精确分析需要:

眼图仿真:
• 电压噪声建模
• 随机抖动(RJ)分析
• 串扰噪声建模
• ISI计算

综合分析:
• 同时考虑电压裕量和时序裕量
• 考虑串扰和ISI的联合分布
• 蒙特卡洛分析PVT变化

最终判决:
• 仿真BER < 目标BER
• 裕量足够(>10%)
• 可生产性验证
