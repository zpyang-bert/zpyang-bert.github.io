ECEN720 Lecture 5: Termination, TX Driver & Multiplexer Circuits — 深度学习报告¶

课程: ECEN720: High-Speed Links Circuits and Systems 授课人: Sam Palermo, Texas A&M University 整理日期: 2026/04/16 研究深度: 深度解读（结合终端匹配架构、CML/V-Mode驱动电路、输出pad带宽扩展、多路复用技术）

一、终端电路体系¶

1.1 离片 vs 在片终端¶

封装寄生效应的关键问题：

封装电感作为stub的问题:
┌──────────────────────────────────────────────────┐
│ 离片终端(Off-chip Termination):                   │
│                                                   │
│ • 封装电感成为未终端的stub                        │
│ • stub反射返回信号路径                            │
│ • 恶化信号完整性                                  │
│                                                   │
│ 在片终端(On-chip Termination):                   │
│                                                   │
│ • 封装电感成为传输线的一部分                       │
│ • 无stub反射                                      │
│ • 封装电感被吸收进传输线特性                        │
└──────────────────────────────────────────────────┘

设计原则: 高速SerDes通常采用在片终端

1.2 串行 vs 并行终端¶

三种基本终端配置对比：

┌─────────────────────────────────────────────────────┐
│ 串联终端(Series Termination)                       │
│                                                     │
│ 结构: Driver → Z₀ → [开路]                         │
│ 特点: 阻抗串联在信号路径上                           │
│ 适用: 低阻抗电压模式驱动器                          │
├─────────────────────────────────────────────────────┤
│ 并联终端(Parallel Termination)                     │
│                                                     │
│ 结构: Driver → Z₀ → GND                           │
│ 特点: 阻抗并联到地                                 │
│ 适用: 高阻抗电流模式驱动器                          │
├─────────────────────────────────────────────────────┤
│ 双终端(Double Termination)                         │
│                                                     │
│ 结构: Source Z₀ → Channel → Z₀ Receiver            │
│ 特点: 两端都匹配Z₀                                 │
│ 适用: 高速SerDes标准方案                           │
│ 优点: 最佳反射抑制                                 │
└─────────────────────────────────────────────────────┘

🔍 深度说明： 【研究背景】阻抗匹配是高速SerDes系统设计的基础问题，决定了信号传输的反射特性。不同终端配置（串联/并联/双端）适用于不同驱动器架构。 【核心结论】双端匹配(Double Termination)是高速SerDes的标准方案，通过在源端和负载端均设置Z₀阻抗实现最优反射抑制，插入损耗仅与信道衰减有关。 【工程价值】设计TX驱动器时，需根据驱动器输出阻抗类型（低阻抗电压模式vs高阻抗电流模式）选择对应终端拓扑，以最大化眼图质量。 【落地注意】实际设计中，双端匹配会增加功耗（约2x于单端终端），需在性能与功耗间权衡；对于低速率/短距离通道，可考虑串联终端以节省功耗。

1.3 直流 vs 交流耦合¶

直流耦合(DC Coupling)：

特点:
• RX共模 = IR/2 (电流模式)
• TX和RX共模电压必须兼容
• 无低频截止
• 支持未编码数据

限制:
• TX和RX的共模电平必须匹配
• 不同协议互连可能有问题

交流耦合(AC Coupling)：

特点:
• RX共模 = V_TT (由终端电阻偏置)
• TX和RX共模电压相互独立
• 信道有低频截止(由耦合电容决定)

限制:
• 低频数据被阻断
• 必须使用编码(如8b/10b)
• DC平衡码要求

低频截止频率:
f_3dB = 1/(2π·R·C_coupling)

其中R为终端电阻，C_coupling为耦合电容

二、无源终端电阻¶

2.1 电阻类型对比¶

90nm CMOS电阻选项：

┌──────────────┬──────────┬────────────────────────┐
│ 电阻类型      │ 方块电阻  │ 寄生电容                 │
├──────────────┼──────────┼────────────────────────┤
│ Poly         │ 90±10Ω  │ 2-3fF/μm²             │
│ N-diffusion  │ 300±50Ω │ 0.9fF/μm²(面积)        │
│ N-well       │ 450±200Ω│ 0.2fF/μm²(周长)        │
└──────────────┴──────────┴────────────────────────┘

Poly电阻的优势:
• 线性度最好 (VC1 ≈ 0)
• 精度较高 (±10%)
• 广泛使用于模拟电路

Poly电阻的劣势:
• ±30% process/temperature variation
• 需要校准电路

2.2 主动FET终端¶

三极管区域FET终端：

工作原理:
• FET工作在深三极管区(线性区)
• 等效为压控电阻
• RDS = 1/(μCox·(W/L)·(VGS - Vt))

电阻公式:
R_FET = 1/(μCox·(W/L)·(V_GS - V_t))

适用场景:
• 低电压摆幅 (<500mV)
• 需要数字可编程控制

增强线性度方案:
┌─────────────────────────────────┐
│ 增加二极管连接FET                │
│ → 扩大线性范围                  │
│ → 通过反馈偏置优化               │
└─────────────────────────────────┘

2.3 数字可调终端¶

数字校准环路：

架构:
┌──────────────────────────────────────────┐
│ 精密片外电阻 (参考)                        │
│        ↓                                 │
│  ← 比较器/ADC                            │
│        ↓                                 │
│  数字控制 → Termination bank              │
│        ↓                                 │
│  调整直到片上电压在1 LSB内                  │
└──────────────────────────────────────────┘

特点:
• 补偿PVT(process, voltage, temperature)变化
• Dither filter降低电压噪声影响
• 可跨多链路共享(但nanometer时代需要per-channel)

电阻调节范围:
• 5-bit trim: 32级
• 7-bit trim: 128级
• 通常步进~1Ω

三、TX驱动器电路¶

3.1 单端 vs 差分信号¶

单端信号的问题：

SSO (Simultaneous Switching Output) 噪声:
┌──────────────────────────────────────────────┐
│ 单端驱动器:                                   │
│ • 有限电源阻抗                                 │
│ • 多个输出同时开关                             │
│ • 通过共享电源/地耦合噪声                       │
│ • 大量地弹(ground bounce)                     │
│                                              │
│ 缓解措施:                                    │
│ • 大去耦电容                                   │
│ • 独立电源层                                   │
│ • 布局隔离                                    │
│ • 去耦占用I/O面积超过电路本身                   │
└──────────────────────────────────────────────┘

差分信号优势：

差分驱动原理:
• 两根线传输相关信号
• 携带差分电压/电流
• 返回电流在差分对内部流动

核心优势:
1. 自参考: 无需地作为参考
2. 共模噪声抑制: 相同干扰被抵消
3. 更大信号摆幅: 差分幅度 = 2×单端
4. 低电源噪声: 开关噪声对两线相同
5. 引脚效率: ~1.3-1.8x (vs 2x单端)

3.2 电流模式 vs 电压模式驱动器¶

电流模式驱动器(Current-Mode)：

基本结构:
• 恒流源驱动差分对
• 通过终端电阻产生电压摆幅
• 输出阻抗高

LVDS标准:
• 3.5mA驱动电流
• ±350mV差分摆幅
• 双端匹配(100Ω差分)

CML (Current Mode Logic):
• 最流行的高速SerDes架构
• 低电压操作(相对推挽)
• 高输出共模(保持电流源饱和)
• 可DC或AC耦合

差分摆幅(双端终端):
V_pp = ±IR/2

其中I为驱动电流，R为终端电阻

电压模式驱动器(Voltage-Mode)：

基本结构:
• 低阻抗电压源直接驱动传输线
• 串联匹配阻抗
• 输出阻抗低

优势:
• 给定摆幅下电流更低
• 理论上4x功率降低(差分)

V_pp = V_s (单端)
V_pp = V_s (差分)

归一化电流对比:
┌──────────────────┬────────────────┐
│ 驱动器/终端       │ 归一化电流     │
├──────────────────┼────────────────┤
│ Current-Mode/SE  │ 1x             │
│ Current-Mode/Diff│ 1x             │
│ Voltage-Mode/SE  │ 0.5x           │
│ Voltage-Mode/Diff│ 0.25x          │
└──────────────────┴────────────────┘

挑战:
• 高线性度要求
• PAM-4需要更大摆幅
• 工艺缩放中电源电压降低

🔍 深度说明： 【研究背景】CML驱动器是当前高速SerDes最流行的架构选择，需要理解其与电压模式驱动器在功耗、线性度、电源电压适应性上的本质差异。 【核心结论】电压模式驱动器在相同摆幅下理论上可降低75%功耗（差分），但高线性度要求使其在纳米级工艺中难以实现；CML驱动器以其恒流特性天然适应高线性度需求，但功耗较高。 【工程价值】对于56Gb/s以上高速SerDes，CML仍是主流选择；电压模式适用于对功耗敏感的中速率场景（<30Gb/s）；设计时需根据应用场景选择驱动器架构。 【落地注意】CML驱动器需要稳定的高输出共模以保持电流源饱和，任何共模扰动会直接影响线性度；版图设计时需注意电流源晶体管的匹配和布局对称性。

3.3 推挽电流模式驱动器¶

LVDS架构：

电路结构:
┌──────────────────────────────────┐
│        VDD                        │
│         │                        │
│      M1 (PMOS)                   │
│         │                        │
│    ──+── TX+                     │
│       │                          │
│    ──+── TX-                     │
│         │                        │
│      M2 (NMOS)                   │
│         │                        │
│        GND                        │
│                                  │
│  恒定电流源 I = 3.5mA (典型)       │
└──────────────────────────────────┘

特点:
• 驱动器电流恒定
• 低dI/dt噪声
• 双电流源改善PSRR
• 但: 电压余度在低电源下是问题

3.4 CML驱动器¶

标准CML架构：

电路结构:
┌──────────────────────────────────┐
│        VDD (高共模)              │
│         │                        │
│      M1 (电流源)                 │
│         │                        │
│    ──+── TX+ ──┬── R            │
│       │        │                 │
│    ──+── TX- ──┴── R            │
│         │                        │
│      M2 (开关)                   │
│         │                        │
│        GND                        │
└──────────────────────────────────┘

特点:
• 低电压操作
• 高输出共模
• DC或AC耦合
• 差分摆幅: ±IR/2 (双端)

单端 vs 差分终端CML：

单端终端:
V_pp = I·2R

差分终端:
V_pp = I·R

驱动电流:
单端: I = V_pp/(2R)
差分: I = V_pp/R

🔍 深度说明： 【研究背景】CML驱动器架构设计中，终端配置方式（单端vs差分）直接影响输出摆幅和驱动电流需求，是SerDes TX设计的关键决策点。 【核心结论】差分终端CML架构在相同驱动电流下输出摆幅减半，但功耗效率更高；单端终端需要2倍驱动电流才能达到相同摆幅，但设计更简单。 【工程价值】对于需要高输出摆幅（>1Vppd）的长距离SerDes，差分终端CML配合高驱动电流是标准方案；在有限功耗预算下，差分终端可有效降低功耗。 【落地注意】差分终端CML的输出共模较高，需要确保RX端AC耦合电容能承受此共模电平；设计时需考虑CML输出的高共模对ESD器件的应力要求。

四、高摆幅发射机技术¶

4.1 高线性度挑战¶

背景:
• >1Vppd摆幅常需要支持高损耗通道
• 电源电压持续降低(工艺缩放)
• 高线性度+低电压 = 困难设计

PAM-4影响:
• PAM-4对线性度要求更高
• 三个判决电平均需精确
• 任何非线性导致失真

4.2 并联泄放电流源¶

ISSCC 2016 (Upadhyaya, VLSI)：

架构:
• 并联厚氧化物泄放电流源
• 从1.8V电源供电
• 提高输出共模
• 16nm FinFET实现 >1.2Vppd摆幅

4.3 高输出级电源¶

ISSCC 2018 (Kim)：

架构:
• VCC_HV: 高压电源
• VCC_NOM: 标称电源
• Vcs ≈ 1V ( cascode偏置)
• PMOS开关晶体管源电压保持在1V
• 10nm可靠性: 允许 >1Vppd摆幅

4.4 无尾电流模式驱动器¶

ISSCC 2017 (Steffan)：

创新点:
• 底晶体管由全速率串行数据驱动
• Replica-bias网络设置cascode栅压
• 实现所需输出摆幅

成果:
• 1.2Vppd输出摆幅
• 94% RLM (线性度度量)

4.5 混合电压模式驱动器¶

PAM-4混合架构 (Bassi, JSSC 2016)：

架构:
• 并行推挽电流模式分段
• MSB和LSB分段独立驱动

优势:
• 高摆幅PAM-4实现
• 1V 28nm CMOS实现1.3Vppd
• >94% RLM

时序:
• 低速操作: 45Gb/s
• MSB/LSB对齐控制

🔍 深度说明： 【研究背景】高摆幅TX驱动器设计面临电源电压持续降低和线性度要求提高的双重挑战；PAM-4调制相比NRZ对线性度要求更高，因为三个判决电平都需要精确。 【核心结论】ISSCC 2016-2018年间出现多种高摆幅方案：并联厚氧化物泄放电流源、高压电源分离、 cascode偏置技术等；Bassi的PAM-4混合架构在28nm实现1.3Vppd同时保持>94% RLM。 【工程价值】对于56Gb/s PAM-4 SerDes，驱动器线性度直接影响误差向量幅度(EVM)；94% RLM是商用PAM-4系统的典型指标要求；设计时需在摆幅、线性度、功耗间做综合优化。 【落地注意】高摆幅驱动器需要关注可靠性问题（栅氧应力、通道长度调制），16nm/10nm FinFET工艺的器件选型需严格遵守foundry提供的安全工作区(SAW)指南。

五、输出阻抗控制¶

5.1 全局电阻校准¶

阻抗校准环路：

流程:
1. 片外精密电阻作为参考
2. 片上终端调整直到电压匹配在1 LSB内
3. Dither滤波器降低噪声
4. 校准码用于设置标称负载

电流模式应用:
• 校准码直接设置负载电流
• 补偿PVT变化

5.2 低摆幅电压模式阻抗控制¶

线性稳压器方案 (Poulton, JSSC 2007)：

架构:
• 线性稳压器设置输出级电源 V_s
• 终端由输出NMOS实现
• Predriver电源反馈控制

优势:
• 阻抗控制与predriver电源独立
• NMOS/PMOS不同尺寸(因V_OD不同)

设计考虑:
• top和bottom输出级晶体管尺寸不同
• 补偿PVT和输出摆幅变化

5.3 高摆幅电压模式阻抗控制¶

无源电阻+晶体管方案：

问题:
• 无源电阻 + 晶体管三极管电阻
• 工艺变化导致阻抗变化
• 引起反射和电平失配

解决: 混合方案
• 多个slice分段控制
• 均衡控制设置tap数
• 终端控制设置使能总数

权衡:
• transistor stacking in full-rate path
• 额外面积(冗余slice)
• 额外功耗(pre-driver尺寸)
• 对P/N skew敏感

混合阻抗控制：

ISSCC 2016 (Chan):
架构:
• 75-85个driver slices
• 10个可编程slice(NAND/NOR predriver)
• coarse + fine双层控制

粗调: programmable slice数量 → 补偿电阻变化
细调: analog impedance loop → 补偿NMOS/PMOS变化

结果: 微分模式回波损耗满足复合返回损耗mask

🔍 深度说明： 【研究背景】输出阻抗控制是高速TX设计的关键挑战，直接影响返回损耗性能；高摆幅电压模式驱动器因器件非线性问题，阻抗控制比电流模式更复杂。 【核心结论】ISSCC 2016的Chan方案采用coarse+fine双层控制：粗调用数字slice数量补偿电阻工艺变化，细调用模拟环路补偿NMOS/PMOS特性差异；75-85个slice配合10个可编程slice实现精确阻抗控制。 【工程价值】复合返回损耗(Composite Return Loss)mask要求S₁₁在Nyquist频率内满足特定模板，是56Gb/s+ SerDes的强制要求；阻抗控制精度直接影响能否通过标准认证测试。 【落地注意】coarse+fine架构的功耗开销来自冗余slice和pre-driver电路；设计时需评估功耗预算 vs 阻抗精度的权衡；P/N skew敏感度在高速场景下可能成为限制因素。

六、输出Pad带宽扩展¶

6.1 T-Coil输出级¶

带宽扩展原理 (Kossel, JSSC 2008)：

T-Coil等效电路:
┌──────────────────────────────────┐
│                                  │
│   L1         L2                 │
│ ─┬──┬──┐    │                   │
│  │  │  │    │                   │
│  C  C  R    C                   │
│ pad ESD term                     │
└──────────────────────────────────┘

L1, L2: T-coil电感
C_pad: Pad电容
C_ESD: ESD电容
R_term: 终端电阻

作用:
• 将driver、ESD、pad电容分离
• 形成带宽增强网络
• 显著提升带宽、改进回波损耗

T-Coil方程：

输出反射因子优化目标:
• S₁₁在Nyquist频率内 < -10dB

设计参数:
• T-coil耦合系数 k = M/√(L1·L2)
• 通常k ≈ 0.7-0.9
• 电感比 L1/L2 影响带宽形状

🔍 深度说明： 【研究背景】TX输出Pad的寄生电容（pad、ESD、package）限制了驱动器带宽；T-coil通过磁耦合将寄生电容分离成多级谐振网络，实现带宽扩展。 【核心结论】T-coil设计核心是耦合系数k优化（典型0.7-0.9）和电感比L1/L2的调节；kossel JSSC 2008方案实现显著带宽提升和回波损耗改善；S₁₁在Nyquist频率内需<-10dB。 【工程价值】T-coil是56Gb/s+ SerDes TX的标配技术；在有限芯片面积内实现高性能T-coil需要精密电磁仿真；耦合系数过低会降低峰值，过高会导致带宽凹陷。 【落地注意】T-coil电感受版图影响显著，需要3D EM仿真验证；电感Q值影响插入损耗；与ESD保护电路的协同设计是关键，避免ESD器件恶化带宽。

6.2 双T-Coil结构¶

ISSCC 2017 (Steffan)：

创新:
• 双T-coil结构允许分离终端电容
• 带宽扩展1.5X

适用:
• 极高数据率发射机
• 1.2Vppd @ 56Gb/s+

6.3 Pi-Coil输出架构¶

Pi型(并-串联)峰化：

结构:
L3
 │
┌┴───┐
│     │
L1  L2
│     │
C    C
pad  term
 │    │
 └────┘
   C_esd1  C_esd2

对比:
T-coil (串联峰化): 适合分离pad和ESD
Pi-coil (并-串联峰化): 适合分离termination capacitance

性能对比：

带宽和回波损耗权衡:
• Pi-coil: 1-2dB带宽峰化
• 但回波损耗略有退化
• 更好的脉冲响应(over低通通道)

典型结果:
RC负载: 基准
T-coil: +30%带宽
Pi-coil: +40-50%带宽

频率响应: Pi-coil略优
脉冲响应: T-coil略优

🔍 深度说明： 【研究背景】T-coil和Pi-coil是两种主流带宽扩展技术，各有优劣；双T-coil进一步提升带宽但增加设计复杂度。 【核心结论】Pi-coil带宽峰化更高（40-50% vs T-coil的30%），但回波损耗略有退化；脉冲响应方面T-coil更优；双T-coil可实现1.5X带宽提升但需要分离终端电容。 【工程价值】对于高损耗通道（如长距离背板），Pi-coil的带宽优势更关键；对于对反射敏感的场景（如高插损信道），T-coil是更好选择；设计需根据信道特性选择合适拓扑。 【落地注意】Pi-coil的并-串联结构对终端电容分离效果好，但版图面积更大；双T-coil需要精确控制两个T-coil之间的耦合，设计窗口更窄；两种方案都需要EM仿真验证。

七、斜率控制¶

7.1 为什么要控制斜率¶

过慢的边沿:
• 限制最大数据率
• 边沿难以被CDR检测

过快的边沿:
• 激发谐振电路 → 振铃导致ISI
• 引起过量串扰
• 电磁干扰(EMI)问题

斜率控制好处:
• 减少反射和串扰
• 改善信号质量
• 降低EMI

7.2 分段驱动器斜率控制¶

实现方法：

分段turn-on:
• 输出驱动器分为n个segment
• 各segment turn-on时间间隔 1/n × t_transition
• 阶梯式边沿

电流模式实现:
• 各segment独立电流源
• 栅极按序触发

电压模式实现:
• 各segment门控V_s
• 时序对齐

八、TX多路复用技术¶

8.1 全速率 vs 半速率架构¶

全速率多路复用：

架构:
• 树形MUX: cascaded 2:1级联
• 需要高频率时钟
• 需要高速触发器

限制:
• 时钟分配高频 → 功耗大
• CMOS buffers限制 ~8FO4周期
• 90nm典型: ~4GHz时钟 → 8Gb/s数据率

优势:
• 时钟duty cycle要求宽松

半速率多路复用：

架构:
• 2相位时钟(180°分离)
• 消除高速时钟和高速FF
• 最终MUX控制 swapped防止glitch

优势:
• 时钟频率减半
• 放宽时钟分配要求
• 关键路径无FF setup time

挑战:
• 输出眼图对时钟duty cycle敏感
• 180°相位精度要求高

8.2 4:1 多路复用¶

ISSCC 2013 (Song)：

架构:
• 8:4 MUX + AND门 + 电平转换
• 2GHz时钟
• 2Gb/s → 8Gb/s

predriver:
• CML to CMOS转换
• Pulse generator
• 4-phase clock (0°/90°/180°/270°)

8.3 1/4速率架构¶

时钟频率进一步降低:
• 4相位时钟(90°间隔)
• 90°相位精度关键
• 时钟分配更容易

极限:
• 8:1 MUX → 45°相位精度
• CMOS MUX可实现
• 但接近极限

8.4 多路复用器速度限制¶

CMOS vs CML：

CMOS MUX限制:
• 最小脉冲宽度 ~2FO4
• 半速率架构限制 ~4FO4
• 90nm: ~8Gb/s

CML MUX优势:
• self-loading factor更小
• 速度更高
• 但: 静态功耗高(与数据率无关)

选择依据:
• 功耗预算
• 数据率需求
• 工艺成熟度

🔍 深度说明： 【研究背景】TX多路复用器架构决定了时钟频率和数据率的映射关系；全速率/半速率/¼速率架构各有优缺点，需根据工艺和功耗预算选择。 【核心结论】CMOS MUX受限于最小脉冲宽度约2FO4，90nm工艺约8Gb/s；CML MUX速度更高但静态功耗大；ISSCC 2013 Song方案用8:4 MUX + 4-phase clock实现2→8Gb/s。 【工程价值】56Gb/s SerDes通常采用¼速率或更低的MUX架构以降低时钟频率；4-phase clock的90°相位精度是关键设计挑战；MUX架构选择直接影响功耗、时钟分配难度。 【落地注意】半速率架构对时钟duty cycle敏感(需<2%误差)，设计时需考虑时钟产生电路的duty cycle控制；¼速率架构的45°相位精度要求需要精密PLL/DCC电路。
