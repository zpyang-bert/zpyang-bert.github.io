## 1. 均衡技术概述

### 1.1 为什么需要均衡

高速串行信号在信道中传输时，会受到带宽限制和损耗影响，导致信号失真：

```
发送信号                              接收信号
┌───────┐                            ┌───────┐
│       │                            │       │
│       │                            │       │
└───────┘                            └───────┘
  清晰眼图    ──── 信道损耗 ────▶    闭合眼图
                                        ││
                                    符号间干扰 (ISI)
```

**信道损耗类型**:
| 损耗类型 | 原因 | 影响 |
|----------|------|------|
| 插入损耗 | 趋肤效应、介质损耗 | 高频衰减 |
| 反射损耗 | 阻抗不匹配 | 回波干扰 |
| 串扰 | 电磁耦合 | 临道干扰 |
| 抖动 | 时钟偏移 | 时序误差 |

### 1.2 均衡的基本原理

**核心思想**: 在接收端或发送端对信号进行补偿，还原失真的信号

```
均衡原理:
原始信号 ──▶ [信道] ──▶ [均衡器] ──▶ 恢复信号
                          ↑
                    补偿信道损耗
```

## 2. 发送端均衡 (TX Equalization)

### 2.1 预加重 (Pre-emphasis)

**原理**: 在信号边沿处增加能量，补偿高频衰减

```
预加重示意:

原始信号:    ┌───────┐
             │       │
             │       │
             └───────┘

预加重信号:  ┌───────┐
             │       │  ▲
             │       │▌▌  ← 边沿增强
             └───────┘
```

### 2.2 FFE (Feed-Forward Equalizer)

**结构**:
```
         延迟 ─┬─ tap n ─┬─ tap n-1 ─ ... ─┬─ tap 0
                │         │                │
原始数据 ───────┼─────────┼────────────────┼──▶ 加法器 ──▶ 输出
                │         │                │
              tap系数    tap系数          tap系数
            (c[n])      (c[n-1])        (c[0])
```

**FFE 公式**:
```
y[n] = Σ c[k] * x[n-k], k = 0, 1, ..., N
```

**Tap 系数设计**:
- 主抽头 (Main tap): 1.0
- 前抽头 (Pre-tap): 负值 (减小前铃)
- 后抽头 (Post-tap): 负值 (减小后铃)

### 2.3 去加重 (De-emphasis)

**原理**: 降低非边沿处的信号幅度

```
去加重示意:

原始信号:    ┌───────┐
             │       │
             │       │
             └───────┘

去加重信号:  ┌───────┐▌▌
             │       │▌▌  ← 非边沿降低
             │       │
             └───────┘
```

## 3. 接收端均衡 (RX Equalization)

### 3.1 CTLE (Continuous Time Linear Equalizer)

**特点**:
- 连续时间模拟均衡器
- 零点/极点配置
- 自动增益控制 (AGC) 配合

**频率响应**:
```
增益
  ▲
  │        ──── 高频提升 ────
  │      ╱
  │    ╱
  │  ╱
  │╱
  └─────────────────────────────▶ 频率
         低频    高频
       抑制      增强
```

**典型配置**:
- DC 增益: 可变 (-6dB 到 +6dB)
- 高频峰值: 可编程
- 带宽控制: 调节频率响应形状

### 3.2 DFE (Decision Feedback Equalizer)

**结构**:
```
                         ┌──────┐
输入 ──▶ 判决器 ──▶ 输出 │      │
            │            │      │ ← 反馈滤波器
            │            │      │    (FIR)
            │            └──────┘
            │              ▲
            └──────────────┘
                   (滤波后反馈)
```

**工作原理**:
1. 采样当前符号
2. 使用历史判决结果估计 ISI
3. 从当前采样中减去 ISI 估计
4. 判决得到最终结果

**DFE 公式**:
```
r[n] = y[n] - Σ w[k] * d[n-k], k = 1, 2, ..., M
其中:
- y[n]: 接收采样
- w[k]: 反馈抽头系数
- d[n-k]: 历史判决结果
- r[n]: 均衡后的样本
```

### 3.3 AGC (自动增益控制)

**作用**: 补偿信道衰减变化

```
输入信号 ──▶ PGA ──▶ CTLE ──▶ AGC ──▶ ADC
              ▲        │         │
              │        │         │
              └────────┴─────────┘
                    增益反馈
```

## 4. 自适应均衡

### 4.1 为什么需要自适应

- 不同信道特性不同
- 温度/老化导致特性变化
- 速率变更需要重新适配

### 4.2 LMS (Least Mean Square) 算法

**原理**: 最小均方误差

```
误差信号: e[n] = d[n] - y[n]
更新规则: w[k] = w[k] + μ * e[n] * x[n-k]

其中:
- d[n]: 期望信号
- y[n]: 实际输出
- μ: 步长参数
- w[k]: 抽头系数
- x[n-k]: 输入样本
```

**收敛条件**:
- 步长 μ 选择: 太大不稳定，太小收敛慢
- 典型值: μ = 0.01 ~ 0.1

### 4.3 CMA (Constant Modulus Algorithm)

**应用**: PAM4/QAM 等高阶调制

**原理**: 维持信号恒定模值

```
CMA 误差: e[n] = R - |y[n]|²
更新: w[k] = w[k] + μ * e[n] * y[n] * x[n-k]
其中 R 是期望的模值
```

### 4.4 盲均衡

**特点**: 无需训练序列

**常用算法**:
- Godard 算法
- 表决反馈均衡

## 5. MLSE (Maximum Likelihood Sequence Estimation)

### 5.1 基本原理

**目标**: 找到最可能的发送序列

```
MLSE vs 线性均衡

线性均衡:
输入 ──▶ 线性滤波器 ──▶ 判决 ──▶ 输出
        (单点处理)

MLSE:
输入 ──▶ 状态机 ──▶ Viterbi 算法 ──▶ 最优序列
        (考虑符号间依赖)
```

### 5.2 信道模型

**离散信道模型**:
```
h[0], h[1], ..., h[L]  (L+1 抽头)
y[n] = Σ h[k] * x[n-k] + n[n]
```

**状态定义**:
- 状态数: 2^(L-1) (NRZ)
- 路径数: 每状态 2 条

### 5.3 Viterbi 算法

**步骤**:
1. 计算所有可能路径的度量
2. 递归更新幸存路径
3. 保留最优路径

**复杂度**:
- 状态数: 2^(L-1)
- 每状态 2 条路径
- 权衡: 精度 vs 计算量

### 5.4 MLSE vs DFE

| 方面 | MLSE | DFE |
|------|------|-----|
| 最优性 | 全局最优 | 局部最优 |
| 复杂度 | 高 (指数) | 低 (线性) |
| 延迟 | 长 | 短 |
| 功耗 | 高 | 低 |
| 适用 | 高性能 | 商用 |

## 6. 均衡器设计考量

### 6.1 带宽与增益权衡

**CTLE 设计**:
- 高频增益 ↑ → 噪声放大 ↑
- DC 增益 ↓ → 低频抑制过度
- 需平衡: 信号恢复 vs 噪声

### 6.2 功耗考虑

| 均衡方式 | 功耗级别 |
|----------|----------|
| 无源CTLE | 低 |
| 有源CTLE | 中 |
| DFE (1-tap) | 中高 |
| DFE (N-tap) | 高 |
| MLSE | 很高 |

### 6.3 延迟预算

- CTLE: ~数百 ps
- DFE: ~数百 ps (N-tap)
- MLSE: ~数 ns (取决于状态数)

## 7. 实际应用配置

### 7.1 PCIe 配置

**TX EQ**:
- Preshoot: 0 ~ 6 dB
- De-emphasis: -3 ~ -6 dB

**RX EQ**:
- CTLE: 可变增益
- DFE: 1-2 tap

### 7.2 Ethernet 配置

**100BASE-T4**:
- 4 级 PAM2 → 2 对双绞线

**1000BASE-T**:
- 5 级 PAM2
- 混合均衡 (Echo cancel + NEXT cancel + FFE)

**10GBASE-T**:
- 4 级 PAM2
- 复杂 DSP 均衡

**40G/100G Ethernet (PAM4)**:
- FFE (TX)
- CTLE + DFE (RX)

## 8. 高速 SerDes 均衡架构

### 8.1 完整链路均衡

```
┌─────────────────────────────────────────────────────────────┐
│                      SerDes 完整均衡链路                      │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  TX:  FFE (Pre-emphasis)                                    │
│       └── 3-tap 或 5-tap FIR                                │
│                                                              │
│  信道: (损耗补偿)                                            │
│       └── 30dB+ 损耗                                        │
│                                                              │
│  RX:  CTLE → AGC → DFE → CDR → 输出                         │
│       └── 可编程均衡器组合                                    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 8.2 PAM4 均衡特殊性

**PAM4 挑战**:
- 三个判决电平
- 更密集的眼图
- 更高 SNR 要求

**PAM4 均衡配置**:
```
TX: FFE (4-tap PAM4)
RX: CTLE + DFE (2-4 tap) + CDR
```
