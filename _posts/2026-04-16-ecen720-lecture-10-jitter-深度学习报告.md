## 一、抖动定义

### 1.1 抖动概念

```
定义:
"抖动 = 信号相对于其理想时间位置的短期变化"

测量类型:
• 周期抖动 (J_PER): 测量周期与理想周期的差异
• 周期到周期抖动 (J_CC): 两个相邻周期的时间差异
• 累积抖动 (J_AC): 测量时钟与理想触发时钟的差异
```

### 1.2 抖动统计参数

```
均值:
• 可解释为固定时间偏移或"skew"
• 通常不重要，可校正

RMS抖动:
• 用于表征随机抖动分量

峰峰值抖动:
• 是确定性(有界)和随机(无界)抖动分量的函数
• 必须在给定BER下引用
```

## 二、抖动分类

### 2.1 随机抖动(RJ)

```
特性:
• 无界
• 高斯分布建模
• 均值假设为零
• 用rms值表征

峰峰值必须给定BER

来源:
• 器件噪声
• 热噪声、散粒噪声、闪烁噪声

PDF:
P(t) = (1/√(2πσ²_RJ)) · exp(-t²/(2σ²_RJ))
```

### 2.2 确定性抖动(DJ)

```
特性:
• 有界
• 峰峰值可预测

来源:
• 传输线损耗
• 占空比失真
• 扩频时钟
• 串扰

分类:
• 正弦抖动 (SJ/PJ)
• 数据相关抖动 (DDJ)
  - ISI
  - DCD
  - BUJ
```

### 2.3 正弦/周期性抖动

```
周期性调制效应:
• 扩频时钟
• PLL参考时钟馈通

单正弦抖动PDF:
• 有界，峰值在±A
• PDF在±A范围内均匀
```

### 2.4 数据相关抖动(DDJ)

```
相关于:
• 发射数据模式
• 攻击者(串扰)数据模式

来源:
• 串行化时钟相位误差
• 信道滤波
• 串扰
```

### 2.5 占空比失真(DCD)

```
来源:
• TX串行化时钟占空比误差
• 串行化后缓冲器上升/下降延迟不匹配

结果PDF:
两个delta函数之和，间隔为α_DCD
```

### 2.6 码间干扰抖动(ISI)

```
来源:
• 信道损耗、色散、反射

均衡可改善ISI抖动
```

### 2.7 有界不相关抖动(BUJ)

```
特性:
• 与数据流时间不对齐
• 最常见来源: 串扰

分类为不相关:
• 因与攻击者信号相关
• 而非victim信号或数据流
```

## 三、总抖动(TJ)

### 3.1 卷积模型

```
总抖动PDF:
PDF_TJ(t) = PDF_RJ(t) * PDF_DJ(t)

其中DJ = SJ * DCD * ISI * BUJ

总抖动是随机和确定性抖动PDF的卷积
```

### 3.2 Dual Dirac抖动模型

![](lecture10_ee720_jitter_images/img-000.jpg)

> 🔍 深度说明：
> 【研究背景】Dual Dirac模型是高速链路抖动分析的核心模型。在10Gb/s以上系统中，抖动分析不能简单用高斯分布描述，因为确定性抖动(DJ)有界且非高斯。Dual Dirac将DJ建模为两个Dirac delta函数，将RJ建模为高斯分布，二者卷积得到总抖动PDF。
> 【核心结论】该模型的关键洞见是：在给定BER下，总抖动可以分解为DJ（决定分布中心位置）和RJ（决定分布宽度）的组合。DJ_δδ表示两个delta函数的间距，σ_RJ表示随机抖动的高斯标准差。BER=10⁻¹²时，总抖动TJ = DJ_δδ/2 + Q(10⁻¹²) × σ_RJ。
> 【工程价值】Dual Dirac模型允许设计者将抖动预算分解为确定性和随机分量分别管理。在系统仿真中，可以分别注入DJ和RJ激励验证系统性能。对于PCIe、USB等标准，可以直接使用标准定义的DJ和RJ指标进行链路预算。
> 【落地注意】实际测量中，DJ_δδ和σ_RJ的提取需要足够多的样本和合适的拟合方法。短尾数据可能导致DJ被低估，长尾数据可能导致RJ被低估。建议使用业界标准的TIE直方图拟合法。

```
用于系统级抖动预算:
• 近似复杂总抖动PDF
• 允许确定性/随机抖动分量预算

模型:
• RJ: 高斯分布
• DJ: 两个Dirac delta函数，间隔DJ_pp

Dual-Dirac PDF:
P(t) = 0.5·[δ(t-DJ/2) + δ(t+DJ/2)] * 高斯

在给定BER的抖动:
BER(t) = 0.5·[erfc((t-DJ/2)/(√2·σ_RJ)) + erfc((t+DJ/2)/(√2·σ_RJ))]
```

## 四、系统抖动预算

### 4.1 系统抖动约束

```
BER性能要求:
UI ≥ DJ_sys + Q(BER)·σ_sys

确定性抖动线性相加:
DJ_sys = Σ DJ_i

随机抖动RSS合成:
σ_sys = √(Σ σ²_i)
```

### 4.2 PCI Express抖动预算示例

```
架构组件:
• TX
• 信道
• RX
• 时钟

抖动模型:
DJ_sys = DJ_TX + DJ_channel + DJ_RX + DJ_clock
σ_sys = √(σ²_TX + σ²_channel + σ²_RX + σ²_clock)
```

## 五、抖动与BER

```
在BER=10⁻¹²:
• 抖动 ~1675ps
• 眼宽裕量 ~200ps

系统可达BER=10⁻¹⁸后才受抖动限制
```
