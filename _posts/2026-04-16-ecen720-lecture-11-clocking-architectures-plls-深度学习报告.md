## 一、时钟术语

### 1.1 时钟同步类型

```
同步(Synchronous):
• 芯片获得相同频率AND相位
• 用于低速总线

等时(Mesochronous):
• 相同频率，但未知相位
• 需要相位恢复电路
• 用于快速存储器、内部系统接口

近同步(Plesiochronous):
• 频率几乎相同，导致缓慢漂移相位
• 需要CDR
• 广泛用于高速链路

异步(Asynchronous):
• 根本无时钟
• 需要请求/应答握手
• 用于嵌入式系统
```

## 二、I/O时钟架构

### 2.1 三种基本I/O架构

```
1. 公共时钟(Common Clock)
   • 同步系统
   • 数据率限于~100Mb/s

2. 前向时钟(Forward Clock)
   • 源同步系统
   • TX芯片转发参考时钟到RX芯片
   • 用于处理器-存储器接口

3. 嵌入式时钟(Embedded Clock)
   • 从数据流提取时钟
   • CDR实现
```

### 2.2 公共时钟I/O

```
特点:
• 原始计算机系统常见
• 同步设计(无主动去偏斜)
• 公共总线时钟控制芯片间传输
• 需要等长布线最小化时钟skew

限制:
• 难控制时钟skew和传播延迟
• 周期时间约束严格
• 对片上电路和板级路由延迟变化敏感
• I/O应用速度有限
```

### 2.3 前向时钟I/O

```
特点:
• 从TX芯片转发公共高速参考时钟到RX芯片
• Mesochronous系统
• 用于处理器-存储器接口和多处理器通信
• Intel QPI, Hypertransport

额外要求:
• 一条额外时钟通道
• "相干时钟"允许低到高频抖动跟踪
• 需要良好的时钟接收放大器

限制:
• 时钟skew限制前向时钟I/O性能
• 驱动器强度和负载不匹配
• 互连长度不匹配
• 低通信道导致抖动放大
• 前向时钟占空比变化
```

### 2.4 前向时钟去偏斜

```
每通道去偏斜允许显著数据率提升:
• 采样时钟调整至数据眼中心

实现方式:
• DLL + 相位插值器
• 注入锁定振荡器

相位采集可实现:
• 基于BER - 无需额外输入相位采样器
• 基于相位检测器 - 需要额外输入相位采样器
```

### 2.5 嵌入式时钟I/O

```
特点:
• 可用于mesochronous或plesiochronous系统
• 时钟频率和最佳相位位置从数据流提取
• 相位检测连续运行

CDR实现:
• 每通道PLL
• 双环路(全局PLL + 本地DLL/PI)
• 本地相位旋转PLL

限制:
• 抖动跟踪受CDR带宽限制
• 技术缩放允许更高带宽CDR
• 通常比前向时钟更多硬件
```

## 三、PLL架构

### 3.1 PLL基本原理

![](/img/serdes/fundamentals/lectures/lecture10_ee720_jitter_深度学习报告/_images/img-000.jpg)

> 🔍 深度说明：
> 【研究背景】PLL是高速链接系统的心脏，负责时钟合成、时钟恢复、偏斜消除等关键功能。理解PLL的负反馈机制是设计高速I/O的基础。在SerDes系统中，PLL用于生成高频率时钟，其相位噪声性能直接决定了系统的抖动性能。
> 【核心结论】PLL通过负反馈使VCO输出锁定到参考时钟。Type-2 PLL（在环路中有两个积分器）可以实现零相位误差，这是高速链接的关键要求。核心模块包括：鉴相器(PD)、电荷泵(CP)、环路滤波器(LF)、VCO和分频器。
> 【工程价值】在系统设计中，PLL的环路带宽决定了抖动跟踪特性——低频抖动被跟踪，高频VCO噪声直通。这对于CDR架构选择和系统抖动预算分配至关重要。正确的PLL设计可以最小化时钟抖动对BER的影响。
> 【落地注意】实际PLL设计中，参考时钟的相位噪声、鉴相器的死区、VCO的闪烁噪声都是实际存在的非理想因素。环路滤波器设计需要在稳定性和噪声抑制之间取得平衡。

```
PLL是负反馈系统:
• 振荡器产生的信号相位AND频率锁定到参考信号

核心模块:
鉴相器(PD) → 电荷泵(CP) → 环路滤波器(LF) → VCO → 分频器
     ↑                                                      │
     └──────────────────────────────────────────────────────┘

应用:
• 频率合成 (100MHz → 10GHz)
•  skew消除
• 时钟恢复
• 调制/解调
```

### 3.2 电荷泵PLL

```
Type-2 PLL特性:
• 两个积分器
• 输入和反馈相位之间理想零相位误差
• 需要稳定零点(由滤波器电阻实现)

典型实现:
• PFD + 电荷泵 + LPF + VCO + 分频器

三阶系统(附加滤波电容):
• 减少参考杂散
```

### 3.3 PLL建模

```
线性模型:
• 相位是关键变量
• 小信号分析适用于理解PLL动态

开环传递函数:
H(s) = K_PD · F(s) · K_VCO / s

闭环传递函数:
Φ_out/Φ_in = H(s) / (1 + H(s)/N)

其中:
• K_PD: 鉴相器增益
• F(s): 环路滤波器传递函数
• K_VCO: VCO增益
• N: 分频比
```

### 3.4 环路滤波器

```
典型二阶滤波器:
F(s) = (1 + s·τ_z) / (s·τ_p)

其中:
τ_z = R·C₁
τ_p = R·C₁·C₂/(C₁+C₂)

提供稳定零点补偿PLL的二阶特性
```

## 四、PLL噪声传递函数

### 4.1 VCO相位噪声

![](/img/serdes/fundamentals/lectures/lecture11_ee720_clocking_arch_plls_深度学习报告/_images/img-004.jpg)

> 🔍 深度说明：
> 【研究背景】VCO相位噪声是PLL输出抖动的根本来源。在高频偏移处（通常>1MHz），VCO的相位噪声直接传递到输出，不被环路滤波抑制。这对于高速SerDes设计至关重要，因为高频抖动直接影响采样时刻的准确性。
> 【核心结论】VCO相位噪声有两个主要来源：低频的闪烁噪声(1/f³区域)和高频的热噪声(1/f²区域)。闪烁噪声会转换为低频的周期抖动，而热噪声决定了VCO的本底噪声水平。设计低相位噪声VCO是实现低抖动时钟的关键。
> 【工程价值】在CDR系统中，VCO相位噪声的高频分量无法被CDR跟踪，会直接转化为采样误差。因此，对于32Gb/s及以上的系统，需要使用LC-VCO（相比Ring-VCO有更低的相位噪声）来满足严格的抖动要求。
> 【落地注意】VCO相位噪声优化需要在硅面积、功耗和噪声性能之间权衡。电感品质因数(Q)、变容管非线性、晶体管闪烁噪声都是实际设计中的关键考虑因素。

```
VCO相位噪声:
• 低频偏移处由闪烁噪声主导
• 高频偏移处由热噪声主导

噪声传递:
• PLLs作为低通滤波器跟踪参考抖动
• VCO噪声直接传递到输出(高通特性)
```

### 4.2 参考时钟噪声

```
参考时钟噪声:
• 通过PLL环路滤波
• 环路带宽内低通特性
• 环路带宽外衰减
```

### 4.3 PLL设计挑战

```
板级参考时钟频率不缩放:
• 156MHz常见

CDR带宽缩放困难(PAM4 + ADC前端):
• 通常2-4MHz
• PLL带宽必须<10MHz(稳定性+滤波参考抖动)

低频偏移VCO相位噪声抑制:
• 闪烁噪声

32.75Gbps收发器PLL仿真抖动数据:

│ RX类型           │ PN@1MHz  │ CDR BW   │ RJ      │
├─────────────────┼──────────┼──────────┼─────────┤
```

## 五、时钟分配

### 5.1 Xilinx 0.5-32Gb/s收发器时钟架构

![](/img/serdes/fundamentals/lectures/lecture11_ee720_clocking_arch_plls_深度学习报告/_images/img-080.jpg)

> 🔍 深度说明：
> 【研究背景】商业SerDes收发器的时钟架构设计体现了理论与工程的深度融合。Xilinx 16nm FinFET CMOS收发器支持0.5-32Gb/s数据率，采用双PLL架构（LC-PLL用于高频，Ring-PLL用于低频）以覆盖宽频率范围，同时优化功耗和抖动性能。
> 【核心结论】该架构的关键创新包括：(1) Frac-N LC PLL覆盖8-16.375GHz高频段；(2) Ring PLL覆盖2-6.25GHz低频段；(3) CML时钟分布+主动感应负载实现低抖动分配；(4) 32.75Gb/s TX实现TJ=5.39ps, RJ=190fs的优异性能。
> 【工程价值】该设计展示了现代高速I/O时钟分配的完整方案：全局PLL负责频率合成，本地PLL负责信道特定的时钟调整。功耗577mW/ch (17.6pJ/b) @ 32.75Gb/s体现了先进CMOS节点的能效优势。
> 【落地注意】实际设计中，时钟分布网络的skew控制至关重要。CML驱动器需要良好的阻抗匹配以避免反射，主动感应负载可以降低功耗但增加了设计复杂度。

```
技术: 16nm FinFET CMOS

PLL配置:
• Frac-N LC PLL: 覆盖高频(8-16.375GHz)
• Ring PLL: 覆盖低频(2-6.25GHz)

时钟分布:
• CML时钟分布 + 主动感应负载
• 低抖动

性能指标:
• 32.75Gb/s TX: TJ=5.39ps, RJ=190fs
• RX JTOL @30MHz: 0.45UI
• 通道损耗@32.75Gb/s: 30dB
• 功耗: 577mW/ch (17.6pJ/b) @ 32.75Gb/s w/ DFE
```
