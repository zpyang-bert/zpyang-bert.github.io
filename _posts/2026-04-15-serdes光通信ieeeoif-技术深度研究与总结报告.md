## 执行摘要

本报告基于本 notebook 资料库中全部的 Markdown 文档与 47 份已提取文本的 PDF（含 16 篇 Texas A&M EE720 高速链路讲义、8 篇 SerDes/光通信中文学位论文/白皮书、23 篇 OFC 2026 Post-Deadline Paper），对 **SerDes 高速串行接口**、**光通信系统与器件**、**IEEE 802.3 以太网标准**、**OIF CEI 规范** 四大领域进行了跨层次、跨学科的深入研究。

**核心发现：**

1. **SerDes 已进入 112G PAM4 量产、224G 预研阶段**：CDR 架构从简单的 PLL-Based 演进为 Dual-Loop（全局 PLL + 本地 PI）和 PS/PI 型；均衡链路标配 TX FFE + RX CTLE + DFE + CDR；功耗效率从 ~15 pJ/b 向 <5 pJ/b（Linear Drive）迈进。

2. **光通信呈现"相干下沉"与"IM/DD 向上"双向演进**：数据中心短距离（<10km）由 PAM4 IM/DD 主导，LPO/CPO 封装技术旨在降低功耗；长距离/城域由相干 DSP 主导，oDSP 集成了色散补偿、偏振解复用（CMA/MMA）、载波恢复、FEC（软判决 LDPC/RS-FEC）等模块。

3. **OIF CEI-112G 是当前产业分水岭，CEI-224G 是下一代必争之地**：CEI-112G-VSR/MR/LR 支撑 800G/1.6T 光模块，而 CEI-224G（112 GBaud PAM4）将支撑 3.2T 系统。IEEE 802.3dj（800G/1.6T）与 802.3df（224G/lane）正处于标准制定关键期。

4. **OFC 2026 PDP 揭示了 6 大前沿方向**：(a) >100 GHz 薄膜铌酸锂/钛酸钡/钽酸锂电光调制器；(b) 200G/lane PAM4 VCSEL/EAM；(c) Linear/Coherent-lite 光学 DAC；(d) 片上纵向功率监测（LPM）DSP；(e) 光通信垂直大模型（Optics GPT）；(f) OAM 轨道角动量海底通信与 TFDM 相干 PON 现场试验。

## 第一章：SerDes 核心技术深度解析

### 1.1 CDR（时钟数据恢复）——SerDes 的"心脏"

#### 1.1.1 基本原理与系统定位

CDR（Clock and Data Recovery）是高速串行通信中的核心模块，负责**从非周期性的数据流中同时恢复出发送端的时钟频率和最佳采样相位**，无需独立的时钟传输线。

```
数据输入 ──▶ 鉴相器(PD) ──▶ 电荷泵(CP) ──▶ 环路滤波器(LF) ──▶ VCO ──▶ 采样时钟
   ▲                                                                        │
   └────────────────────────────────────────────────────────────────────────┘
```

与 PLL 的核心差异：

| 特性 | PLL | CDR |
|------|-----|-----|
| 参考信号 | 周期性参考时钟 | 非周期性数据流（含边沿） |
| 鉴相器 | 传统 PFD | 边沿检测型 PD（Hogge/Alexander/Bang-Bang） |
| 锁定目标 | 频率 + 相位锁定参考 | 频率 + 相位从数据中提取 |
| 抖动来源 | 参考时钟噪声、VCO 相位噪声 | 数据抖动、CDR 带宽限制、PD 非线性 |

#### 1.1.2 CDR 架构分类与技术细节

基于资料库中的曹启富硕士论文（电子科技大学，40nm CMOS，16Gb/s）、张震博士论文（东南大学，SiGe 0.13μm，25Gb/s Bang-bang CDR）以及 EE720 Lecture 12，CDR 架构可分为三大类：

**（1）PLL-Based CDR（单环锁相环型）**

```
数据 ──▶ PD ──▶ CP ──▶ LF ──▶ VCO ──▶ 恢复时钟
 ▲                                          │
 └──────────────────────────────────────────┘
```

- **特点**：频率合成与相位跟踪一体化，VCO 直接输出多相时钟。
- **代表设计**：曹启富论文中的 **PS/PI 型 CDR**（Phase Selector / Phase Interpolator）。
- **关键创新**：
  - 采用 **12 相环形振荡器**（反相器型结构），比传统 8 相时钟具有更高的相位插值线性度。
  - 反相器型 Ring VCO **无静态功耗**，版图面积远小于 LC-VCO，但调谐范围宽（~50%）。
  - 采用 **1:2 + 1:5 两级分频**，降低后续解串器（Deserialzer）的 D 触发器数量。

**（2）Dual-Loop CDR（双环架构）**

```
全局 PLL ──▶ 本地相位插值器(PI) ──▶ 恢复时钟
    ▲
    │
鉴相器(PD) ◀── 数据边沿检测
```

- **特点**：全局 PLL 提供低抖动的高频参考时钟，本地 PI（Phase Interpolator）进行精细相位调整。
- **代表设计**：Xilinx 16nm FinFET Transceiver（0.5–32.75 Gb/s）。
- **性能指标**：
  - TX 抖动 @32.75Gb/s：TJ 5.39 ps，RJ 190 fs
  - RX 抖动容限：0.45 UI @ 30 MHz
  - 功耗：577 mW/ch（17.6 pJ/b）
  - 支持通道损耗：30 dB @ 32.75 Gb/s

**（3）Bang-Bang CDR（非线性/二进制 CDR）**

- **鉴相器**：Alexander/Bang-Bang PD，只输出相位误差的符号（Early/Late），不输出大小。
- **优势**：增益在零误差点理论上无限大，实现简单，对噪声敏感但鲁棒性强。
- **代表设计**：张震论文中的 **25 Gb/s 全速率 Bang-bang PLL**。
- **性能指标**：
  - RMS 抖动：750 fs
  - 峰峰抖动：3.46 ps
  - 核心电路面积：0.48 mm²（SiGe 0.13μm）

#### 1.1.3 鉴相器类型对比

| 鉴相器 | 线性度 | 输出信息 | 特点 | 应用 |
|--------|--------|----------|------|------|
| **Hogge PD** | 线性 | 相位误差的大小和符号 | 脉冲宽度 ∝ 相位差 | 高阶应用、需要精确跟踪 |
| **Alexander/Bang-Bang** | 非线性 | 仅符号（Early/Late） | 零点附近增益极大 | 最广泛应用（PCIe、Ethernet、USB） |
| **Mueller-Muller** | Baud-rate | 基于脉冲幅度推断 ISI | 每符号采样一次，低功耗 | 高速低功耗场景 |

**Bang-Bang PD 的逻辑规则：**
- 如果边沿采样与第二 bit 相同 → **Late**（时钟晚）
- 如果边沿采样与第一 bit 相同 → **Early**（时钟早）

#### 1.1.4 VCO 设计：Ring vs LC

| 特性 | Ring VCO | LC-VCO |
|------|----------|--------|
| 频率范围 | 宽（~50%） | 窄（~20%） |
| 版图面积 | 小（无电感） | 大（需电感） |
| 功耗 | 低（反相器无静态功耗） | 高 |
| 相位噪声 | 较差 | 优秀 |
| 典型应用 | CDR 本地环、SerDes 多相时钟 | 高性能 PLL 主环、低抖动参考 |

**曹启富论文的选择依据**：在高速 SerDes 中，需要在面积、功耗、调谐范围之间权衡。反相器型 12 相 Ring VCO 提供了足够的相位精度和频率范围，同时避免了 LC-VCO 的大面积开销。

#### 1.1.5 CDR 抖动特性与带宽权衡

CDR 本质上是一个窄带跟踪系统，其抖动传递特性决定了系统的信号完整性：

```
增益
  ▲    ┌──────────────
  │    │  带宽外（抖动直通到输出）
1 │────┘
  │     \\          带宽内（抖动被跟踪，不传递）
  │      \
  └────────────────────────────▶ 频率
```

**关键结论**：
- **CDR 带宽越高**：能跟踪更多的高频输入抖动，但参考时钟噪声敏感度上升。
- **CDR 带宽越低**：对高频抖动跟踪能力下降，超出带宽的抖动会传递到输出。
- 典型 CDR 抖动带宽：2–30 MHz。
- 典型抖动容限：0.3–0.5 UI。

### 1.2 均衡技术体系——对抗信道损伤的"武器库"

高速信号在 PCB 走线、连接器、光缆中传输时，受到趋肤效应、介质损耗、反射、串扰等影响，导致眼图闭合、ISI（码间串扰）严重。均衡技术通过在发射端或接收端对信号进行补偿，恢复清晰的眼图。

#### 1.2.1 发送端均衡（TX Equalization）

**FFE（Feed-Forward Equalizer，前馈均衡器）**

```
         延迟线
原始数据 ─┬─ c[-1] ─┬─ c[0] ─┬─ c[1] ───▶ 加法器 ──▶ 输出
          │         │        │
        pre-tap   main-tap  post-tap
```

- **Pre-tap（前抽头）**：补偿前导 ISI（pre-cursor）
- **Main-tap（主抽头）**：当前符号，通常归一化为 1.0
- **Post-tap（后抽头）**：补偿后拖尾 ISI（post-cursor）

**去加重（De-emphasis）**：
通过降低低频分量的幅度，相对增强高频分量。典型 3-tap FFE 设计：
```
W(z) = -0.131 + 0.595·z⁻¹ - 0.274·z⁻²
低频增益: -14.4 dB（衰减）
高频增益: 0 dB（直通）
```

**TX FFE 优缺点**：
- **优点**：不放大噪声，电路简单，可消除前/后 ISI。
- **缺点**：峰值功率受限（需要降低整体幅度），需要回传信道调节系数。

#### 1.2.2 接收端均衡（RX Equalization）

**(1) CTLE（Continuous Time Linear Equalizer，连续时间线性均衡器）**

CTLE 通过高通频率响应补偿信道的高频衰减：

```
增益
  ▲        ──────── 高频提升
  │      ╱
  │    ╱
  │  ╱
  │╱
  └────────────────────────▶ 频率
     低频抑制    高频增强
```

- **无源 CTLE**：由 R-C 网络构成，线性度极佳，但无增益。
- **有源 CTLE**：在输入放大器中加入 R-C 退化网络，可提供增益。
- 张震论文实测：CTLE 均衡后眼宽达到 0.6 UI，误码率 < 10⁻³ 时光灵敏度为 -30 dBm。

**(2) DFE（Decision Feedback Equalizer，判决反馈均衡器）**

```
输入 ──▶ [判决器] ──▶ 输出
            │
            ▼
         [反馈滤波器 FIR]
            │
            └──────▶ 从当前采样减去 ISI 估计
```

- **工作原理**：利用历史判决结果估计 ISI，从当前采样中减去。
- **优点**：不放大噪声（反馈路径处理的是已判决的干净信号）。
- **缺点**：存在反馈环路延迟问题（"关键路径"限制），高速实现难度大。
- **公式**：`r[n] = y[n] - Σ w[k]·d[n-k]`

**(3) AGC（Automatic Gain Control）**

补偿信道衰减的动态变化，配合 CTLE/DFE 工作。

#### 1.2.3 MLSE（Maximum Likelihood Sequence Estimation）

MLSE 是基于 **Viterbi 算法** 的全局最优序列估计技术，通过状态机遍历所有可能的发送序列，选择最可能的路径。

```
线性均衡: 输入 ──▶ 线性滤波器 ──▶ 判决 ──▶ 输出 (单点处理)
MLSE:     输入 ──▶ 状态机 ──▶ Viterbi 算法 ──▶ 最优序列 (考虑符号间依赖)
```

**复杂度**：
- 状态数：2^(L-1)（NRZ），其中 L 为信道记忆长度
- 每状态路径数：2 条

**MLSE vs DFE 对比**：

| 方面 | MLSE | DFE |
|------|------|-----|
| 最优性 | 全局最优 | 局部最优 |
| 复杂度 | 高（指数级） | 低（线性级） |
| 延迟 | 长 | 短 |
| 功耗 | 高 | 低 |
| 适用 | 实验室/高性能系统 | 商用 SerDes/光模块 |

资料库中的 MLSE 论文（光纤通信系统中 MLSE 均衡器的性能分析及优化）深入研究了：
- 热噪声高斯信道下的 Viterbi 算法优化
- ASE（放大自发辐射）噪声下的 Chi 平方分布模型与简化判决公式
- 通过优化算法，MLSE-2 可补偿 ~400km 色散，MLSE-3 可达 ~600km

#### 1.2.4 自适应均衡算法

**(1) LMS（Least Mean Square）算法**

```
误差: e[n] = d[n] - y[n]
更新: w[k] = w[k] + μ · e[n] · x[n-k]
```

- μ 为步长参数：太大不稳定，太小收敛慢（典型 0.01–0.1）。

**(2) CMA（Constant Modulus Algorithm，恒模算法）**

用于 PAM4/QAM 等高阶调制：
```
误差: e[n] = R² - |y[n]|²
更新: w[k] = w[k] + μ · e[n] · y[n] · x*[n-k]
```

在相干光通信的 oDSP 中，CMA/MMA 是偏振解复用的核心算法。

#### 1.2.5 完整 SerDes 均衡链路

```
TX:  并行数据 ──▶ Gray 编码 ──▶ PAM4 映射 ──▶ TX FFE ──▶ 驱动器
                          │
                          ▼
信道:  PCB/光缆（损耗 30dB+）
                          │
                          ▼
RX:  CTLE ──▶ AGC ──▶ DFE ──▶ CDR ──▶ 判决 ──▶ 并行数据
```

对于 **112G PAM4 SerDes**（800G/1.6T 应用）：
- TX：4-tap FFE
- RX：CTLE + DFE（2–4 tap）+ CDR
- 功耗目标：< 5 pJ/b（Linear Drive 架构）

### 1.3 PAM4 调制与高速 SerDes

#### 1.3.1 为什么从 NRZ 转向 PAM4？

| 参数 | NRZ @ 56G | PAM4 @ 56G |
|------|-----------|------------|
| 比特/符号 | 1 | 2 |
| 波特率 | 56 GBaud | 28 GBaud |
| 带宽需求 | 28 GHz | 14 GHz |
| SNR 要求 | 较低 | 高约 9.5 dB |
| 眼图 | 1 个眼 | 3 个眼 |

PAM4 的核心优势是**在相同波特率下数据速率翻倍**，或者在相同数据速率下带宽需求减半，从而降低光电器件的设计难度。

#### 1.3.2 PAM4 的眼图与挑战

- **眼高**：顶层/底层约 100 mV，中间眼仅约 50 mV（典型值）
- **眼宽**：~0.3 UI
- **SNR 恶化**：由于 4 电平之间的间隔是 NRZ 的 1/3，噪声容限显著降低
- **线性度要求**：驱动器、调制器、ADC 都需要极高的线性度
- **ADC 精度**：NRZ 需 3–4 bit，PAM4 需 5–6 bit

#### 1.3.3 112G SerDes 与 224G 路线图

| 年份 | 单通道速率 | 调制 | 应用 |
|------|-----------|------|------|
| 2024 | 112 Gbps | PAM4 (56 GBaud) | 800G/1.6T 光模块 |
| 2025-2026 | 224 Gbps | PAM4 (112 GBaud) | 3.2T 光模块 |
| 2026+ | 448 Gbps | PAM4/PAM6/PAM8 (研究) | 超高速互联 |

224G SerDes 面临的技术挑战：
1. **信道损耗**：~40 dB @ 112 GHz
2. **封装密度**：传统可插拔封装难以容纳
3. **功耗管理**：必须采用 Linear Drive/CPO 等低功耗架构
4. **散热**：ASIC + 光模块总功耗逼近热设计极限

## 第二章：光通信系统技术深度解析

### 2.1 光通信基本架构：IM/DD vs 相干检测

光通信系统按照检测方式可分为两大类：**强度调制直接检测（IM/DD）** 和 **相干检测（Coherent Detection）**。

#### 2.1.1 IM/DD 系统架构

```
电信号 ──▶ [驱动器] ──▶ [激光器/调制器] ──▶ 光纤 ──▶ [光电探测器] ──▶ [TIA] ──▶ 电信号
                (DML/VCSEL/EML)                          (PIN/APD)
```

- **发射端**：DML（直接调制激光器）、VCSEL（垂直腔面发射激光器）、EML（电吸收调制激光器）
- **接收端**：PIN 光电二极管、APD（雪崩光电二极管）、TIA（跨阻放大器）
- **适用场景**：数据中心互联（DCI）、短距离（<10km）、成本敏感应用
- **调制格式**：NRZ、PAM4

**IM/DD 的优势**：架构简单、成本低、功耗低、延迟小。

#### 2.1.2 相干检测系统架构

相干检测利用本地振荡激光器（LO）与信号光干涉，可同时检测光的**振幅、相位和偏振态**，大幅提升接收灵敏度。

```
发送端:
数据 ──▶ [Gray 编码] ──▶ [符号映射] ──▶ [上采样/RRC 滤波] ──▶ [I/Q 调制器] ──▶ [激光器] ──▶ 光纤
                                              ↑
                                         [偏振分束器]

接收端:
光信号 ──▶ [偏振分束器 PBS] ──▶ [90° 混频器] ──▶ [平衡探测器 BPD] ──▶ [ADC] ──▶ [oDSP] ──▶ 数据
                                     ↑
                               [本地振荡器 LO]
```

- **适用场景**：长距离传输（>80km）、城域网/骨干网、400G/800G+ 相干光模块
- **调制格式**：QPSK、16QAM、64QAM
- **优势**：接收灵敏度高（比 IM/DD 高 10–20 dB）、色散容忍度高、支持偏振复用
- **劣势**：结构复杂、功耗高（主要来自 oDSP）、成本高

#### 2.1.3 IM/DD vs 相干检测对比

| 参数 | PAM4 IM/DD | 相干检测 |
|------|------------|----------|
| 典型距离 | < 10km | 10km – 1000km+ |
| 复杂度 | 低 | 高 |
| 功耗 | 低（~4–8W/800G） | 高（~15–30W/800G） |
| 成本 | 低 | 高 |
| 接收灵敏度 | 一般 | 优秀 |
| 色散容忍 | 低 | 高 |
| 典型速率 | 100G/400G/800G | 400G/800G/1.6T |

**产业趋势**：相干技术正在"下沉"到数据中心边缘（DCI 80–120km），而 IM/DD 则在向更高速率（200G/lane PAM4）推进。

### 2.2 oDSP（光数字信号处理）——相干通信的"大脑"

oDSP 是相干光通信系统的核心，负责在数字域完成所有物理层补偿和恢复算法。

#### 2.2.1 完整 oDSP 处理流程

```
发射端 DSP:
Gray 编码 ──▶ 符号映射(QPSK/16QAM/64QAM) ──▶ 上采样(2x) ──▶ RRC 脉冲成形 ──▶ I/Q 生成 ──▶ 偏振复用

接收端 DSP:
ADC 采样(64+ GSa/s) ──▶ 数字下变频 ──▶ 色散补偿(CD) ──▶ 偏振解复用(CMA/MMA) 
                                                    │
                                                    ▼
              数据输出 ◀── 判决解码 ◀── FEC 解码 ◀── 载波恢复(PLL/Costas) ◀── LMS/RLS 自适应均衡
```

#### 2.2.2 关键算法模块详解

**(1) 色散补偿（CD Compensation）**

光纤色散导致不同频率分量传播速度不同，引起脉冲展宽。

**时域 FIR 滤波器**：
```
y[n] = Σ h[k] · x[n-k]
```

**频域色散补偿**（更高效，适合长距离）：
```
X(f) = FFT{x[n]}
Y(f) = X(f) · exp(j · β₂ · L · f² / 2)
y[n] = IFFT{Y(f)}
```
其中 β₂ 为群延迟色散参数，L 为光纤长度。

**(2) 偏振解复用（Polarization Demultiplexing）**

利用 **CMA（恒模算法）** 或 **MMA（多模算法）** 实现偏振态跟踪和 MIMO 均衡：

```
CMA 误差: e = R² - |y|²
更新: w = w + μ · e · y · x*

MMA 误差: e = (R_I² - |I|²) + j(R_Q² - |Q|²)
```

**(3) 载波恢复（Carrier Recovery）**

- **QPSK**：Viterbi-Viterbi 算法（4 次幂相位恢复）
  ```
  φ = arg(x⁴) / 4
  ```
- **高阶 QAM（16QAM/64QAM）**：相位搜索算法或 Costas 环的扩展版本。

**(4) FEC（前向纠错）**

相干系统通常采用 **软判决 LDPC** 或 **级联 FEC（CFEC/OFEC）**：

| FEC 类型 | 编码增益 | 复杂度 | 应用 |
|----------|----------|--------|------|
| 硬判决 RS-FEC | ~6 dB | 低 | 以太网（400G/800G） |
| 软判决 LDPC | ~8–11 dB | 高 | 长距离相干 |
| 级联 FEC | ~11–13 dB | 很高 | 海底光缆 |

IEEE 802.3 中 400G Ethernet 的 RS-FEC 配置为 **RS(544,514)**：
- 544 符号块，514 数据符号，30 纠错符号
- 可纠正 15 个符号错误
- 将 BER 从 ~1e-5 提升到 ~1e-12

### 2.3 LPO、CPO 与封装演进——功耗与密度的博弈

随着 AI/ML 集群对带宽需求的爆炸式增长，传统可插拔光模块的功耗和密度已成为系统瓶颈。

#### 2.3.1 AI/ML 集群的光互连挑战

根据 Nvidia/IPEC 的 Linear Drive 白皮书（LPO_serdes_oDSP.pdf）：

| 挑战 | 描述 |
|------|------|
| 单 GPU 带宽 | 4 Tb/s |
| 8-GPU 服务器 | 28.8 Tb/s |
| 集群规模 | 1000+ GPUs，跨 100 米 |
| PCIe 卡功耗限制 | 模块功耗 ~10W |
| 目标 | 800G 接口：<4W（多模），<8W（单模） |

#### 2.3.2 LPO（Linear Pluggable Optics）

**传统 DSP-Based 架构**：
```
ASIC ──▶ CDR ──▶ oDSP ──▶ O/E ──▶ 光学输出
         （功耗高，~40W per 100G）
```

**Linear Drive 架构**：
```
ASIC ──▶ 线性驱动器 ──▶ O/E ──▶ 光学输出
         （功耗低，<4W per 100G）
```

LPO 的核心思想是**去掉光模块内部的 DSP/CDR 芯片**，将信号处理负担转移到主机侧 ASIC 的 SerDes 中。

**LPO 优缺点**：
- **优点**：功耗降低 50%+、延迟接近 0、成本降低
- **缺点**：距离受限（需要线性信道）、交换机端必须具备高性能 DSP、互通性挑战大

#### 2.3.3 CPO（Co-Packaged Optics）

CPO 将光引擎与交换芯片**共封装**在同一基板上，进一步缩短电互连距离：

```
传统: ASIC ──PCB──▶ 可插拔光模块
CPO:   ASIC ──2mm──▶ 共封装光引擎
```

**优势**：
- 电链路损耗从 20–30 dB 降至 <5 dB
- 功耗降低 30–50%
- 支持更高密度（3.2T+）

**挑战**：
- 热管理（ASIC + 光引擎集中散热）
- 可维护性（不可现场更换）
- 标准尚未完全成熟

OIF 正在制定 CPO 的电气接口规范，预计 2025–2026 年发布相关 IA。

#### 2.3.4 OIF CEI-Linear 规范

OIF 针对 LPO 制定了 **CEI-Linear** 电气接口规范，定义了主机与线性光模块之间的接口：

| 测试点 | 位置 | 说明 |
|--------|------|------|
| TP1 | 主机 ASIC 输出 | 发射电气接口 |
| TP2 | 模块输入（光） | 光发射机输入 |
| TP3 | 模块输出（光） | 光接收机输出 |
| TP4 | 模块输出（电气） | 接收电气接口 |

**CEI-Linear vs CEI-Retimed 对比**：

| 特性 | CEI-112G (Retimed) | CEI-112G-Linear |
|------|---------------------|-----------------|
| 重定时 | 是 | 否 |
| CDR | 模块内含 | 主机侧 |
| 功耗 | ~15 pJ/b | ~5 pJ/b |
| 延迟 | ~100 ns | ~0 |

**技术意义**：薄膜材料（TFLN、BTO、TFLT）正在打破传统硅光调制器的带宽瓶颈（~50 GHz），为 200G/lane 甚至 400G/lane 铺平道路。

#### 2.4.2 高速激光器与 VCSEL（Th4A）

**(1) NTT 的 EA-DFB 激光器阵列**

- **Th4A.1**：异质集成的 4 通道 O 波段膜状 InGaAlAs EA-DFB 激光器阵列，每通道实现 **400–448 Gbps PAM4**，100 μm 长调制器在 0.5V 摆幅下工作， Shoreline 密度达到 **3.2 Tbps/mm**。

**(2) Sony 的高温 VCSEL**

- **Th4A.3**：850nm 氧化物限制 VCSEL，在 25°C 至 80°C 范围内保持 **>35 GHz 带宽**，支持 **200G-PAM4** 链路，TDECQ < 4 dB，寿命超过 10 年。

#### 2.4.3 DSP 智能化与网络管理（Th4B/Th4C）

**(1) NTT 的片上纵向功率监测（LPM）**

- **Th4B.4**：首次在 OSFP 模块内的实时相干 DSP 中实现**距离分辨纵向功率监测**。使用 800ZR+/400ZR+ 业务流量，可在 **1005 km** 范围内实现 **2.1 km 分辨率**的分布式功率剖面测量，且功耗开销可忽略（<1% DSP 面积）。

**(2) 上海交通大学的 Optics GPT**

- **Th4C.1**：首个面向光学与光通信的**垂直预训练基础模型**（8B 参数）。通过三阶段认知训练，在光学专业基准测试上超越 GPT-4o 和 DeepSeek-R1（671B）。
- **实际能力**：
  - 从自然语言生成可部署的 DSP Python 代码（PMD 补偿代码 347 行，直接运行）
  - 自动 OSNR 优化（将链路 OSNR 纹波降低 1.48 dB）
  - 故障诊断准确率 100%（20 种诱导故障）

#### 2.4.4 新型传输技术与现场试验（Th4C）

**(1) 复旦大学/中国电信的 200G TFDM 相干 PON 现场试验**

- **Th4C.4**：全球首个基于实时 FPGA 接收的双向 200G TFDM（时分频分复用）相干 PON 现场试验，通过真实部署光纤链路（29–30 km），下游灵敏度 -34 dBm，上游灵敏度 -31 dBm，动态范围 21 dB。

**(2) 中山大学/暨南大学/长飞的 OAM 海底通信**

- **Th4C.5**：全球首个基于**轨道角动量（OAM）模式复用**的海底光纤现场试验，在南海铺设专用 OAM 光缆，实现 **2232 通道**传输。通过优化纤芯掺杂（Δ=0.75%），实现了低损耗（~0.21 dB/km）和低串扰（<-29.4 dB/km）。

**(3) 西湖大学的 200G VHSP SC-PON**

- **Th4C.3**：采用无制冷 TO-Can DFB 激光器 + SOA 放大器，在 200Gb/s VHSP（超高速 PON）中实现 **40.72 km 传输**和 **37 dB 功率预算**的记录，消除了 OLT 端快速 LO 跟踪的需求。

## 第五章：详细总结与研究建议

### 5.1 各方向技术细节总结

#### SerDes 方向
- **CDR** 已从简单的 PLL 型发展为 Dual-Loop（全局 PLL + 本地 PI）和 PS/PI 型。Bang-Bang PD 因其简单性和高增益成为主流。12 相 Ring VCO 在面积和功耗上优于 LC-VCO，但相位噪声稍差。
- **均衡链路** 的标准配置是 TX FFE（3-5 tap）+ RX CTLE + DFE（2-4 tap）。MLSE 虽然理论最优，但复杂度指数增长，难以商用。
- **PAM4** 使 112G SerDes 成为可能，但要求 SNR 提高约 9.5 dB，ADC 精度从 3-4 bit 提升到 5-6 bit。

#### 光通信方向
- **IM/DD PAM4** 主导短距数据中心互联，**相干 DSP** 主导长距传输。两者的边界正在模糊（相干下沉到 80–120km DCI）。
- **oDSP** 的核心算法包括：色散补偿（频域 FIR）、偏振解复用（CMA/MMA）、载波恢复（Viterbi-Viterbi/相位搜索）、软判决 FEC（LDPC）。
- **LPO/CPO** 是解决功耗瓶颈的关键路径。LPO 去掉模块内 DSP，CPO 进一步将光引擎与 ASIC 共封装。

#### 标准方向
- **IEEE 802.3dj** 定义 800G/1.6T，**802.3df** 定义 224G/lane，是 2025-2027 年产业关注的焦点。
- **OIF CEI-112G** 支撑当前 800G 产业，**CEI-224G** 将支撑 3.2T 下一代系统。**FlexE/FlexO** 提供速率灵活性。

#### 前沿方向（OFC 2026）
- **薄膜材料**（TFLN、BTO、TFLT）调制器突破 100 GHz 带宽，为 200G/lane 及以上铺平道路。
- **高速激光器**（NTT EA-DFB 448G/lane、Sony VCSEL 200G-PAM4）推动单通道速率极限。
- **智能化**（Optics GPT、NTT LPM）标志着光网络向 AI 原生演进。
- **新型复用**（TFDM 相干 PON、OAM 海底通信）探索频谱效率的新维度。

### 5.2 研究空白与后续建议

基于资料库分析，以下方向值得进一步深入研究：

1. **224G SerDes 的 CDR 与均衡协同设计**：112 GHz 带宽下，CDR 带宽、DFE 关键路径延迟、ADC 功耗的联合优化。
2. **PAM4 CDR 的特殊挑战**：多电平信号的边沿检测、多级判决与相位恢复。
3. **Linear Drive 的通道建模与兼容性**：LPO 去掉 DSP 后，主机侧 SerDes 必须补偿更大的信道非线性。
4. **TFLN/BTO 调制器与 CMOS 驱动器的集成**：高频电光器件的阻抗匹配、热管理、封装技术。
5. **oDSP 的低功耗实现**：将 CMA、载波恢复、FEC 等算法向 ASIC/FPGA 高效映射。
6. **AI 在光网络中的应用**：Optics GPT 代表的垂直大模型，可能改变光网络的运维、设计和算法开发范式。
