## 二、LPM原理

### 2.1 LPM基本原理

**从端点获取分布信息**：
```
传统相干接收：
- 监测总功率
- 无位置信息

LPM方法：
- 利用瑞利背散射
- 从接收信号提取
- 无需专用硬件
```

**信号模型**：
```
接收信号：
y(t) = Σ h_i · x(t-τ_i) + n(t)

其中：
- h_i = 第i段信道响应
- τ_i = 延迟
- n(t) = 噪声

通过DSP反演获取分布
```

### 2.2 传统方法的问题

**离线处理的负担**：
```
原始LPM算法：
- 计算量: 巨大
- 内存需求: MB-GB级
- 不适合芯片实现

需要：
- 简化算法
- 低复杂度
- 低内存占用
```

### 2.3 闭式辅助方法

**低复杂度实现**：
```
本文方法：
- 闭式辅助近似
- 基于[11]的方法
- 大幅降低计算量

核心逻辑占用: <1% 总核心逻辑面积

功耗增加: 可忽略
```

## 三、DSP架构

### 3.1 DSP芯片布局

**LPM引擎位置**：
```
DSP芯片内：
┌────────────────────────────────────────┐
│              DSP主链路                  │
│  (相干检测 → 均衡 → FEC解码)           │
├────────────────────────────────────────┤
│              LPM引擎                   │  ← 新增
│  (接收波形获取 → 功率分布估算)           │
├────────────────────────────────────────┤
│              标准I2C接口               │
└────────────────────────────────────────┘
```

### 3.2 数据流

**LPM处理流程**：
```
1. 从Rx DSP管线获取接收波形
         ↓
2. 判决导向信号重建
   (使用pre-FEC硬判决)
         ↓
3. 闭式辅助LPM算法
         ↓
4. 计算分布式功率
         ↓
5. 结果输出 (I2C接口)
```

### 3.3 流量假设

**无需先验知识**：
```
传统方法问题：
- 需知道发送信号
- 需训练序列
- 限制多厂商互操作

本文创新：
- 从解调信号重建
- 使用硬判决pre-FEC数据
- 无需特殊探测信号
- 可用服务流量
```

## 七、算法深度

### 7.1 瑞利背散射

**光纤中的弹性散射**：
```
瑞利散射系数:
α_R ≈ 0.2-0.3 dB/km

背散射功率:
P_back ∝ P_launch · e^(-2αL)

提供位置信息:
- 散射系数分布
- 随距离指数衰减
```

### 7.2 闭式辅助估计

**低复杂度算法**：
```
核心近似:

传统方法:
y = Hx + n
需要迭代求解H⁻¹

闭式辅助:
x_hat = (H^T H + λI)⁻¹ H^T y
→ 固定矩阵运算

复杂度: O(n²) → O(n log n)
```

### 7.3 噪声平均

**噪声抑制**：
```
多次测量平均:
σ²_avg = σ² / N

N = 1000-10000 符号

4σ检测门限:
→ SNR ≈ 12 dB即可靠检测
```
