## 1. 研究背景

### 1.1 为什么需要 CPO

数据中心交换机的带宽增长速度远超功耗效率提升速度。传统可插拔光模块（Pluggable Optics）将光引擎放置在交换机面板外，通过长距离电气互连（10~30cm）与交换芯片连接，存在两大根本问题：

1. **电气损耗不可接受**：56 Gbaud PAM4 信号在 30cm PCB trace 上的损耗可达 20+dB，224G 时更高达 35~40dB，需要在接收端使用复杂的高功耗均衡器
2. **功耗墙（Power Wall）**：超大规模数据中心单个交换机柜的功耗可达 30~40kW，传统光模块的功耗占比持续上升

CPO（Co-Packaged Optics）将光引擎与交换芯片封装在同一基板上或非常接近的位置（< 2cm 电气通道），从根本上解决上述问题。这一趋势在 2022 年 IEEE 802.3ck（100G/lane）和 2024 年 802.3dj（200G/lane）标准中已明确定义支撑架构。

### 1.2 CPO 与 OIO 的关系

**OIO（Optical Input/Output，光学输入/输出）** 是更广义的概念，涵盖三种封装形态，按照光引擎与交换芯片的距离从近到远排序：

| 形态 | 全称 | 光引擎位置 | 电气通道长度 | 功耗 | 典型应用 |
|------|------|-----------|-------------|------|---------|
| **CPO** | Co-Packaged Optics | 与 ASIC 同封装 | < 2cm | < 5W/800G | 51.2T/102.4T 交换机 |
| **NPO** | Near-Packaged Optics | 非常接近 ASIC | 2~10cm | 参考 CPO | 过渡方案 |
| **OBO** | On-Board Optics | 板上可插拔 | 10~30cm | 参考可插拔 | 成本敏感场景 |

**演进路径**：可插拔 → OBO → NPO → CPO，代表光引擎逐步向交换芯片靠近的产业趋势。

### 1.3 标准进展（2022-2026）

- **2022, IEEE 802.3ck**: 定义 100G/lane 电接口，为 CPO/LPO 预研奠定基础
- **2024, IEEE 802.3dj**: 定义 200G/lane（106 GBaud PAM4），其中 **1.6TAUI-16** 接口（16 lane × 106 Gb/s = 1.6 Tb/s）专为 CPO 场景设计，直接与 800G 光引擎兼容
- **OIF CEI-Linear**: 针对 Linear Drive（非重定时）光模块定义电气接口规范，是 CPO 电气层面的核心标准

## 2. 系统架构：Linear Drive 与 OIO

### 2.1 从 DSP-Based 到 Linear Drive 的架构演进

传统可插拔光模块采用 **Retimed（重定时）** 架构：光模块内置完整 oDSP，对经过长距离电气通道衰减的信号进行时钟恢复、均衡、FEC 解码，再重新驱动光输出。这种架构的优点是链路性能与交换机 ASIC 无关，缺点是功耗高（oDSP 约占光模块功耗 50%）。

**Linear Drive（线性驱动）** 的核心理念是"去掉 oDSP"，光模块只包含线性驱动/接收（Linear Driver/Linear TIA），信号质量完全依赖交换芯片 SerDes 的发送预均衡和接收均衡能力。这要求电气通道足够短、损耗足够低——这正是 CPO 的核心价值。

两种架构的功耗对比（来自 Nvidia 数据）：

| 架构 | 功耗级别 | 说明 |
|------|---------|------|
| 传统 DSP-Based | ~15 pJ/bit | 含 CDR、均衡、FEC |
| Linear Drive (LPO) | ~5 pJ/bit | 无 DSP，仅线性 TIA/Driver |
| Linear Drive (CPO) | < 3 pJ/bit | 极短电气通道，功耗最优 |

### 2.2 Linear Drive 的信号链

```
交换芯片 (ASIC)                 光引擎 (OE)
┌────────────────┐            ┌────────────────┐
│   SerDes TX   │ 电气通道    │ Linear Driver │ → E/O → 光纤
│  (FFE + CDR)  │ (< 2cm)    │               │
├────────────────┤            ├────────────────┤
│   SerDes RX   │ 电气通道    │ Linear TIA     │ ← O/E ← 光纤
│  (CTLE+ADC+DFE)│ (< 2cm)    │               │
└────────────────┘            └────────────────┘
        ↑                              ↑
   ASIC 内部封装                 光引擎封装
   (CPO 同封装)                  (独立或共封装)
```

关键变化：
- **TX 方向**：交换芯片 SerDes 输出预均衡后的 PAM4 信号，经过极短电气通道（< 2cm，损耗 < 5dB）直接驱动 Linear Driver，无需额外重定时。TX 预均衡强度可以降低（因为通道短了），节省发送功耗
- **RX 方向**：光信号经 Linear TIA 转换为电信号后，通过极短电气通道进入交换芯片 SerDes RX，由 ASIC 内部的 CTLE+ADC+DFE 完成均衡处理

### 2.3 CPO 对 SerDes 的具体影响

**TX 侧（发送方向）**：
- 预均衡强度降低：传统可插拔 30cm+ 通道需要强预加重来补偿损耗，CPO < 2cm 通道所需预加重大幅降低
- 驱动负载减轻：线性驱动器的负载阻抗更高（可能 50Ω 而非 25Ω），降低 SerDes TX 的驱动功耗
- 目标：SerDes TX 功耗降低 20~30%

**RX 侧（接收方向）**：
- 通道损耗从 20+dB 降至 < 5dB，接收均衡压力大幅减轻
- 但 Linear TIA 的线性输出摆幅有限，要求 ADC 在更宽动态范围内保持线性
- ADC 采样率要求进一步提升（因为没有 oDSP 在光模块侧做降采样）
- RX 均衡复杂度可能降低（CTLE/DFE 深度减小），但时钟和数据恢复仍由 ASIC SerDes 完成

## 3. 电气接口规范：1.6TAUI-16 与 CEI-Linear

### 3.1 1.6TAUI-16 接口（IEEE 802.3dj）

1.6TAUI-16 是 802.3dj 为 CPO 场景定义的标志性电气接口：

| 参数 | 规格 |
|------|------|
| 通道数 | 16 lane |
| 每 lane 速率 | ~106 Gb/s（PAM4）|
| 总带宽 | 1.6 Tb/s |
| 应用场景 | 800G 光引擎（8λ × 100G）|
| 封装形式 | CPO（共封装）|
| 目标距离 | < 2cm（芯片间）|

该接口直接对应 800G DR4/FR4 光引擎，每 lane 106Gbps 与 100G Lambda 标准对齐，形成完整的 1.6T 交换机到 800G 光引擎的 CPO 解决方案。

### 3.2 CEI-Linear 规范（OIF）

CEI-Linear 是 OIF 针对 Linear Drive（非重定时）光模块的电气接口规范，定义了发射端测试点 TP1a 和接收端测试点 TP4：

| 接口类型 | 目标距离 | 功耗目标 | 特点 |
|---------|---------|---------|------|
| CEI-Linear | < 2m | < 5W/800G | 无重定时，纯线性驱动 |
| CPO/NPO | < 2m | < 5W/800G | 与 ASIC 同封装或近封装 |

CEI-Linear 的关键挑战是**互操作性和标准化测试方法**：由于没有 oDSP 做信号重置，链路性能高度依赖端到端信道质量，需要明确定义 TP1a（主机侧发射参考点）和 TP4（光模块接收参考点）的规范和测试方法。

### 3.3 224G 系统的 CPO 演进

| 参数 | 800G CPO (当前) | 1.6T CPO (近中期) | 3.2T CPO (长期) |
|------|-----------------|-------------------|-----------------|
| 速率/lane | 106 Gb/s | 212 Gb/s (224G) | 212+ Gb/s |
| 调制 | PAM4 | PAM4 | 可能更高阶 |
| 端口配置 | 16 × 106G | 16 × 212G | 16 × 212G+ |
| SerDes 功耗 | ~5 pJ/bit | ~4 pJ/bit | < 3 pJ/bit |
| 光模块功耗 | < 5W/800G | < 10W/1.6T | 参考 1.6T |

## 4. 热管理： CPO 的核心工程挑战

### 4.1 热密度问题

CPO 将光引擎（包含激光器、调制器、TIA、Driver）和交换芯片封装在同一基板上，带来前所未有的热密度挑战：

- **交换芯片功耗**：51.2T 交换芯片功耗可达 1000W+，热密度极高
- **光引擎功耗**：800G 光引擎约 15~25W（取决于架构），其中激光器对温度非常敏感（波长随温度漂移约 0.1nm/℃）
- **共封装后**：两者热源非常接近，必须作为统一热管理系统设计

### 4.2 OIF 关注的热管理要素

OIF 对 CPO 的三大关注领域之一即为热管理：

1. **光引擎与 ASIC 的热耦合**：两者的最佳工作温度范围不同——ASIC 结温可达 100~110℃，而 InP/硅光激光器的工作温度范围通常为 -40~85℃，且对温度漂移敏感
2. **散热路径设计**：必须从系统级（而非模块级）角度设计散热路径，包括冷板、液冷、热管等
3. **温度感测与控制**：需要精确的热监控来保护激光器和补偿温度漂移

### 4.3 工程落地权衡

| 权衡点 | 可插拔方案 | CPO 方案 |
|--------|----------|---------|
| 可服务性 | 优点：现场可更换 | 缺点：需整机维护 |
| 链路性能 | 依赖 oDSP 重定时 | 依赖主机侧 SerDes 质量 |
| 功耗效率 | ~15 pJ/bit | ~3~5 pJ/bit |
| 热管理 | 模块级散热 | 系统级散热设计 |
| 部署灵活性 | 高 | 低 |
| 成本（大批量）| 较高 | 潜力更低 |

## 5. 前沿技术： OFC 2026 TFLN CPO 引擎

### 5.1 晶圆级异构集成 TFLN CPO 引擎

上海交大在 OFC 2026 发表的 Th4A.6 论文实现了首个晶圆级异构集成 TFLN（Thin-Film Lithium Niobate）CPO 引擎，核心突破：

**封装创新**：
- 在 4 英寸高阻硅晶圆上刻蚀腔体，精度 ±5μm
- TFLN EOM 芯片与 CMOS SoC、GaN PA、InP LNA 共封装
- 使用 BCB（苯并环丁烯）作为介质层，12μm 厚度的光敏 BCB 实现细间距 RDL（再分布层）
- 芯片间互连：3μm 铜+1μm 金电镀，插损 < 0.5dB @ 100 GHz

**性能结果**：
- 封装前 EOM 带宽：110 GHz
- 封装后 3-dB 带宽：**> 100 GHz**（几乎无性能损失）
- 仿真预测：封装结构支持 **> 200 GHz** 带宽
- 封装引入的额外插损：< 4.2 dB（含光纤耦合损耗）
- Vπ = 4 V（半波电压，低功耗调制）

### 5.2 对 SerDes 设计的映射意义

TFLN CPO 引擎展示的 100+ GHz 调制带宽对 SerDes 设计的直接映射：

1. **光侧带宽不再是瓶颈**：当光引擎自身支持 > 100 GHz 调制带宽时，系统的瓶颈完全转移到电侧 SerDes 通道（TX FFE、电气通道、RX 均衡）
2. **电-光接口带宽匹配**：SerDes 输出的 56 GBaud PAM4 信号（基带 ~28 GHz）在 TFLN 调制器端不会引入额外带宽限制，但 112 GBaud（基带 ~56 GHz）已经需要确认 EOM 的频率响应是否平坦
3. **112 GBd PAM4 的挑战**：对于 224G/lane（112 GBd PAM4），EOM 的 S21 在 56 GHz 处的滚降必须 < 1~2 dB，否则需要额外的电均衡补偿

## 6. 工程价值与落地注意

### 6.1 对 SerDes 芯片设计的直接参考

**TX 设计影响**：
- CPO 场景下电气通道 < 2cm、损耗 < 5dB，SerDes TX 预加重强度可以降低，驱动功耗节省约 20~30%
- 但仍需考虑：老化后通道特性变化、多个光引擎的批次一致性
- Linear Drive 架构下 TX 输出的眼图质量直接影响链路性能，无法靠 oDSP 弥补，TX 设计余量必须更保守

**RX 设计影响**：
- 接收均衡压力降低，但 Linear TIA 的输出动态范围有限，要求 ADC 在更大摆幅下保持线性
- ADC 采样率要求可能更高（因为没有 oDSP 降采样）
- 对信道非线性的容忍度降低——TX 预均衡和 RX 均衡的联合优化必须在设计阶段充分验证

**SerDes 架构选择**：
- 传统 Retimed 可插拔：SerDes 只需驱动 30cm+ 通道，眼图质量要求相对宽松
- Linear Drive CPO：SerDes 必须直接驱动光模块，眼图质量要求严苛，与光模块的互通性测试成为必备环节

### 6.2 系统级工程注意事项

**互通性挑战**：
- CEI-Linear 规范仍处于完善阶段，TP1a/TP4 的标准化测试方法尚未完全统一
- 不同光引擎供应商的 Linear TIA 输入敏感度可能存在差异，需要在 SerDes 输出眼图合规性上留有充足余量
- 建议：建立完整的互通性测试矩阵，覆盖至少 3 个不同光引擎供应商

**可服务性与维护**：
- CPO 模式下，光引擎与 ASIC 共封装，出现光模块故障时需要整机维护，成本远高于现场可更换的可插拔模块
- 行业倾向于将 CPO 用于固定部署的交换机（大型数据中心），而可插拔用于需要频繁变更的场景
- 落地选型时：只有在运维模式明确、支持热插拔备份、或整机维护成本可接受的情况下才推荐 CPO

**224G 向后兼容性**：
- 224G CPO（1.6T 端口）对 SerDes 的要求比 112G 更严苛：UI 从 17.86 ps 减半至 8.93 ps
- 1.6TAUI-16 接口在 224G 时代可能演进为 1.6TAUI-16++ 或等效接口
- 功耗目标 < 3 pJ/bit（Linear Drive CPO）意味着 SerDes 本身必须采用更先进工艺（3nm/2nm）和更高效的架构

**热管理协同设计**：
- SerDes IP 必须与热管理团队协同设计，确认在最坏情况结温下（110℃）的 SerDes 性能退化
- TFLN 等新型调制器技术的热漂移特性需要与 SerDes 的自适应均衡能力联动

## 7. 核心结论汇总

| 维度 | 关键结论 |
|------|---------|
| **架构** | CPO 是 Linear Drive 的终极形态，光引擎与 ASIC 同封装，电气通道 < 2cm，损耗 < 5dB |
| **功耗** | Linear Drive CPO 可实现 < 3 pJ/bit，相比传统 DSP-Based 的 ~15 pJ/bit 降低 5x |
| **SerDes 影响** | TX 预均衡降低节省功耗，RX 均衡压力减轻但 ADC 线性度要求更高，眼图质量余量必须更保守 |
| **标准** | IEEE 802.3dj 1.6TAUI-16 是 CPO 核心电接口；CEI-Linear 规范定义 Linear Drive 测试点 |
| **热管理** | 最大工程挑战，光引擎（激光器）与 ASIC 共封装后必须统一热设计 |
| **可服务性** | 最大商用化障碍，需要新的运维模式和维护策略 |
| **前沿技术** | OFC 2026 TFLN CPO 引擎展示晶圆级异构集成能力，封装后带宽 > 100 GHz，支持 224G 及更高速率 |

---

> 🔍 **深度说明**：CPO 本质上是一场"以封装复杂度换取功耗效率"的赌注。对于 SerDes 工程师而言，这意味着设计目标的根本转变——从"强驱动、长距离兼容"变为"精控制、短距离直驱"。224G CPO 的 UI 仅 8.93 ps，TX 眼图质量、通道阻抗控制、封装寄生参数都成为系统性能的直接约束，而非靠 oDSP 弥补的次要问题。工程落地时，切忌用 112G 可插拔的经验直接外推 CPO 场景——两者的信道模型、功耗预算、设计余量完全不同，建议在 224G CPO 项目启动初期就建立完整的电-光联合仿真平台，而非分别独立设计后再做系统集成验证。
