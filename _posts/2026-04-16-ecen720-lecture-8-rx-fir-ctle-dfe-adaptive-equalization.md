## 一、RX FIR均衡

### 1.1 RX FIR原理

```
时域实现:
• 延迟模拟输入信号
• 乘以均衡系数
• 叠加输出

FIR滤波器:
y[n] = Σ w_k · x[n-k]

其中:
• x[n]: 输入信号
• w_k: FIR抽头系数
• y[n]: 均衡后输出
```

**5-tap均衡器示例(ISSCC 2007)**：
```
抽头间隔: T_b/2
数据率: 1Gb/s
基于3阶延迟单元

延迟线架构:
• 模拟延迟(不能用移位寄存器)
• 使用有源/无源延迟单元
• 抽头系数乘法器
```

### 1.2 RX FIR优缺点

```
优点:
• 有足够动态范围时，可放大高频内容(而非衰减低频)
• 可消除pre-cursor和超过滤波器范围的ISI
• 抽头系数可自适应调优，无需反馈信道

缺点:
• 放大噪声和串扰(线性均衡器不区分信号/噪声/串扰)
• 模拟延迟实现困难
• 抽头精度限制
```

### 1.3 数字RX FIR

**ADC-based RX**：
```
架构:
• 高速低/中等分辨率ADC数字化输入信号
• 数字域进行FIR均衡
• 数字延迟、乘法器、加法器

限制:
• ADC分辨率有限
• 功耗高(快速ADC+数字滤波器)

示例 (Harwood ISSCC 2007):
• 12.5GS/s 4.5-bit Flash ADC (65nm CMOS)
• 2-tap FFE + 5-tap DFE
• XCVR功耗: 330mW (TX inc.)
• Analog: 245mW, Digital: 85mW
```

## 二、RX CTLE均衡

### 2.1 无源CTLE

**传递函数**：
```
H(s) = (1 + R₁C₁s) / (1 + (R₁||R₂)(C₁+C₂)s)

其中:
• R₁, R₂: 电阻
• C₁, C₂: 电容

直流增益: H(0) = R₂/(R₁+R₂)
高频增益: H(∞) = 1

零极点:
ω_z = 1/(R₁C₁)
ω_p = (R₁+R₂)/(R₁R₂(C₁+C₂))
```

**峰化计算**：
```
高频增益/直流增益 = ω_p/ω_z = (R₁+R₂)C₁/(R₁(C₁+C₂))

典型值:
• DC gain: 0dB (衰减)
• HF gain: 6-12dB (峰化)
```

### 2.2 有源CTLE

**架构**：
```
输入放大器+RC退化:
• 提供频率峰化+增益
• 受限于放大器增益带宽积

传递函数(双极点系统):
H(s) = g_mR_D · (1 + g_mR₂C₁s)/(1 + g_mR₂C_D)s · 1/(1 + s/R_DC_D)

其中:
• g_m: 跨导
• R_D: 负载电阻
• C_D: 总负载电容
```

**峰化与直流增益**：
```
直流增益: A_DC = g_mR_D/(1 + g_mR₂)
理想峰化增益: A_peak = g_mR_D
峰化比: A_peak/A_DC = (1 + g_mR₂)

零点频率: ω_z = 1/(R₂C₁)
第一极点: ω_p1 = g_m/(C_D)
第二极点: ω_p2 = 1/(R_DC_D)
```

### 2.3 CTLE调谐

**调整方法**：
```
调整退化电阻和电容:
• 调整零频率和第一极点
• 设置峰化和直流增益

增加C_S:
• 零点向低频移动
• 第一极点也向低频移动
• 不影响(理想)峰化

增加R_S:
• 零点向低频移动
• 增加峰化(降低直流增益)
• 对第一极点影响最小
```

### 2.4 CTLE限制

```
PVT敏感:
• 极点/零点位置随工艺、电压、温度变化
• 需要自适应调谐

通常限于一阶补偿:
• 难以实现高阶均衡
• 对于长尾ISI补偿能力有限
```

![](lecture8_ee720_rx_adaptive_eq_images/img-102.jpg)

> 🔍 深度说明：
> 【研究背景】CTLE通过零点补偿信道高频衰减，是高速SerDes RX的标配模拟前端；无源CTLE和有源CTLE各有适用场景，需要根据功耗、面积、增益需求选择。
> 【核心结论】无源CTLE的峰化比 = (R₁+R₂)C₁/(R₁(C₁+C₂))，典型实现6-12dB高频峰化；有源CTLE通过跨导 gm 和负载电阻 R_D 控制峰化比 A_peak/A_DC = (1 + g_mR₂)；CTLE极点/零点对PVT敏感，需要自适应调谐。
> 【工程价值】对于56Gb/s+ SerDes，CTLE提供必要的模拟前端均衡，配合DFE消除残余ISI；PVT敏感度要求设计时考虑corner覆盖，自适应调谐是必需的。
> 【落地注意】CTLE的一阶补偿特性限制了长尾ISI的消除能力；对于长尾ISI主导的信道，需要更多依赖DFE；CTLE调谐环路的收敛速度影响启动时间和动态信道变化响应。

## 三、RX DFE均衡

### 3.1 DFE原理

**非线性判决反馈均衡**：
```
结构:
输入 → 前向滤波 → 判决器 → 反馈滤波 → 减法器
                              ↑
                        已判决符号

数学表达:
ỹ_k = y_k - Σ w_i · d_{k-i}
        (前向)    (反馈，d为已判决符号)

特点:
• 非线性处理(判决引入)
• 直接从输入减去已判定符号造成的ISI
```

### 3.2 DFE优缺点

```
优点:
• 不放大噪声和串扰
• 抽头系数可自适应调优，无需反馈信道
• 可消除长尾post-cursor ISI

缺点:
• 无法消除pre-cursor ISI
• 错误传播风险(但实际BER=10⁻¹²时概率极低)
• 关键反馈时序路径
• ISI消除时序复杂化CDR相位检测
```

### 3.3 DFE抽头系数设置

**基础规则**：
```
如果仅有DFE均衡:
• DFE抽头系数 = 未均衡信道脉冲响应值
• [w₁, w₂, ...] = [a₁, a₂, ...]

如果有其他均衡(TX EQ等):
• DFE抽头系数 = pre-DFE脉冲响应值
• TX EQ优化时不需关心DFE会处理的ISI项

灵活性:
• DFE提供了其他均衡器电路优化的灵活性
```

### 3.4 直接反馈DFE

**TI 6.25Gb/s 4-tap DFE**：
```
架构:
• ½ rate架构
• 4-tap DFE
• 自适应抽头算法

时序约束:
• 必须在1 UI内解决数据和反馈
• 实际设计: ½ UI收敛(自适应均衡+CDR)

抽头精度:
• TAP1: 5 bits
• TAP2: 4 bits + sign
• TAP3,4: 3 bits + sign
```

**关键时序路径**：
```
t_CLK→Q + t_PROP_MUX + t_PROP_A2 ≤ 1 UI

其中:
• t_CLK→Q: 锁存器时钟到Q延迟
• t_PROP_MUX: MUX传播延迟
• t_PROP_A2: 反馈路径延迟
```

### 3.5 DFE环路解卷绕

**loop unrolling技术**：
```
原理:
• 不在1 UI内反馈并减去ISI
• 预计算2种可能(1-tap DFE)
• 可调判决器阈值

单抽头DFE解卷绕:
• d_{k-1}=+1时: 阈值 = +α
• d_{k-1}=-1时: 阈值 = -α
• 其中 α = w₁

多抽头时:
• 比较器数量 = 2^#taps
• 复杂度指数增长
```

![](lecture8_ee720_rx_adaptive_eq_images/img-403.jpg)

> 🔍 深度说明：
> 【研究背景】DFE环路解卷绕（loop unrolling）是高速DFE实现的关键技术，通过预计算所有可能的判决结果来消除反馈时序瓶颈，是56Gb/s+ SerDes的主流架构。
> 【核心结论】Loop unrolling将反馈延迟从关键路径移除，但代价是复杂度指数增长（2^N个比较器）；单抽头DFE需要2个可调阈值比较器，双抽头需要4个，以此类推。
> 【工程价值】对于56Gb/s NRZ，单抽头loop unrolled DFE是实际可行的；更多抽头需要考虑unrolled结构的面积和功耗开销；实际设计通常采用混合方案——部分抽头unrolled，部分串行反馈。
> 【落地注意】Loop unrolled DFE的比较器阈值精度影响均衡效果；阈值调谐需要专门的校准电路；错误传播在实际BER=10⁻¹²时概率极低，但设计时仍需考虑corner条件下的行为。

### 3.6 DFE summer类型

**电阻负载summer**：
```
性能关键:
• summer必须达到一定精度(>95%)
• 用于ISI消除

设计权衡:
• 输出摆幅 vs 建立时间
• 可能需要大偏置电流

Swing = I·R, τ = RC
```

**积分summer**：
```
ISSCC 2007 (Park):
• 电流积分到负载电容
• 消除RC建立时间
• ΔT/C > R (相同输出摆幅)

优势:
• 偏置电流可降低3倍
• 降低功耗
```

### 3.7 FIR反馈vs IIR反馈DFE

**FIR反馈DFE**：
```
问题:
• FIR反馈需要很多抽头消除ISI
• 对于长尾ISI效率低

适用场景:
• 谐振信道(stub、多重反射)
• 需要精确消除每个后标
```

**IIR反馈DFE**：
```
ISSCC 2009 (Liu):
架构:
• 大第一后标H₁: 用正常FIR反馈抽头消除
• 平滑长尾ISI(第二后标及以后): 用低通IIR反馈滤波器消除

适用条件:
• 信道平滑(无显著反射)
• 长尾ISI可近似指数衰减
• 示例: 片上导线、硅载波线

注意:
• 对于有明显反射的信道，效果不佳
```

![](lecture8_ee720_rx_adaptive_eq_images/img-405.jpg)

> 🔍 深度说明：
> 【研究背景】FIR反馈DFE和IIR反馈DFE是两种不同的长尾ISI消除策略，各有其适用场景；FIR适用于谐振信道（stub、多重反射），IIR适用于平滑信道（片上导线、硅载波线）。
> 【核心结论】IIR反馈DFE（ISSCC 2009 Liu）通过低通滤波器消除长尾ISI，效率高于FIR；第一后标用FIR抽头消除，后续用IIR滤波器；选择依据是信道特性——有反射选FIR，平滑选IIR。
> 【工程价值】对于不同信道类型，需要选择合适的DFE架构；混合方案（FIR+IIR）可以兼顾效率和精度；积分summer（ISSCC 2007 Park）可降低功耗3倍。
> 【落地注意】IIR滤波器的系数设计需要精确信道模型；FIR/IIR混合架构的时序收敛更复杂；积分summer的精度依赖于时钟抖动和电容匹配。

## 四、自适应均衡技术

### 4.1 均衡器设置方法

```
固定设置:
• 基于实验室测量选择最优值
• 对制造和环境变化敏感

自适应调谐:
• 根据实际信道优化
• 适应变化的条件和数据率

自适应均衡关键问题:
1. 提取均衡校正(误差)信号
2. 自适应算法和硬件开销
3. 将校正信息传达给均衡器电路
```

### 4.2 TX FIR自适应

**误差提取**：
```
Stojanovic JSSC 2005:
• 在RX输入测量响应
• 误差信息: RX输入 vs 期望符号电平dLev
• 需要RX额外采样器(可编程阈值)
```

**Sign-sign LMS算法**：
```
抽头更新:
w_{k,n+1} = w_{k,n} + Δ · sign(d_n) · sign(e_n)

其中:
• w_k: 抽头系数
• n: 时间索引
• k: 抽头索引
• d_n: 接收数据
• e_n: 相对于期望数据电平的误差
• dLev: 期望数据电平

dLev自适应:
dLev_{n+1} = dLev_n - Δ_dLev · sign(e_n)
```

**反馈信道**：
```
选项1: 低速共模反馈(~10Mb/s)
• 从RX到TX在同一差分对上共模信令

选项2: 高速TX通道
• RX侧高速TX通道通信回TX
• 8B/10B/Q编码灵活性允许低数据率tap适配信息
```

### 4.3 CTLE调谐

**PSD测量**：
```
方法:
• 比较随机数据的低频和高频谱含量
• 理想随机数据: 低频功率和高频功率有可预测比率

误差信号:
• 来自两种功率分量的误差
• 用于伺服环路调谐CTLE

随机数据频谱:
S_x(f) = T_b · [sin(πfT_b)/(πfT_b)]²
```

**输出幅度测量**：
```
方法:
• 比较低频和高频平均幅度
• 将均衡数据近似为正弦波
• 调整直到高频峰峰值摆幅匹配低频峰峰值

目标:
• 高频幅度 = 低频幅度
• 表明频率响应已展平
```

**数据边沿分布监测**：
```
方法:
• 子周期精度过采样数据位
• 测量边沿分布的宽度和形状

目标:
• 最大化眼图张开
• 等价于最小化边沿分布标准差
```

### 4.4 DFE调谐

**FIR反馈DFE**：
```
方法:
• 2x过采样均衡信号(边沿)
• 提取信息调谐DFE和驱动CDR环路
• Sign-sign LMS算法

采样位置:
• 数据采样: 眼图中心
• 边沿采样: 眼图边沿
```

**IIR反馈DFE**：
```
ISSCC 2011 (Huang):
• 专门针对IIR反馈抽头的调谐方法
• 类似FIR反馈的边沿采样技术
```

![](lecture8_ee720_rx_adaptive_eq_images/img-407.jpg)

> 🔍 深度说明：
> 【研究背景】自适应均衡通过实时调整均衡器参数来跟踪信道变化，是高速SerDes系统可靠工作的关键；sign-sign LMS算法以其低复杂度成为实际部署的主流选择。
> 【核心结论】Sign-sign LMS算法简化了乘法运算（仅需符号比较），适合硬件实现；误差信号提取方法包括PSD测量、输出幅度测量、边沿分布监测；反馈信道可采用低速共模或高速编码通道。
> 【工程价值】自适应均衡对于应对制造variation和环境变化至关重要；CTLE的PSD/幅度调谐和DFE的边沿采样是成熟技术；设计时需权衡收敛速度 vs 稳定性。
> 【落地注意】sign-sign LMS的量化效应可能导致收敛慢或震荡；反馈信道的带宽需要满足参数更新需求；边沿采样时钟抖动的直接影响均衡精度。

## 五、高级调制与多频段信令

### 5.1 高级调制目标

```
均衡目标:
• 展平信道响应至Nyquist频率

高级调制方案:
• 减少频率依赖损耗
• 将Nyquist频率移至更低值
• 4-PAM (PAM-4)
• 双二进制(Duo-binary)
```

### 5.2 多频段信令

**频段划分**：
```
传统方法:
• 均衡至基带Nyquist频率

多频段方法:
• 将信道划分为损耗较少的多个频段
• 每个子频段独立均衡
• 减少每个子频段的均衡复杂度

要求:
• 上/下变频
• 复杂度增加

应用:
• DSL调制解调器(极端挑战信道)
• 近期出现在高速链路研究原型中

示例:
• Kim ISSCC 2019: 56Gb/s 总计15个频段
• Beyene: 2×10Gb/s双二进制，15个频段
```
