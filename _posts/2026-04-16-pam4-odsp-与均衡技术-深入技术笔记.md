## 1. PAM4 调制基础

### 1.1 PAM4 信号定义

**PAM4 (Pulse Amplitude Modulation 4-Level)** 用四个离散的幅度电平来表示两比特信息：

```
电平
  3  ──────────────────  11 (逻辑11)
  2  ──────────────────  10 (逻辑10)
  1  ──────────────────  01 (逻辑01)
  0  ──────────────────  00 (逻辑00)
      ◄─── 眼高 ───►
```

**格雷码映射**：相邻电平只差1比特，最大限度降低误码传播

### 1.2 PAM4 vs NRZ 对比

| 技术指标 | NRZ (PAM2) | PAM4 | 变化 |
|---------|-----------|------|------|
| 电平数 | 2 | 4 | 2x |
| 比特/符号 | 1 | 2 | 2x |
| 谱效率 | 1 bit/s/Hz | 2 bit/s/Hz | 2x |
| 所需带宽 | R_b | R_b/2 | 减半 |
| 最小幅度间距 | 2 | 2/3 | -9.54 dB |
| SNR 代价 | 0 dB | ~9.54 dB | -9.54 dB |
| 眼图数量 | 1 | 3 | +2 |
| ADC 分辨率需求 | 4-6 bit | 6-8 bit | 更高 |
| FEC 需求 | KR4 | KP4 | 更强 |
| 典型应用 | 25G/28G NRZ | 56G/112G PAM4 | 升级 |

### 1.3 PAM4 眼图特性

```
PAM4 眼图结构:

       上眼
  ──────┼───── 眼高1
        │     ← 中眼
  ──────┼───── 眼高2
        │     ← 下眼
  ──────┼───── 眼高3
        低

眼高分析:
- 最上/下眼: ~100 mV (典型)
- 中间眼: ~50 mV (最差)
- 眼宽: ~0.3 UI (典型)
```

**关键结论**: PAM4 中间眼的线性度和张开度通常最差，因为同时受到上下两个方向的 ISI 影响

### 1.4 PAM4 系统参数

| 参数 | NRZ | PAM4 | 说明 |
|------|-----|------|------|
| 符号率 | 26 GBaud | 26 GBaud | 相同 |
| 数据率 | 26 Gb/s | 52 Gb/s | 翻倍 |
| 眼高(理论) | 1V | 0.33V | 1/3 |
| SNR 要求 | 低 | 高 ~9.5dB | 补偿需求 |
| ADC 精度 | 3-4 bit | 5-6 bit | 更高分辨率 |

## 2. oDSP 系统架构

### 2.1 oDSP 定义与功能

**oDSP (Optical Digital Signal Processor)** 是高速光传输系统中将模拟光信号转换为数字域并进行智能处理的核心技术模块。

**核心优势**:
- 数字域实现精确的线性与非线性均衡
- 自适应配置和算法迭代升级的灵活性
- 可补偿光路中的多种损伤

### 2.2 完整收发链路架构

```
发送端 (TX)                              接收端 (RX)
┌─────────┐                          ┌─────────────────────┐
│ Gray    │                          │  Gray               │
│ Mapping │                          │  Demapping          │
└────┬────┘                          └─────────┬───────────┘
     │                                         │
┌────▼────┐                          ┌────────▼───────────┐
│  Pre-   │                          │  CD Compensation   │
│ coding   │                          │  (色散补偿)        │
└────┬────┘                          └─────────┬───────────┘
     │                                         │
┌────▼────┐                          ┌────────▼───────────┐
│  TX FFE │                          │  PMD/Polarization  │
│ (预加重) │                          │  Demux             │
└────┬────┘                          └─────────┬───────────┘
     │                                         │
┌────▼────┐                          ┌────────▼───────────┐
│ DAC +   │                          │  Carrier Recovery   │
│ Driver  │  ──────────────▶       │  (载波恢复)        │
└─────────┘    光纤                  └─────────┬───────────┘
                                               │
                                      ┌────────▼───────────┐
                                      │  LMS/CMA           │
                                      │  Equalization      │
                                      └─────────┬───────────┘
                                               │
                                      ┌────────▼───────────┐
                                      │  Decision &       │
                                      │  BER Counting     │
                                      └───────────────────┘
```

### 2.3 TX 端处理流程

```
1. Gray 编码
   └── 将二进制转换为 Gray 码 (降低误码传播)

2. 符号映射
   └── 映射到 PAM4 星座点 (-3, -1, +1, +3)

3. 上采样
   └── 2x 过采样 (便于滤波)

4. 滤波器 (RRC)
   └── 根升余弦滤波 (脉冲成形)

5. TX FFE
   └── FIR 预失真滤波

6. DAC
   └── 数字转模拟

7. 驱动
   └── 驱动调制器
```

### 2.4 RX 端处理流程

```
1. 光电检测 (PD)
   └── 光信号转电信号

2. TIA
   └── 跨阻放大

3. CTLE
   └── 连续时间线性均衡

4. ADC
   └── 高速采样

5. FFE
   └── 前馈均衡

6. DFE
   └── 判决反馈均衡

7. CDR
   └── 时钟数据恢复

8. Gray 解码
   └── 还原二进制数据
```

### 2.5 PAM4 IM/DD vs 相干检测

| 参数 | PAM4 IM/DD | 相干 PAM4 |
|------|------------|-----------|
| 距离 | < 10km | 10km - 1000km+ |
| 复杂度 | 低 | 高 |
| 功耗 | 低 | 高 (oDSP) |
| 成本 | 低 | 高 |
| 灵敏度 | 一般 | 优秀 |
| 色散容忍 | 低 | 高 |
| 速率 | 100G/400G | 400G/800G+ |

## 3. 发送端预处理技术

### 3.1 FIR 滤波与 TX FFE

**FIR (Finite Impulse Response)** 滤波器是最常用的发送端预均衡技术。

**数学表达式** (2-tap):
```
y(n) = c(-1) × x(n) + c(0) × x(n-1) + c(1) × x(n-2)

其中:
  c(-1) = 前标 (pre-cursor) 系数
  c(0) = 主标 (main-cursor) 系数
  c(1) = 后标 (post-cursor) 系数
```

**Z 域传递函数**:
```
H(z) = c(-1) + c(0) × z^(-1) + c(1) × z^(-2)
```

**核心作用**: 在发送端对信道的高频衰减特性进行预补偿

**Tap 系数设计**:
| Tap 类型 | 典型值 | 作用 |
|---------|--------|------|
| Main tap | 1.0 | 主信号 |
| Pre-tap | 负值 | 减小前铃 (pre-ringing) |
| Post-tap | 负值 | 减小后铃 (post-ringing) |

**预均衡效果示例**:
- 信道损耗: 13 dB
- FIR 预均衡: 9.7 dB
- 剩余损耗: 3.3 dB (由 RX 端均衡处理)

### 3.2 预加重 vs 去加重

```
预加重示意:
原始信号:    ┌───────┐
             │       │
             │       │
             └───────┘

预加重信号:  ┌───────┐
             │       │  ▲
             │       │▌▌  ← 边沿增强
             └───────┘

去加重示意:
原始信号:    ┌───────┐
             │       │
             │       │
             └───────┘

去加重信号:  ┌───────┐▌▌
             │       │▌▌  ← 非边沿降低
             │       │
             └───────┘
```

### 3.3 预编码 (Precoding)

**核心目的**: 将信道引入的 ISI 转化为已知的、可消除的确定性干扰模式

**模二加预编码**:
```
发送端: p_n = b_n XOR p_(n-1)
接收端: b_hat_n = p_hat_n XOR p_hat_(n-1)

效果: 将"差错传播"转化为"差错累积"
      → 误码模式更集中，有利于 FEC 纠错
```

### 3.4 DAC 架构对比

| 架构类型 | 特点 | 应用 |
|---------|------|------|
| 纯模拟 TX | FIR 在模拟域实现，功耗最低 | 短距离、低功耗场景 |
| DAC-based TX | 数字 FIR + 高速 DAC | **主流方案**，灵活可编程 |

## 4. 接收端均衡技术

### 4.1 CTLE (连续时间线性均衡器)

**CTLE (Continuous Time Linear Equalizer)** 是接收端模拟域的第一个线性均衡模块。

**位置**: TIA → CTLE → ADC

**频率响应**:
```
         增益
           ▲
           │        ──── 高频提升 ────
           │      ╱
           │    ╱
           │  ╱
           │╱
           └─────────────────────────────▶ 频率
              低频    高频
            抑制      增强
```

**传递函数**:
```
H_CTLE(s) = (ω_z / (s + ω_z)) × ((s + ω_p) / (s + α × ω_p))

其中:
  ω_z = 零点频率
  ω_p = 极点频率
  α = 直流增益控制参数
```

**工程目标**: "打开眼图" — 恢复因信道带宽限制而模糊的 PAM4 三个眼图的垂直开口度

**典型配置**:
| 参数 | 范围 |
|------|------|
| DC 增益 | -6dB 到 +6dB |
| 高频峰值 | 可编程 |
| 带宽控制 | 调节频率响应形状 |

**设计权衡**:
- 高频增益 ↑ → 噪声放大 ↑
- DC 增益 ↓ → 低频抑制过度
- 需要平衡: 信号恢复 vs 噪声放大

### 4.2 FFE (前馈均衡器)

**FFE (Feed-Forward Equalizer)** 的核心是一个抽头延迟线滤波器。

**结构**:
```
         延迟 ─┬─ tap n ─┬─ tap n-1 ─ ... ─┬─ tap 0
                │         │                │
原始数据 ───────┼─────────┼────────────────┼──▶ 加法器 ──▶ 输出
                │         │                │
              tap系数    tap系数          tap系数
            (c[n])      (c[n-1])        (c[0])
```

**数学表达式**:
```
y_out = Σ c[k] × x_in[n-k], k = 0, 1, ..., N
```

**FFE vs CTLE**:
| 特性 | CTLE | FFE |
|------|------|-----|
| 实现域 | 模拟 | 数字 |
| 精度 | 受工艺影响 | 高精度 |
| 自适应 | 困难 | 容易 |
| 功耗 | 低 | 中高 |

### 4.3 DFE (判决反馈均衡器)

**DFE (Decision Feedback Equalizer)** 利用已判决符号的信息来消除当前符号所受的 post-ISI。

**结构**:
```
                         ┌──────┐
输入 ──▶ 判决器 ──▶ 输出 │      │
            │            │      │ ← 反馈滤波器
            │            │      │    (FIR)
            │            └──────┘
            │              ▲
            └──────────────┘
                   (滤波后反馈)
```

**工作原理**:
1. 采样当前符号
2. 使用历史判决结果估计 ISI
3. 从当前采样中减去 ISI 估计
4. 判决得到最终结果

**DFE 公式**:
```
r[n] = y[n] - Σ w[k] × d[n-k], k = 1, 2, ..., M

其中:
  y[n] = 接收采样
  w[k] = 反馈抽头系数
  d[n-k] = 历史判决结果
  r[n] = 均衡后的样本
```

**DFE vs FFE 关键差异**:
| 方面 | FFE | DFE |
|------|-----|-----|
| 噪声放大 | 会放大 | **不放大** |
| post-ISI 处理 | 线性抑制 | **利用判决信息消除** |
| 错误传播 | 无 | 有 (判决错误反馈) |

**错误传播问题**:
- 一旦发生判决错误，错误会通过反馈滤波器传播到后续符号
- PAM4 系统中眼图开口较小，判决错误概率高于 NRZ
- 解决方案: 软判决 DFE、限制反馈抽头数、FFE 预均衡降低误判率

### 4.4 接收端完整均衡链

```
完整链路:

TX:  FFE (Pre-emphasis)
     └── 3-tap 或 5-tap FIR

信道: (损耗补偿)
     └── 30dB+ 损耗

RX:  CTLE → AGC → DFE → CDR → 输出
     └── 可编程均衡器组合
```

**PAM4 均衡配置**:
```
TX: FFE (4-tap PAM4)
RX: CTLE + DFE (2-4 tap) + CDR
```

## 5. 自适应均衡算法

### 5.1 为什么需要自适应

- 不同信道特性不同
- 温度/老化导致特性变化
- 速率变更需要重新适配

### 5.2 LMS 算法 (最小均方)

**LMS (Least Mean Square)** 是自适应均衡中最广泛使用的系数更新算法。

**原理**: 沿着误差梯度负方向迭代更新抽头系数

**更新规则**:
```
误差信号: e[n] = d[n] - y[n]
更新规则: w[k] = w[k] + μ × e[n] × x[n-k]

其中:
  d[n] = 期望信号
  y[n] = 实际输出
  μ = 步长参数 (学习率)
  w[k] = 抽头系数
  x[n-k] = 输入样本
```

**收敛条件**:
```
0 < μ < 2/λ_max

其中 λ_max 为输入相关矩阵最大特征值
```

**LMS 算法优点**:
| 特性 | 说明 |
|------|------|
| 计算复杂度 | O(N) - 低 |
| 实现 | 简单，硬件友好 |
| 鲁棒性 | 对信道统计特性变化鲁棒 |

**步长选择**:
| μ 值 | 收敛速度 | 稳态误差 |
|------|---------|---------|
| 大 | 快 | 大 |
| 小 | 慢 | 小 |
| 典型值 | 0.01 ~ 0.1 | - |

### 5.3 CMA (恒模算法)

**CMA (Constant Modulus Algorithm)** 适用于无需训练序列的盲均衡场景。

**应用**: PAM4/QAM 等高阶调制

**原理**: 维持信号恒定模值

**代价函数**:
```
J_CMA = (|y_out|² - R)²

其中 R 是期望的模值
```

**更新规则**:
```
误差: e[n] = R - |y[n]|²
更新: w[k] = w[k] + μ × e[n] × y[n] × x[n-k]
```

**CMA vs DDLMS**:
| 特性 | CMA | DDLMS |
|------|-----|-------|
| 参考信号 | 幅度统计 (盲) | 判决结果 (有监督) |
| 精度 | 粗均衡 | 精确均衡 |
| 收敛速度 | 慢 | 快 |
| 应用阶段 | 初始收敛 | 精细跟踪 |

**两阶段级联**:
1. CMA → 完成粗收敛
2. DDLMS → 精确跟踪

### 5.4 RLS 算法 (递归最小二乘)

**RLS (Recursive Least Squares)** 收敛速度远快于 LMS。

**更新规则**:
```
w(n) = w(n-1) + μ × e(n) × P(n) × v(n)

其中 P(n) 为协方差矩阵的递归估计
```

**复杂度对比**:
| 算法 | 复杂度 | 收敛速度 | 跟踪能力 |
|------|--------|---------|---------|
| LMS | O(N) | 慢 | 好 |
| RLS | O(N²) | 快 (指数) | 优秀 |

**RLS 缺点**:
- 计算复杂度高: O(N²)
- 数值稳定性问题
- 存储需求大

### 5.5 算法选择指南

| 场景 | 推荐算法 |
|------|---------|
| 链路建立初期 | CMA (盲均衡) |
| 精细跟踪 | DDLMS |
| 快变信道 | RLS |
| 低功耗/简单实现 | LMS |

## 6. MLSE 序列估计

### 6.1 MLSE 基本原理

**MLSE (Maximum Likelihood Sequence Estimation)** 代表均衡技术的理论最优方案。

**核心思想**: 在所有可能的发送序列中搜索最可能导致接收序列的那一个

```
MLSE vs 线性均衡

线性均衡:
输入 ──▶ 线性滤波器 ──▶ 判决 ──▶ 输出
        (单点处理)

MLSE:
输入 ──▶ 状态机 ──▶ Viterbi 算法 ──▶ 最优序列
        (考虑符号间依赖)
```

### 6.2 信道模型

**离散信道模型**:
```
h[0], h[1], ..., h[L]  (L+1 抽头)

y[n] = Σ h[k] × x[n-k] + n[n]
```

**状态定义**:
| 参数 | 说明 |
|------|------|
| 状态数 | M^(L-1) (NRZ) |
| 路径数 | 每状态 2 条 |
| M | 调制阶数 |

### 6.3 Viterbi 算法

**步骤**:

1. **初始化**: 设置时刻 k=0 的路径度量为 0

2. **分支度量计算**: 对每个状态计算分支度量
   ```
   λ_k(σ_{k-1}, s_k) = |y_k - Σ h_i × s_{k-i}|²
   ```

3. **ACS (加-比-选)**: 对每个新状态，计算累计度量并选择幸存路径

4. **回溯**: 在序列末端选择最优路径

### 6.4 复杂度问题

**状态数膨胀**:
| 调制 | L=2 | L=3 | L=4 |
|------|-----|-----|-----|
| QPSK (M=4) | 16 | 64 | 256 |
| 16QAM (M=16) | 256 | 4096 | 65536 |

**MLSE vs DFE 对比**:
| 方面 | MLSE | DFE |
|------|------|-----|
| 最优性 | 全局最优 | 局部最优 |
| 复杂度 | 高 (指数) | 低 (线性) |
| 延迟 | 长 | 短 |
| 功耗 | 高 | 低 |
| 适用 | 高性能 | 商用 |

### 6.5 简化 MLSE 方案

- **Truncated MLSE**: 限制搜索深度
- **Volterra-MLSE**: 结合非线性建模
- **神经网络辅助 MLSE**: 用 DNN 学习信道特性

## 7. 前向纠错编码 FEC

### 7.1 FEC 基本原理

**FEC (Forward Error Correction)** 通过引入冗余比特，使接收端能够纠正一定范围内的错误比特。

**编码参数**:
```
RS(n, k) = RS(544, 514)
  n = 544 symbols (10bit/symbol)
  k = 514 data symbols
  t = 15 error correction
```

**纠错能力**:
- 最多纠正 15 个符号错误
- 对应约 150 bits 错误（突发）

### 7.2 KP4 FEC vs KR4 FEC

| FEC 方案 | 编码结构 | 开销 | 编码增益 | 译码前 BER 阈值 |
|---------|---------|------|---------|----------------|
| KR4 | RS(528,514) | ~3% | ~5.6 dB | 2.2e-4 |
| **KP4** | RS(544,514) | ~6% | **~7 dB** | 2.4e-4 |

**KP4 FEC 核心定义**:
- RS(528, 514, T=7, M=10): 用于 25G NRZ 的 KR4 FEC
- RS(544, 514, T=15, M=10): 用于 28G/56G/112G PAM4 的 **KP4 FEC**

### 7.3 软判决 vs 硬判决

**硬判决**:
```
输入: 0/1 判决结果
输出: 纠正后的码字
复杂度: 低
```

**软判决**:
```
输入: 量化后的置信度 (多bit)
输出: 最可能的码字
增益提高: 约 1-2 dB
```

### 7.4 RS 解码器四步架构

**第一步: 伴随式计算**
```
s = [s_1, s_2, ..., s_{2t}] = H × R^T

若所有 s_i = 0 → 无错误
```

**第二步: BM 算法**
```
λ(x) = (1 - P_1 × x)(1 - P_2 × x)...(1 - P_v × x), v ≤ t

恢复错误位置多项式和错误值多项式
```

**第三步: Chien 搜索**
```
在 GF(2^10) 中找到使 λ(x) = 0 的根
```

**第四步: Forney 算法**
```
计算各错误位置上的正确符号值
```

### 7.5 各型 FEC 方案对比

| FEC 方案 | 编码结构 | 开销(OH) | 编码增益 | 译码前 BER | 典型应用 |
|---------|---------|---------|---------|-----------|---------|
| KP4 | RS(544,514) | 5.83% | 7.04 dB | 2.4e-4 | 56G/112G PAM4 |
| CI-BCH | BCH(1022,990) | 6.67% | 9.15 dB | 4.2e-3 | 短距离高增益 |
| CFEC | Staircase+Hamming | 14.8% | 10.6 dB | 1.21e-2 | 超高速长距 |
| **OFEC** | Braided+BCH | 15.3% | **11.1 dB** | 1.93e-2 | 下一代高增益 |
| KP4+Hamming | KP4+Hamming | 12.9% | 9.56 dB | 4.5e-3 | 高增益双层 |

### 7.6 FEC 发展趋势

```
结论: FEC 在 oDSP 会越来越重要

原因:
- PAM4 符号率从 56G → 112G → 224G
- 系统对 FEC 依赖程度持续提升
- 需要更强编码增益的 FEC 方案
```

## 8. 线侧光路损伤与补偿

### 8.1 铜缆 vs 光缆损伤对比

| 损伤类型 | 铜缆特性 | 光缆特性 |
|--------|--------|---------|
| 带宽限制 | 电气走线损耗 | 调制器、驱动器、TIA 带宽限制 |
| 射频反射 | 电连接器不匹配 | 光电元件界面反射 |
| 噪声 | 量化噪声、抖动、热噪声 | 暗电流、热噪声、散粒噪声、RIN |
| 幅度非线性 | — | 调制器、驱动器、TIA 非线性 |
| 时间非线性 | — | 调制器啁啾 |
| 非线性光纤损伤 | — | 色散(CD)、偏振模色散(DGD)、MPI |

### 8.2 激光调制器啁啾

**MZM (Mach-Zehnder 调制器)**:
- 双臂干涉结构
- 理论上无啁啾 (chirp-free)
- **长距离高速传输首选**

**EML (电吸收调制激光器)**:
- 主要受瞬态啁啾影响
- 与调制信号上升/下降沿强相关

**DML (直接调制激光器)**:
- 同时受瞬态啁啾和绝热啁啾影响
- 啁啾特性最复杂

### 8.3 色度色散 (Chromatic Dispersion, CD)

**定义**: 不同频率的信号具有不同的传输速率

**影响一 (符号展宽)**:
```
ΔT = D × L × Δλ

其中:
  D = 色散系数 (ps/nm/km)
  L = 光纤长度
  Δλ = 光脉冲谱宽
```

**影响二 (功率衰落)** - IM/DD 系统特有:
```
P(dB) = 10 × log10(|sin(φ)/φ|)

φ = D_z × f × λ × π
```

**色散补偿**:
```math
H_{cd}(ω) = exp(j × β₂ × L × (ω-ω₀)²/2)
```

**实现方法**:
1. FFT 将信号变换到频域
2. 与色散传递函数相乘
3. IFFT 变回时域

### 8.4 偏振模色散 (PMD)

**定义**: 光纤中两个正交偏振态以不同速度传输

```
X偏振 ──────────────────────────▶  快轴
Y偏振 ──────                    ─▶  慢轴
        ◄──── ΔT 差分群延时(DGD)
```

**PMD 参数**:
- 平均 DGD: < ps/√km
- 一阶 PMD: 随机变化
- 二阶 PMD: DGD 频率相关

**MIMO 均衡**:
```
       ┌─────────────────────────────────┐
       │   2x2 MIMO 复数均衡器           │
       └─────────────────────────────────┘
  hxx ──▶ ──▶ ──▶ ──┐
                       ├──▶ X'out
  hxy ──▶ ──▶ ──▶ ──┘
                       │
  hyx ──▶ ──▶ ──▶ ──┐│
                       ├──▶ Y'out
  hyy ──▶ ──▶ ──▶ ──┘
```

### 8.5 多径干涉 (MPI)

**关键差异**:
| 类型 | 受影响符号长度 |
|------|--------------|
| 电反射 | 几十 ~ 几百 UI |
| **光 MPI** | **约 10000 UI** (~1ms @ 10 Gbaud) |

**补偿挑战**: MPI 导致极长时延的干扰，传统有限长 FIR 无法有效补偿

### 8.6 线侧均衡方案对比

| 均衡方案 | 原理 | 优点 | 缺点 |
|---------|------|------|------|
| DFE | 利用已判决符号消除 post-ISI | 不放大噪声 | 错误传播，长 ISI 需多抽头 |
| FFE | 线性滤波补偿 | 简单，无错误传播 | 消除长 ISI 需极多抽头 |
| **MLSE** | Viterbi 最优序列搜索 | **理论最优** | 复杂度高 |
| Volterra FFE | 非线性建模 | 建模激光啁啾等 | 复杂度极高 |

## 9. LPO 线性可插拔光模块

### 9.1 LPO 核心概念

**LPO (Linear Pluggable Optics)** 特点:
- 去掉 DSP 芯片
- 使用线性驱动/接收
- 与 CPO (共封装光学) 互补

### 9.2 架构对比

```
传统可插拔 (Traditional Pluggable):
┌─────────────┐
│  DSP       │  ← 大功耗
│  CDR       │
│  Driver    │
│  Laser     │
│  TIA       │
└─────────────┘

LPO:
┌─────────────┐
│  Driver    │  ← 低功耗
│  Laser     │
└─────────────┘
     ↑
   Linear (无 DSP)
```

### 9.3 LPO vs 传统 DSP

| 对比项 | 传统 DSP | Linear Drive |
|--------|---------|--------------|
| 功耗 | ~15 pJ/bit | ~5 pJ/bit |
| 延迟 | ~100ns | ~0 |
| 复杂度 | 高 | 低 |
| 性能 | 最优 | 依赖信道质量 |

### 9.4 应用场景

- 数据中心交换机互联
- AI/ML GPU 互联
- 超大规模数据中心

**Nvidia 在 IPEC 的目标**:
```
单 GPU: 4 Tb/s
集群: 1000+ GPUs
距离: 100s 米

Linear Drive 目标:
- <4W (多模)
- <8W (单模)
- 800G 接口
```

### 9.5 LPO 挑战

1. **鲁棒性**: 无 CDR 意味着对链路损伤更敏感
2. **互操作性**: 需要标准化的测试点 (TP1a, TP4)
3. **可扩展性**: 200G/lane？

## 10. 224G PAM4 系统挑战

### 10.1 112G vs 224G PAM4 对比

| 参数 | 56 GBaud (112G) | 112 GBaud (224G) | 变化 |
|------|-----------------|------------------|------|
| 波特率 | 56 GBaud | 112 GBaud | 2x |
| Nyquist 频率 | 28 GHz | 56 GHz | 2x |
| UI | 17.86 ps | 8.93 ps | 减半 |
| 典型信道损耗 | 25-30 dB | 35-40 dB | +10 dB |
| 抖动预算 | ~0.3 UI | ~0.25 UI | 更紧 |
| CDR 带宽需求 | 10-30 MHz | 30-50 MHz | 更高 |

### 10.2 112G PAM4 技术挑战

1. **带宽压力**: 56 Gbaud 的 PAM4 信号的 3dB 带宽约为 28 GHz
2. **ADC 分辨率**: PAM4 眼高仅为 NRZ 的约 1/3，要求 ADC 有效位数 6.5-7 bit+
3. **均衡深度**: 56 Gbaud 下信道高频损耗更严重
4. **CDR 精度**: 更高符号率意味着相同相位抖动对应更大时间误差
5. **FEC 设计**: 112G PAM4 原生 BER 更高

### 10.3 800G/1.6T 以太网

**400G 以太网光接口标准**:
| 标准 | 波长×调制 | 距离 |
|------|---------|------|
| 400GBASE-LR8 | 8λ×50G PAM4 | 10km |
| 400GBASE-DR4 | 4λ×100G PAM4 | 500m |
| 400GBASE-FR8 | 8λ×50G PAM4 | 2km |

**800G 以太网 (IEEE 802.3ck)**:
- 每通道 112 Gbps → 56 Gbaud PAM4
- KP4 FEC 作为标准 FEC

### 10.4 CPO (共封装光学)

**CPO 优势**:
- 光引擎与交换芯片紧耦合
- 高速电气互连损耗从 20+dB 降至 5dB 以下

**oDSP 变化**:
- TX 预均衡强度可降低 → 节省 TX 功耗
- RX 均衡复杂度可能降低
- ADC 采样率和分辨率要求进一步提升
