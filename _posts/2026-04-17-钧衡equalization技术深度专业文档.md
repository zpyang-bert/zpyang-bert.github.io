## 执行摘要

"钧衡"即 Equalization，是高速数字通信系统中弥补信道损伤、恢复信号完整性的核心技术。随着高速SerDes从56G向224G演进，均衡技术从简单的模拟滤波演进为涉及TX FFE、CTLE、DFE、MLSE乃至神经网络均衡的复杂系统工程。IEEE 802.3标准对各速率等级下的均衡器配置、抽头系数范围、均衡性能指标均有严格定义。本文档整合知识库中所有均衡相关研究，从理论原理、架构实现、标准规范、设计挑战四个维度进行系统性深度论述，力图成为均衡技术领域的完整技术参考。

## 第一章 钧衡基础理论

### 1.1 信号失真与码间干扰

高速数字信号在信道（印刷电路板、背板、铜缆、光纤）中传输时，主要受到以下物理效应影响导致信号失真：

**插入损耗（Insertion Loss）**：导体趋肤效应和介质损耗导致信号高频分量衰减，表现为信号边沿斜率降低和脉冲展宽。

**频率相关性损耗**：信道的传递函数H(f)并非平坦，其幅度响应|H(f)|随频率增加而下降，相位响应∠H(f)呈现非线性。不同频率分量以不同速度传播，进一步加剧脉冲展宽。

**反射与回波损耗**：阻抗不匹配导致部分信号能量被反射回源端，反射信号与入射信号叠加形成相长或相消干涉，在时域表现为特定位置的过冲和下冲。

**串扰（Crosstalk）**：相邻通道的电磁耦合在受害通道引入噪声，表现为临近信号边沿的干扰脉冲。

上述效应的共同结果是在接收端产生**码间干扰（Inter-Symbol Interference, ISI）**——当前符号的判决时刻受到相邻符号残留能量的影响，导致眼图闭合、判决裕量减少。

### 1.2 均衡的根本目标

均衡的数学本质是**对信道传递函数H(z)的逆滤波**。若接收信号为y[n] = h[n] * x[n] + noise[n]，则均衡器试图构建一个逆滤波器W(z) ≈ 1/H(z)，使得W(z)·y[n] ≈ x[n]。

均衡的目标包含三个层面：

**频域目标**：补偿信道频率响应，使通带内增益趋于平坦，总传递函数H(z)·W(z) ≈ 1。

**时域目标**：使合成脉冲响应满足Nyquist无ISI准则——在符号采样时刻，周围符号的贡献之和为零。

**系统目标**：最大化眼图张开度（眼宽×眼高），最小化误码率（BER），在给定功耗和延迟预算内完成信号恢复。

### 1.3 均衡技术的完整分类

## 第二章 发送端均衡技术

### 2.1 TX FFE 原理与架构

发送端前向均衡（TX Feed-Forward Equalizer, TX FFE）是发送器对信号进行预失真的技术，通过在发送端对当前符号及其相邻符号进行加权组合，补偿信道的频率选择性衰减。

TX FFE的核心是一个N-tap FIR滤波器：

```
          c[-1]     c[0]      c[1]      c[2]
           │        │         │         │
x[n+1] ──▶[D]──┬──▶[D]──┬──▶[D]──┬──▶ Summier ──▶ 输出
               │        │         │
              tap     tap      tap
             pre     main     post1    post2
```

**Z域传递函数**：
```
W(z) = c[-1]·z^{+1} + c[0]·z^{0} + c[1]·z^{-1} + c[2]·z^{-2}
```

**IEEE 802.3标准中的TX FFE规范**：

802.3ck（100G/lane）定义的4-tap FFE系数范围：

| 抽头 | 符号 | 系数范围 | 步长 | 说明 |
|------|------|---------|------|------|
| c[-1] | pre-cursor | -0.1 ~ 0.0 | 0.01 | 前一个符号的预补偿 |
| c[0] | main-cursor | 0.6 ~ 1.0 | 固定 | 当前符号增益 |
| c[1] | post-cursor | -0.4 ~ 0.0 | 0.01 | 后一个符号的衰减 |
| c[2] | post2 | -0.2 ~ 0.0 | 0.01 | 后第二个符号的衰减 |

802.3dj（200G/lane）继承并扩展了上述规范，要求发射机支持4-tap FFE以补偿26.6 GHz处的~28 dB背板损耗。

**去加重（De-emphasis）度量**：工程中常以dB表述去加重深度，定义为低频增益相对高频增益的衰减比：

```
De-emphasis (dB) = 20·log10(低频幅度/高频幅度)
```

典型10Gb/s系统的去加重设置：主抽头增益0 dB，后抽头增益-6 dB至-14 dB。

### 2.2 预加重与去加重的物理机制

**预加重（Pre-emphasis）**在信号边沿（即高频分量）增加能量，同时保持低频分量的幅度不变。这等效于使发送信号的频谱预补偿信道的频域衰减特性。

**去加重（De-emphasis）**降低信号的低频分量幅度，同时保持边沿（高频）不变。从频域看，两者本质相同；从时域看，预加重增强边沿，去加重压缩平坦区。

**功率约束**：预加重和去加重都在峰值功率约束下工作。预加重通过将能量从"平顶"转移到"边沿"来补偿高频衰减，不增加峰值功率。去加重则通过整体压缩动态范围来减少ISI，同样不增加峰值功率。

**前抽头（Pre-cursor）均衡的意义**：前抽头c[-1]用于补偿反射引起的**前向码间干扰**（pre-cursor ISI）。当信道中存在远端反射时，当前符号的能量会延迟到后续符号，但反射路径也可能使后续符号的能量提前泄漏到当前符号。前抽头均衡专门用于对抗这种前向干扰，而DFE由于采用反馈结构，原则上无法消除pre-cursor ISI（这是DFE的根本缺陷之一）。

### 2.3 TX FFE的自适应与训练

TX FFE系数的获取有两种主要方式：

**固定系数设计**：基于典型信道模型预设系数。适用于信道特性相对稳定的环境（如固定背板），优点是实现简单、无需训练序列开销。

**链路训练（Link Training）**：通过发送已知训练序列，接收端测量信道响应，计算最优TX FFE系数并回传至发送端。802.3标准定义了完整的链路训练协议（Link Training State Machine），包括：

- **Training Pattern 1 (TR_TP1)**：发送固定码型（通常是重复的55AA...），用于接收端时钟恢复和自动增益控制
- **Preset**：将TX FFE系数重置为默认状态
- **Preset + Equalization**：发送特定FFE系数配置请求
- **Update**：请求接收端报告当前信道质量（SNR/BER），发送端据此调整系数

**TX FFE的训练算法**：接收端在链路训练期间运行信道估计，然后通过以下公式计算最优发送端预均衡系数：

```
c_opt = argmin_c ||h * c - δ[0]||²
```

即找到使合成信道脉冲响应h*c接近单位冲激响应δ[0]的发送系数向量c。

## 第三章 接收端线性均衡技术

### 3.1 CTLE 连续时间线性均衡器

CTLE（Continuous Time Linear Equalizer）是接收端模拟域的第一个线性均衡模块，位于跨阻放大器（TIA）之后，通常在ADC或采样器之前。

**无源CTLE**：基于RC网络的有频率选择性的衰减器：

```
        R₁
Vin ──┬──┤├───┬── Vout
      │  R₂   │
      │        C₁
      └────────┘ (接地)
```

**传递函数**：
```
H(s) = (1 + R₁C₁s) / (1 + (R₁||R₂)C₁s)
```

- 零点频率：ωz = 1/(R₁C₁)——提供高频提升
- 极点频率：ωp = (R₁+R₂)/(R₁R₂C₁)——限制低频增益

**有源CTLE**：利用跨导放大器（gm-C）实现可调增益的均衡器：

```
H(s) = gm·RD / (1 + gm·RS/2) · (1 + s/ωz) / (1 + s/ωp)
```

可调参数包括源极退化电阻RS、源极电容CS和跨导gm，通过调节这些参数改变均衡深度（高频提升量）。

**CTLE的关键性能参数**：

| 参数 | 典型值 | 说明 |
|------|--------|------|
| DC增益范围 | -6 dB ~ +6 dB | 可编程的低频增益 |
| 高频峰值 | 0 ~ 15 dB | 均衡深度 |
| 带宽 | 20 ~ 50 GHz | 处理PAM4信号所需 |
| 噪声 | < 5 nV/√Hz | 等效输入噪声 |
| 功耗 | 30 ~ 80 mW | 有源CTLE |

**CTLE的工程权衡**：CTLE在提升高频（补偿信道损耗）的同时也会放大高频噪声，这种效应称为**噪声增强（Noise Enhancement）**。噪声增强量与均衡深度正相关——过度的CTLE峰值虽然改善了眼高，但恶化了信噪比。工程中需要在"眼图张开"和"噪声放大"之间寻找最优平衡点。

**CTLE与信道特性的匹配**：CTLE的零点/极点位置需要与信道损耗频率特性匹配。对于典型PCB背板（损耗约按f^0.5增长），CTLE的零点通常设置在1-5 GHz范围，极点设置在15-30 GHz范围，以实现最优的高频补偿。

### 3.2 RX FFE 数字前馈均衡器

RX FFE是接收端数字域实现的抽头延迟线滤波器，与TX FFE的FIR结构本质相同，但位于ADC之后，能够利用ADC采样的全精度进行滤波。

**数学模型**：
```
y_out[n] = Σ H[k] · x_in[n-k], k = -N, ..., M
```

其中H[k]为抽头系数，x_in为ADC采样序列，y_out为均衡后输出。

**RX FFE与TX FFE的协同**：TX FFE和RX FFE共同分担均衡任务。TX FFE负责预补偿，减少信号在信道中的高频衰减；RX FFE负责精细均衡，处理残余ISI和TX FFE未能完全补偿的高频分量。

**自适应算法**：RX FFE的抽头系数通过自适应算法实时更新，以跟踪信道特性的慢变化（如温度漂移、老化效应）。主要有两种算法：

**CMA（Constant Modulus Algorithm，恒模算法）**：
```
误差: e = R² - |y[n]|²
更新: H[k] = H[k] + μ · x_in[n-k] · y*[n] · e
```

CMA是一种盲均衡算法，无需训练序列，通过维持信号恒定模值来更新系数。适用于均衡器初始收敛阶段。CMA的缺点是收敛速度慢、稳态误差大、无法区分载波频率偏移与信道失真。

**DDLMS（Decision-Directed Least Mean Square，判决导向LMS）**：
```
误差: e = y_out[n] - d_hat[n]
更新: H[k] = H[k] + μ · x_in[n-k] · e
```

其中d_hat[n]为判决器（slicer）的输出。CMA先完成粗收敛（建立初始均衡），DDLMS再进行精确跟踪（细化系数）。两者级联形成两阶段自适应均衡体系，是当前高速SerDes的标配方案。

**LMS步长参数**：步长μ控制收敛速度和稳态误差的权衡。μ越大收敛越快，但稳态误差越大；μ越小稳态误差越小，但收敛时间越长。典型值为μ = 0.01 ~ 0.1。

## 第四章 非线性均衡技术

### 4.1 DFE 判决反馈均衡器原理

DFE（Decision Feedback Equalizer）是接收端最重要的非线性均衡模块，利用已判决符号的信息来消除当前符号所受的ISI。

**DFE的结构**：
```
                    ┌─────────────────────────────────┐
输入 ──▶ 采样器 ──▶ │      ┌─────┐    ┌─────┐          │──▶ 判决器 ──▶ 输出
                  │      │ w₁  │    │ w₂  │   ...   │      ▲
                  │      └─────┘    └─────┘          │      │
                  │        ▲          ▲              │      │
                  └────────┼──────────┼───────────────┴──────┘
                           │          │                  │
                           └──────────┴──────────────────┘
                              (反馈路径，已判决符号)
```

**DFE的数学表达**：
```
r[n] = y[n] - Σ w[k] · d_hat[n-k], k = 1, 2, ..., M
```

其中y[n]为采样输入，w[k]为反馈抽头系数，d_hat[n-k]为已判决的历史符号，r[n]为均衡后的判决变量。

**DFE的核心优势**：由于反馈的是已判决的确定符号（而非线性均衡器处理含噪声的信号），DFE在消除post-ISI时**不会放大噪声**。这是DFE相对于FFE/CTLE的根本优势——FFE/CTLE在提升高频增益以补偿ISI时，会同等地放大该频段的噪声功率，而DFE利用已判决符号的确定信息来抵消ISI，不引入额外噪声。

### 4.2 DFE的错误传播

DFE的固有缺陷是**错误传播（Error Propagation）**——当某个符号被错误判决后，错误值通过DFE反馈路径影响后续符号的判决，可能引发突发错误。

**错误传播机制**：
1. 符号d[n]被错误判决（假设应为0但判决为1）
2. 错误值e = d_hat[n] - d[n] = 1通过反馈路径传到d[n+1]
3. d[n+1]的均衡结果减去w₁·e
4. 如果e足够大，d[n+1]也可能被错误判决
5. 错误持续传播，形成突发错误

**突发错误长度分布**：
- 短突发错误（1-2符号）：最常见
- 长突发错误：概率极低（BER<10^-12时）
- 总体而言，DFE的错误传播在低BER系统（如KP4 FEC后，BER<10^-15）中影响可忽略

**缓解策略**：
- 在DFE前设置足够强的FFE/CTLE，将判决错误概率降至极低水平
- 采用**软判决DFE**（soft DFE），反馈软信息而非硬判决值
- 限制反馈抽头数量M，避免过度深的反馈路径
- 采用**speculative DFE**（预测型DFE），通过并行计算消除关键路径中的判决器延迟

### 4.3 Speculative DFE 与 Loop-Unrolled DFE

**Speculative DFE（预测型DFE）**：在当前UI内，前一个符号的判决结果尚未可用时，提前按两种假设（d[n-1]=0 或 d[n-1]=1）并行计算两种ISI抵消值。当前符号判决完成后，用MUX选择正确路径。

```
         假设1 (d[n-1]=0)    假设2 (d[n-1]=1)
              │                   │
              ▼                   ▼
        计算 y[0]           计算 y[1]
              │                   │
              └────────┬──────────┘
                       ▼
                   MUX (根据d_hat[n-1]选择)
                       │
                       ▼
                   判决器
```

**Loop-Unrolled DFE**：将DFE的反馈环路完全展开，对前N个符号的所有可能组合做并行计算。N=1时就是speculative DFE；N=2时需要计算4种假设组合（对于PAM4则是16种）。硬件复杂度随N指数增长，通常只展开1-2个tap。

**224G SerDes的DFE挑战**：

| 参数 | 112G PAM4 | 224G PAM4 | 挑战 |
|------|-----------|-----------|------|
| UI | 17.86 ps | 8.93 ps | 减半 |
| DFE 1st tap关键路径 | ~30 ps | < 10 ps | 极大挑战 |
| 架构需求 | 半速率可用 | Speculative/Loop-Unrolled必需 | |

224G的UI只有8.93 ps，而DFE的采样器+反馈路径延迟在传统架构下超过30 ps，无法在一个UI内完成。因此224G必须采用speculative或loop-unrolled架构。

### 4.4 MLSE 最大似然序列估计

MLSE（Maximum Likelihood Sequence Estimation）是理论上最优的均衡方案，基于Viterbi算法在整个接收序列中搜索最可能的发送序列。

**与DFE的根本区别**：DFE是逐符号判决，每个符号独立优化；MLSE是序列估计，考虑符号间的依赖关系，在全局最优的框架下找到最可能的发送序列。

**MLSE的网格图状态**：若信道脉冲响应长度为L+1个抽头，状态数为2^(L+1)（二进制调制）。例如：

- MLSE-1（ISI影响前后各1个符号）：状态数 = 2^2 = 4
- MLSE-2（ISI影响前后各2个符号）：状态数 = 2^3 = 8

**Viterbi算法步骤**：
1. 计算所有可能路径的分支度量（branch metric）：BM = |y[k] - Σ h[i]·d[k-i]|²
2. 递归更新每状态的累计路径度量（path metric）
3. 保留每状态的最优幸存路径
4. 判决时回溯最优路径

**MLSE性能分析（电子科技大学论文）**：

| 传输距离 | MLSE-1 | MLSE-2 | MLSE-3 |
|---------|--------|--------|--------|
| 100 km | 接近最优 | 接近最优 | 接近最优 |
| 200 km | 较差 | 最优 | 最优 |
| 300 km | 失效 | 较差 | 最优 |
| 400 km | 失效 | 较差 | 良好 |

状态数越多，可补偿的ISI越长，但计算复杂度也越高（每增加一个状态数，复杂度翻倍）。

**MLSE的优化变体**：

**SMLSE（平均状态均值优化）**：将同一扩展状态的4个状态均值求平均，大幅降低计算复杂度的同时几乎保持最优性能。

**MMLSE（最大状态均值优化）**：选择最大度量的状态均值，比MLSE提升1-2 dB，但比SMLSE差。

**MLSE vs DFE 全面对比**：

| 特性 | MLSE | DFE |
|------|------|-----|
| 理论基础 | 最大似然序列估计 | 逐符号判决 |
| ISI处理 | 考虑整个序列 | 仅反馈已判决符号 |
| 噪声放大 | 无 | 无 |
| 错误传播 | 无（序列估计） | 有 |
| 计算复杂度 | 高（指数增长） | 低（线性） |
| 延迟 | 长 | 短 |
| 性能 | 最优（AWGN信道） | 次优 |
| 适用场景 | 高性能专用 | 商用主流 |

## 第五章 联合均衡架构

### 5.1 经典均衡链路：TX FFE + CTLE + DFE

当前高速SerDes的均衡链路采用三级级联架构：

```
发送端                          接收端
──────                          ──────
Data ──▶ TX FFE ──────────────────▶ Channel (损耗) ──▶ 
                                            │
                                            ▼
                                    CTLE (模拟域)
                                       (初步均衡+放大)
                                            │
                                            ▼
                                    VGA/AGC (增益控制)
                                            │
                                            ▼
                                    RX FFE (数字域)
                                       (CMA粗收敛→DDLMS精调)
                                            │
                                            ▼
                                    DFE (判决反馈)
                                       (消除post-ISI，无噪声放大)
                                            │
                                            ▼
                                    CDR (时钟恢复)
                                            │
                                            ▼
                                      判决输出
```

**各模块分工**：

| 均衡模块 | 主要功能 | 技术特征 |
|---------|---------|---------|
| TX FFE | 预失真，补偿信道频率响应 | FIR滤波，预加重/去加重 |
| CTLE | 模拟域线性均衡+放大 | 零点/极点可调，高频提升 |
| RX FFE | 数字域精细线性均衡 | CMA+DDLMS两阶段自适应 |
| DFE | 消除长尾post-ISI | 反馈滤波，无噪声放大 |

**自适应协同**：TX FFE系数、CTLE设置、RX FFE抽头、DFE抽头共同构成完整的均衡参数空间。理想情况下，这些参数应联合优化以达到全局最优。但在实际系统中，由于参数空间的维度很高（TX FFE 4维 + CTLE 3维 + RX FFE 8-16维 + DFE 4-8维），通常采用分步优化策略：

1. 先优化TX FFE（通过链路训练）
2. 再优化CTLE（通过信道特性分析）
3. 最后RX FFE和DFE自适应收敛（运行时）

### 5.2 224G SerDes的均衡架构演进

**56G/112G SerDes的均衡配置**：

| 速率 | TX EQ | RX EQ | 说明 |
|------|-------|-------|------|
| 56G NRZ/PAM4 | 3-tap FFE | CTLE + 1-tap DFE | 基本够用 |
| 112G PAM4 | 4-tap FFE | CTLE + 2-tap DFE | 当前主流 |

**224G SerDes的均衡配置**：

| 速率 | TX EQ | RX EQ | 说明 |
|------|-------|-------|------|
| 224G PAM4 | 5-tap FFE + 预编码 | CTLE + 4-tap DFE + 弱DSP | 新标准需求 |
| 224G (ADC-based) | 5-tap FFE + 预编码 | ADC + FFE + DFE + CDR | 全数字架构 |

**224G均衡的特殊挑战**：

1. **DFE 1st tap关键路径延迟**：UI只有8.93 ps，DFE的反馈必须在极短时间内完成，需要speculative或loop-unrolled架构。

2. **CTLE噪声增强更严重**：200G/lane的Nyquist频率达53 GHz，CTLE在补偿如此高频率损耗时会产生更大的噪声放大。

3. **TX FFE系数更多**：5-tap FFE（相比4-tap）增加了pre2和post3 tap，以处理更长的信道脉冲响应。

4. **ADC分辨率需求更高**：PAM4的三个眼图垂直间距更小，要求ADC ENOB达到6.5-7 bit以上。

### 5.3 PAM4系统的均衡特殊性

PAM4（4电平脉冲幅度调制）相比NRZ对均衡技术有特殊要求：

**多电平判决**：PAM4有3个眼图（4个电平），需要3组判决阈值。每个判决电平的精确设置影响系统BER性能。

**眼图密度**：PAM4的三个眼图中，中间眼的垂直开口最小，受到上下两个方向的ISI影响，噪声累积最严重。

**FFE抽头系数的PAM4适配**：PAM4的FFE系数矩阵需要考虑相邻符号的多个电平组合：

| 系数 | 符号 | NRZ FFE | PAM4 FFE |
|------|------|---------|----------|
| c[-1] | pre-cursor | 负值（减小前铃） | 需适配多电平 |
| c[0] | main-cursor | 正值 | 正值（保持幅度） |
| c[1] | post-cursor | 负值（减小后铃） | 需适配多电平 |

**PAM4的SNR代价**：PAM4相邻电平间距是NRZ的1/3（假设同样峰值功率约束），导致约9.54 dB的SNR代价（20·log10(3)）。这意味着PAM4对均衡器的性能要求更高——同样的信道，均NRZ能正常工作的系统，PAM4可能需要更强的均衡才能达到相同BER。

**预编码（Precoder）的角色**：PAM4系统中常用预编码技术将信道ISI转化为已知的、可通过接收端处理消除的确定性干扰模式，避免DFE的错误传播问题。

## 第七章 色散补偿与先进均衡技术

### 7.1 光纤色散与均衡

光纤中的色散（Dispersion）主要包括：

**色度色散（Chromatic Dispersion, CD）**：不同波长光信号的传输速度不同，导致脉冲展宽。色散系数D（ps/nm/km）在1310nm处约为0 ps/nm/km，在1550nm处约为17 ps/nm/km。

**偏振模色散（Polarization Mode Dispersion, PMD/DGD）**：光纤中两个偏振态的传输速度差异，导致脉冲分裂。

**色散对IM/DD系统的影响**：

在强度调制直接检测（IM/DD）系统中，色散与平方律检测相互作用，产生**功率衰落（Power Fading）**：

```
P(dB) = 10·log10(|sin(φ)/φ|²), φ = D·f·λ·π
```

其中f为信号频率，λ为光波长。当φ接近π的整数倍时，出现深度衰落。

**数字域色散补偿**：

频域色散补偿滤波器：
```
Y(f) = X(f) · exp(j·β₂·L·f²/2)
```

其中β₂为群延迟色散参数，L为光纤长度。通过FFT将信号转换到频域，应用上述滤波器，再IFFT转回时域。

### 7.2 Volterra非线性均衡

传统的线性均衡器（FFE/DFE）假设信道是线性时不变系统，但实际光通信系统存在多种非线性效应，需要非线性均衡器建模。

**Volterra级数**：是线性FIR滤波器的非线性推广，能建模任意有限记忆的非线性系统：

```
y[n] = Σ h₁[k₁]·x[n-k₁] 
     + Σ h₂[k₁,k₂]·x[n-k₁]·x[n-k₂]
     + Σ h₃[k₁,k₂,k₃]·x[n-k₁]·x[n-k₂]·x[n-k₃]
     + ...
```

- h₁：一阶核（线性均衡，等效于FFE）
- h₂：二阶核（建模二次非线性，如直流偏置波动、谐波失真）
- h₃：三阶核（建模三次非线性，如交叉相位调制、XPM）

**实际应用**：在直接调制激光器（DML）系统中，Volterra均衡器用于补偿激光啁啾与光纤色散的相互作用。

### 7.3 神经网络均衡器

近年来，深度神经网络（DNN）和循环神经网络（RNN/LSTM）在光通信均衡中的应用研究日益活跃。

**DNN均衡器原理**：

```
输入层: ADC采样序列 (N个样本)
  ↓
隐藏层1: 全连接 + ReLU激活
  ↓
隐藏层2: 全连接 + ReLU激活
  ↓
...
  ↓
输出层: 判决输出 (符号预测)
```

**DNN均衡器的优势**：

1. **自动学习复杂非线性**：神经网络能自动学习信道中的任意非线性特性，无需人工设计均衡器结构
2. **端到端优化**：整个均衡链路可以端到端训练
3. **迁移学习能力**：在一个信道上训练的模型可以通过迁移学习快速适配到相似信道

**RNN/LSTM均衡器**：RNN能建模时序依赖关系，LSTM在此基础上增加了记忆单元，更好地处理长距离依赖。适用于光纤中的PMD、色散等长记忆信道。

**实验结果**（基于OFC论文）：

- 基于DNN的均衡器在长距离相干光通信系统中可获得1-2 dB的额外增益
- 在PAM4直接检测系统中，DNN均衡器可替代或增强传统的FFE/DFE结构

### 7.4 数字孪生辅助均衡

数字孪生（Digital Twin）通过高精度的数字建模，在离线状态下完成最优均衡器系数的搜索，再将结果部署到实际系统。

**工作流程**：

1. **信道建模**：对真实信道进行高精度电磁建模（ADS/HFSS/Sigrity）
2. **离线优化**：在仿真环境中运行强化学习或贝叶斯优化，寻找最优均衡参数
3. **部署验证**：将优化后的参数部署到实际系统
4. **在线微调**：运行时进行少量在线调整

**优势**：减少在线训练时间，支持更复杂的均衡算法，缩短产品开发周期。

## 第八章 FEC与均衡的协同

### 8.1 均衡与FEC的关系

均衡和FEC（Forward Error Correction）是相互依存的：

- **均衡**在物理层消除/减轻符号间干扰，降低判决错误概率
- **FEC**在链路层纠正剩余错误，提供额外的编码增益

两者的协同设计影响系统整体性能。

### 8.2 各速率等级的FEC方案

| 速率 | 标准 | FEC方案 | 编码增益 | FEC前BER阈值 |
|------|------|---------|---------|-------------|
| 25G NRZ | 802.3by | KR4 (RS(528,514)) | ~5.6 dB | ~1e-5 |
| 50G/100G | 802.3cd | KP4 (RS(544,514)) | ~7.0 dB | ~2.4e-4 |
| 200G/400G | 802.3ck/df | KP4 (RS(544,514)) | ~7.0 dB | ~2.4e-4 |
| 800G/1.6T | 802.3dj | KP4 + Outer FEC | ~9-11 dB | ~1e-2 (Outer) |

### 8.3 两级FEC架构

802.3dj引入了两级FEC架构（Inner FEC + Outer FEC）以满足长距离传输需求：

**Inner FEC**：KP4 (RS(544,514))，位于PCS子层，延迟~100 ns，处理粒度为每lane

**Outer FEC**：

| 外码类型 | 编码增益 | 延迟 | 应用 |
|---------|---------|------|------|
| Staircase Code | ~10.5 dB | ~2-5 μs | 相干光通信 |
| OFEC (Braided+BCH) | ~11.1 dB | ~4 μs | 超长距 |
| KP4+Hamming | ~9.56 dB | ~60 ns | 高增益+低延迟 |

**延迟预算**：AI网络对延迟极为敏感，总FEC延迟需控制在10 μs以内，这对两级FEC架构的设计提出了挑战。

## 第九章 均衡器设计挑战

### 9.1 高速实现的核心挑战

**时序路径挑战**：DFE的反馈必须在采样瞬间完成。对于224G PAM4，UI只有8.93 ps，但采样器+DFE反馈路径延迟在传统架构下超过30 ps。

**解决方案**：
- Half-Rate / Quarter-Rate 架构（4×并行）
- Speculative DFE（预测型，提前计算）
- Loop-Unrolled DFE（完全展开前N个tap）

**噪声放大**：CTLE在提升高频增益时也会放大高频噪声。对于224G，均衡深度更大，噪声增强更严重。

**PVT变化**：工艺（Process）、电压（Voltage）、温度（Temperature）变化影响均衡器性能。自适应校准是必需的。

**功耗限制**：高速SerDes的功耗预算紧张（< 5 pJ/bit），均衡器（特别是ADC-based RX）功耗占比大。

### 9.2 PAM4均衡的特殊挑战

1. **多电平判决精度**：3个判决电平需要精确设置，判决阈值偏移会直接降低BER
2. **中间眼最脆弱**：中间眼同时受上下两个方向的ISI影响
3. **FFE抽头更多**：PAM4的FFE系数矩阵需要考虑多电平组合
4. **ADC分辨率要求更高**：ENOB需达6.5-7 bit以上

### 9.3 ADC-based RX的均衡架构

```
     │
     ▼
ADC ──▶ FFE ──▶ DFE ──▶ CDR
     │        │       │
     ▼        ▼       ▼
   数字域    数字域   数字域
```

**优势**：
- 灵活性高，可实现复杂算法（MLSE、神经网络）
- 可编程，可适应不同信道
- 便于工艺迁移和PVT补偿

**劣势**：
- 功耗高（高速ADC是最大的功耗源）
- 精度受限
- 量化噪声影响

**功耗分配（典型224G ADC-based RX）**：

| 模块 | 功耗 |
|------|------|
| TX Driver + FFE | ~150-200 mW |
| RX CTLE | ~50-80 mW |
| RX DFE | ~150-250 mW |
| CDR (PLL+PI) | ~80-150 mW |
| ADC | ~500-1000 mW |
| **总计** | **~1000 mW** |

## 第十章 均衡技术总结与展望

### 10.1 均衡技术全景图

**钧衡技术体系总结**：

```
均衡策略选择指南：

信道条件轻 (损耗<15dB)：
  └── TX FFE (3-tap) + 轻量CTLE + 1-tap DFE

信道条件中 (损耗15-25dB)：
  └── TX FFE (4-tap) + CTLE + 2-tap DFE

信道条件重 (损耗25-35dB, NRZ/PAM4)：
  └── TX FFE (4-tap) + CTLE + 4-tap DFE + KP4 FEC

信道条件极端 (损耗>35dB, PAM4)：
  └── TX FFE (5-tap) + CTLE + Speculative DFE + 两级FEC
  或 ADC-based RX + 神经网络均衡

相干传输 (LR/ER/ZR)：
  └── oDSP: 色散补偿 + 偏振解复用 + 载波恢复 + LDPC FEC
```

### 10.2 核心技术趋势

1. **224G SerDes**：200G/lane PAM4是下一代主流，需要speculative/loop-unrolled DFE、7-8 bit PI、低功耗ADC
2. **两级FEC**：Inner KP4 + Outer FEC (Staircase/OFEC) 是长距离1.6T的标配
3. **神经网络均衡**：端到端可训练的DNN/RNN均衡器提供1-2 dB额外增益
4. **数字孪生**：离线优化+在线微调加速均衡器设计
5. **协同设计**：CDR+均衡联合优化成为224G系统的必然选择

### 10.3 设计原则

1. **TX FFE作为第一道防线**：在发送端预补偿信道损耗，减少接收端均衡压力
2. **CTLE提供初步均衡**：在模拟域完成高频补偿，降低ADC分辨率需求
3. **DFE精细调节**：消除长尾post-ISI，不放大噪声
4. **MLSE作为最后手段**：在极端ISI条件下使用，付出复杂度代价换取最优性能
5. **自适应跟踪慢变化**：CMA粗收敛 + DDLMS精调 + 定期链路训练
