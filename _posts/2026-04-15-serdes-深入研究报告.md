## 一、SerDes 系统架构

### 1.1 什么是 SerDes

SerDes（Serializer/Deserializer）是高速串行数据传输的核心接口电路。其基本架构如下：

```
发射端 (Tx)                         接收端 (Rx)
┌─────────┐    串行高速链路    ┌─────────────┐
│ Parallel │ ──────────────▶ │  CDR         │
│   Data   │    n × Gb/s     │  采样/判决   │
│    D     │                  │  Deserialize│
└─────────┘                  └─────────────┘
    ▲                                │
    │                                ▼
┌─────────┐                    ┌─────────┐
│  Clock  │                    │ Parallel│
│   PLL   │                    │   Data  │
└─────────┘                    └─────────┘
```

### 1.2 并行vs串行

| 对比项 | 并行通信 | 串行通信 (SerDes) |
|--------|---------|-------------------|
| 引脚数量 | 多（每bit一条线） | 少（差分1-4对） |
| 同步难度 | 高（多路延时不同） | 低（单路延时可控） |
| 封装依赖 | 高 | 低 |
| 速率 | 数百 Mb/s | 数 Gb/s ~ 数百 Gb/s |
| 典型应用 | 旧式计算机总线 | 现代高速接口 |

### 1.3 高速串行接口标准

| 协议 | 应用领域 | 速率 |
|------|---------|------|
| XFI (10G Ethernet) | Ethernet/Storage | 10.5188 Gbps |
| 10Gbps BASE-KR | Ethernet | 10.3125 Gbps |
| CEI-6G-LR | Ethernet背板 | 6.25 Gbps |
| SATA | 存储 | 6.0/3.0/1.5 Gbps |
| PCI-E | 计算机I/O | 5.0/2.5 Gbps |
| Fibre Channel | 存储 | 8.4/4.25 Gbps |

## 二、时钟恢复电路 (CDR)

### 2.1 CDR 基本原理

CDR（Clock and Data Recovery）从数据流中同时恢复时钟信号和采样位置。关键指标：

- **抖动 (Jitter)**: 恢复时钟的时序噪声
  - RMS抖动：750fs（张震论文，东南大学，25Gb/s CDR）
  - 峰峰抖动：3.46ps
- **眼图张开度**: 决定误码率
- **抖动容限 (Jitter Tolerance)**: CDR能容忍的输入抖动

### 2.2 CDR 架构分类

#### 架构1: PLL-Based CDR

```
数据 ──▶ 鉴相器(PD) ──▶ 电荷泵 ──▶ 环路滤波器 ──▶ VCO ──▶ 采样时钟
   ▲                                                            │
   └────────────────────────────────────────────────────────────┘
```

**代表论文**: 曹启富（电子科技大学）- 40nm CMOS，16Gb/s

#### 架构2: Dual-Loop CDR（双环CDR）

```
全局PLL ──▶ 本地相位插值器(PI) ──▶ 恢复时钟
    ▲
    │
鉴相器(PD) ◀── 本地DLL/PI ◀── 数据
```

- 全局PLL提供高频时钟
- 本地PI进行相位精调
- Xilinx 0.5-32Gb/s transceiver采用此架构（16nm FinFET）

#### 架构3: PS/PI型CDR（曹启富论文核心）

**Phase Selector / Phase Interpolator**

```
十二相时钟VCO ──▶ PS(相位选择器) ──▶ PI(相位插值器) ──▶ 恢复时钟
```

**设计特点**:
- **12相环形振荡器**：比传统8相时钟插值线性度更高
- 反相器结构：无静态功耗，比LC-VCO面积小，比差分环形振荡器省功耗
- 频率比传统单反相器环形振荡器高50%
- 1:2 + 1:5分频结构：降低工作速率，节省解串器D触发器数量

### 2.3 鉴相器类型

#### Hogge 鉴相器（线性）

- 提供相位误差的**大小和符号**
- 需要数据边沿转换才能工作
- 脉冲宽度与相位差成正比

```
Late信号 = Tb/2 - 相位差
```

#### Alexander / Bang-Bang 鉴相器（二进制，最常用）

- 只提供相位误差的**符号**（Early/Late）
- 增益在零误差点理论上无限大
- 对噪声敏感，但实现简单

```
逻辑:
- 如果边沿采样与第二bit相同 → Late（时钟晚）
- 如果边沿采样与第一bit相同 → Early（时钟早）
```

**代表论文**: 张震（东南大学）- Bang-bang PLL，25Gb/s

#### Mueller-Muller 鉴相器（波特率）

- 每个符号只采样一次
- 通过比较前、后脉冲幅度推断ISI
- 适合高速、低功耗

### 2.4 CDR 抖动特性

CDR是一个窄带跟踪系统：

- **抖动追踪带宽**：决定能跟踪的输入抖动频率
- **超出带宽的抖动**：传递到输出（增加抖动）
- **带宽内抖动**：被CDR跟踪（不传递）

```
抖动传递特性:
                    ┌──────────────
    增益            │    带宽外
     │   1          │   （直通）
     │  ───────     └──────────────
     │       \          带宽内
     │        \         （跟踪）
     └────────────────────────────
          频率 (log)
```

**关键结论**: CDR带宽越高，能跟踪的高频抖动越多，但参考时钟噪声敏感度上升。

## 三、PLL 核心电路

### 3.1 PLL 基本组成

```
参考时钟 ──▶ PFD ──▶ 电荷泵 ──▶ 环路滤波器 ──▶ VCO ──▶ 输出时钟
    ▲                                                              │
    └──────────────────────────────────────────────────────────────┘
```

### 3.2 VCO 类型对比

| 特性 | Ring VCO | LC-VCO |
|------|----------|--------|
| 频率范围 | 宽（调频范围大） | 窄 |
| 版图面积 | 小（无电感） | 大（需电感） |
| 功耗 | 低（反相器无静态功耗） | 高 |
| 相位噪声 | 较差 | 较好 |
| 调谐范围 | ~50% | ~20% |
| 典型应用 | CDR本地环、SerDes | 高性能PLL主环 |

**曹启富论文**: 采用**反相器型环形振荡器**实现12相时钟
- 比LC-VCO面积小、制造成本低
- 比差分环形振荡器无静态功耗
- 频率受灌入电流影响大，调频范围宽

### 3.3 时钟分配架构

| 架构类型 | 特点 | 同步方式 |
|---------|------|---------|
| **Common Clock** | 同一时钟分布到所有芯片 | 完全同步 |
| **Forward Clock** | 时钟从TX前向发送到RX | 源同步 |
| **Embedded Clock** | 从数据流中恢复时钟（CDR） | 异步/近同步 |

**Embedded Clock（嵌入式时钟）**:
- 从数据流中提取频率和相位
- 相位检测持续运行
- 抖动跟踪受CDR带宽限制
- 技术进步使CDR带宽提高，优势减弱

## 四、光接收机前端

### 4.1 典型光接收机结构

```
光纤 ──▶ 光检测器 ──▶ TIA ──▶ VGA ──▶ CTLE ──▶ CDR ──▶ 解串
           (PIN/APD)   (跨阻   (可变   (连续时间
                      放大器)  增益)    线性均衡)
```

### 4.2 TIA（跨阻放大器）设计要点

**张震论文（东南大学，SiGe 0.13μm）**:
- 带宽: 31 GHz
- 跨阻增益: 71 dBΩ
- 输入参考噪声: 14.54 pA/√Hz
- 裸片面积: 0.9 × 0.6 mm²

### 4.3 CTLE（连续时间线性均衡器）

**功能**: 补偿信道高频衰减

**张震论文实测结果**:
- 均衡后眼宽: 0.6 UI
- 误码率 < 10⁻³ 时，光灵敏度: -30 dBm
- 10-Gb/s 连续时间线性自适应均衡器
- 电源噪声抑制: 4-MHz带宽内提升 >30 dB

**自适应方法**:
- 高/低通滤波频谱平衡
- 功率检测与误差比较
- 带隙基准 + LDO 电源管理

## 六、LPO (Linear Pluggable Optics) 与系统功耗

### 6.1 AI/ML集群的光互连挑战

| 挑战 | 描述 |
|------|------|
| 带宽需求 | 单GPU 4Tb/s，8-GPU服务器 28.8Tb/s |
| 集群规模 | 1000+ GPUs，跨100米 |
| 功耗限制 | PCIe卡限制模块功耗 ~10W |
| 目标 | 800G接口: <4W(多模), <8W(单模) |

### 6.2 Linear Drive 技术（Nvidia, IPEC）

**传统 DSP-Based 架构**:
```
ASIC ──▶ CDR ──▶ oDSP ──▶ O/E ──▶ 光学输出
         (功耗高，~40W per 100G)
```

**Linear Drive 架构**:
```
ASIC ──▶ 线性驱动器 ──▶ O/E ──▶ 光学输出
         (功耗低，<4W per 100G)
```

**核心优势**:
1. 降低功耗（4W vs 40W per 100G）
2. 降低延迟（无CDR/oDSP处理）
3. 降低成本

## 七、SerDes 关键技术指标总结

### 7.1 性能指标

| 指标 | 说明 | 典型值 |
|------|------|--------|
| 数据率 | 每通道每秒比特数 | 25-112 Gb/s |
| 功耗效率 | 每比特消耗能量 | 1-20 pJ/b |
| RMS抖动 | 时钟抖动（短周期） | 200fs-1ps |
| 抖动容限 | 能跟踪的最大抖动 | 0.3-0.5 UI |
| 眼图张开度 | 判决裕度 | >0.3 UI |

### 7.2 关键电路模块

| 模块 | 功能 | 技术挑战 |
|------|------|---------|
| CDR | 恢复时钟和数据 | 高速鉴相、低抖动VCO |
| VCO | 产生高频时钟 | 低相位噪声、宽调谐范围 |
| CTLE | 信道均衡 | 自适应、高带宽 |
| DFE | 判决反馈均衡 | 高速、高精度 |
| TIA | 光信号转电流 | 低噪声、高带宽 |

- CDR架构（PLL型、双环型、PS/PI型）

3. **ADC-Based CDR**: 数字化的CDR架构
4. **MLSE/DFE均衡器**: 在高速CDR中的集成
5. **SerDes 测试方法**: 眼图、抖动测量、BERT
