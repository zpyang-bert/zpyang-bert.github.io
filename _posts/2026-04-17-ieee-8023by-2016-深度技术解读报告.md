## 25 Gigabit Ethernet (25GbE) 标准详解

**文档信息**
- 标准: IEEE Std 802.3by™-2016
- 标题: Media Access Control Parameters, Physical Layers, and Management Parameters for 25 Gb/s Operation
- 批准日期: 2016年6月30日
- 发布日期: 2016年7月29日

---

## 一、概述

### 1.1 标准定位

IEEE 802.3by-2016 是对 IEEE Std 802.3-2015 的第二次修正案（Amendment 2），由 IEEE LAN/MAN Standards Committee 开发，专注于定义 25 Gb/s 速率的以太网物理层规范。该标准填补了 10GbE 和 40GbE/100GbE 之间的性能空白，为数据中心提供了一种经济高效的 25Gbps 速率解决方案。

### 1.2 主要新增内容

本修正案新增了以下规范：
- **Clause 105**: 25GbE 总体架构和术语定义
- **Clause 106**: 25GMII (25 Gigabit Media Independent Interface)
- **Clause 107**: 25GBASE-R PCS (64B/66B 编码)
- **Clause 108**: RS-FEC (Reed-Solomon 前向纠错) 子层
- **Clause 109**: 25GBASE-R PMA (Physical Medium Attachment)
- **Clause 110**: 25GBASE-CR/CR-S (铜缆双绞线)
- **Clause 111**: 25GBASE-KR/KR-S (背板)
- **Clause 112**: 25GBASE-SR (多模光纤)
- **Annex 109A/109B/109C**: 25GAUI 接口规范
- **Annex 110A/110B/110C**: 测试夹具和连接器规范

## 二、物理层类型

### 2.1 PHY 类型总览

| 类型 | 介质 | FEC | 典型应用 |
|------|------|-----|----------|
| 25GBASE-CR | 双绞铜缆 (SFP28/QSFP28) | 可选 | 交换机互联 < 5m |
| 25GBASE-CR-S | 双绞铜缆 (无RS-FEC) | 无 | 短距离铜缆 |
| 25GBASE-KR | 电气背板 | 可选 | 线路卡到背板 |
| 25GBASE-KR-S | 电气背板 (无RS-FEC) | 无 | 背板直连 |
| 25GBASE-SR | 多模光纤 (OM3/OM4) | 必需 | 数据中心 < 100m |

### 2.2 25GBASE-CR/CR-S (铜缆)

**定义**: 25Gb/s 速率在单通道双绞铜缆上的物理层规范

**关键参数**:
- 传输速率: 25.78125 GBd (25.78125 Gbaud)
- 编码: 64B/66B (25GBASE-R)
- 通道数: 1 lane
- 传输距离: 长达 5 米 (取决于线缆质量)
- 连接器: SFP28 或 QSFP28

**FEC 模式**:
- no-FEC 模式: 25GBASE-CR-S 等效
- BASE-R FEC 模式: 802.3 Clause 74 定义
- RS-FEC 模式: Clause 108 定义 (RS(528,514))

**电缆组件特性**:
- 特性阻抗: 100Ω ± 5%
- 差分插入损耗: 符合 Channel Operating Margin (COM) 要求
- 回波损耗: 符合规范要求
- COM 要求: ≥ 3 dB

### 2.3 25GBASE-KR/KR-S (背板)

**定义**: 25Gb/s 速率在电气背板上的物理层规范

**关键参数**:
- 传输速率: 25.78125 GBd
- 编码: 64B/66B
- 通道数: 1 lane
- 背板走线长度: 长达 1 米

**与 CR 类型的区别**:
- KR 类型针对背板走线优化
- CR 类型使用 SFP28/QSFP28 铜缆模块
- KR-S 和 CR-S 表示不支持 RS-FEC 的版本

### 2.4 25GBASE-SR (多模光纤)

**定义**: 25Gb/s 速率在多模光纤上的物理层规范

**关键参数**:
- 传输速率: 25.78125 GBd
- 波长: 850 nm (VCSEL)
- 光纤类型: OM3 (2000 MHz·km) 或 OM4 (4700 MHz·km)
- 最大传输距离: 100m (OM3/OM4)
- 光纤数量: 2 根 (双向)
- RS-FEC: 必需 (不可旁路)

**BER 要求**:
- BER < 5×10⁻⁵
- 帧丢失率 < 6.2×10⁻¹⁰ (64字节帧，最小帧间隔)

**光学特性**:
- 平均发射功率: -8.4 dBm (最大值)
- 接收灵敏度: -11.1 dBm (OMA)
- 消光比: ≥ 3 dB

## 三、子层架构

### 3.1 25GbE 协议栈

```
MAC (Media Access Control)
    |
    v
RS (Reconciliation Sublayer)
    |
    v
25GMII (25 Gigabit Media Independent Interface) [可选]
    |
    v
PCS (Physical Coding Sublayer) - Clause 107
    | [64B/66B 编码]
    v
RS-FEC - Clause 108 [可选, SR类型必需]
    | [RS(528,514) 纠错]
    v
PMA (Physical Medium Attachment) - Clause 109
    | [信号驱动/接收]
    v
25GAUI [可选] - Annex 109A/109B
    | [芯片到芯片/芯片到模块接口]
    v
PMD (Physical Medium Dependent)
    - Clause 110 (CR/CR-S)
    - Clause 111 (KR/KR-S)  
    - Clause 112 (SR)
    |
    v
MDI (Medium Dependent Interface)
    |
    v
介质 (铜缆/光纤/背板)
```

### 3.2 25GMII (Clause 106)

**定义**: Reconciliation Sublayer (RS) 与 Physical Coding Sublayer (PCS) 之间用于 25 Gb/s 操作的接口

**信号定义**:
- 25GMII 包含 25 位数据总线
- 25GMII_TXCLK: 发送时钟 (25.78125 MHz)
- 25GMII_RXCLK: 接收时钟
- TXD[31:0]: 发送数据 (使用低32位)
- RXD[31:0]: 接收数据
- TX_EN, TX_ER: 发送使能/错误
- RX_DV, RX_ER: 接收数据有效/错误
- CRS, COL: 载波侦听/冲突检测

### 3.3 PCS (Clause 107) - 25GBASE-R

**功能**: 64B/66B 编码的物理编码子层

**关键特性**:
- 使用 64B/66B 块编码 (与 10GBASE-R 相同机制)
- 扰码器: 使用状态机同步扰码
- LPI (Low Power Idle) 支持: 用于 Energy Efficient Ethernet
- 测试模式: PRBS9, PRBS31 等

**块类型**:
- Sync Header: 2 bits (01 = data, 10 = control)
- Block Type: 8 bits (控制块识别)
- Data Block: 64 bits payload
- Control Block: 64 bits payload + 控制信息

## 四、Reed-Solomon 前向纠错 (RS-FEC)

### 4.1 RS-FEC 概述 (Clause 108)

**代码规格**:
- 编码: RS(528, 514)
- Galois Field: GF(2¹⁰) - 10位符号
- 码字长度: 528 符号 (5280 bits)
- 信息长度: 514 符号 (5140 bits)
- 校验长度: 14 符号 (140 bits)
- 纠错能力: 每码字最多 7 个符号错误

### 4.2 RS-FEC 架构

**发送方向处理流程**:
1. **块同步 (Block Synchronization)**: 识别 64B/66B 块的边界
2. **速率补偿**: 处理码字标记的速率补偿
3. **64B/66B → 256B/257B 转码**: 20 个 66 位块转换为 257 位块
4. **码字标记插入**: 标记特定码字
5. **RS 编码**: 添加 14 个校验符号
6. **码字串行化**: 输出到 PMA

**接收方向处理流程**:
1. **码字标记锁定**: 恢复码字边界
2. **RS 解码**: 提取信息符号，纠错
3. **码字监控**: 检测未纠错码字
4. **码字标记移除**: 删除码字标记
5. **256B/257B → 64B/66B 转码**: 恢复原始块格式
6. **速率补偿**: 补偿删除的码字标记

### 4.3 纠错机制

**错误检测与纠正**:
- RS 解码器可纠正最多 7 个符号错误
- 无法纠正的错误会通过破坏同步头通知 PCS
- 高 SER 监控: 连续 8192 码字内超过 417 个符号错误时触发

**错误指示机制**:
- 当检测到不可纠正的码字时:
  - 每隔一个 257 位块的第一个 66 位块的同步头被设为 11
  - 这会导致 PCS 丢弃受影响的帧

**可选功能**:
- FEC_bypass_correction_enable: 旁路纠错 (仅错误检测)
- FEC_bypass_indication_enable: 旁路错误指示
- 注意: 25GBASE-SR 不支持旁路纠错

### 4.4 EEE Deep Sleep 集成

**快速码字锁定**:
- 支持 LPI (Low Power Idle) 模式
- 从 ALERT 模式转换到 DATA 模式时:
  - 设置 tx_down_count = 40
  - 码字标记插入功能开始插入特殊标记
  - 接收端可快速同步到新的码字边界

## 五、管理接口

### 5.1 MDIO 寄存器定义

**新增 PMA/PMD 寄存器**:

| 寄存器 | 名称 | 功能 |
|--------|------|------|
| 1.0 | PMA/PMD Control 1 | 复位、速度选择 |
| 1.1 | PMA/PMD Status 1 | 故障状态 |
| 1.4 | PMA/PMD Speed Ability | 25G 能力指示 |
| 1.7 | PMA/PMD Control 2 | PMA/PMD 类型选择 |
| 1.8 | PMA/PMD Status 2 | 发送/接收故障 |
| 1.9 | PMD Transmit Disable | 发送禁用 |
| 1.11 | PMA/PMD Extended Ability | 扩展能力 |
| 1.19 | 25G PMA/PMD Extended Ability | 25G 专用能力 |
| 1.150 | BASE-R PMD Control | BASE-R FEC 控制 |
| 1.151 | BASE-R PMD Status | BASE-R FEC 状态 |
| 1.200 | RS-FEC Control | RS-FEC 使能 |
| 1.201 | RS-FEC Status | RS-FEC 状态 |
| 1.202-1.203 | RS-FEC Corrected CW Counter | 纠错码字计数 |
| 1.204-1.205 | RS-FEC Uncorrected CW Counter | 未纠错码字计数 |
| 1.210-1.211 | RS-FEC Symbol Error Counter Lane 0 | 符号错误计数 |

**PMA/PMD Type Selection (1.7.5:0)**:
```
1 1 1 0 1 0 = 25GBASE-SR PMA/PMD
1 1 1 0 0 1 = 25GBASE-KR 或 25GBASE-KR-S PMA/PMD
1 1 1 0 0 0 = 25GBASE-CR 或 25GBASE-CR-S PMA/PMD
```

### 5.2 MAU 类型定义

**新增 MAU 类型**:
- 25GBASE-R: PCS + PMA (无特定 PMD)
- 25GBASE-CR: 双绞铜缆
- 25GBASE-CR-S: 双绞铜缆 (无 RS-FEC)
- 25GBASE-KR: 背板
- 25GBASE-KR-S: 背板 (无 RS-FEC)
- 25GBASE-SR: 多模光纤

## 六、Auto-Negotiation

### 6.1 25GbE AN 支持

**AN 能力**:
- Clause 73 定义了背板和铜缆组件的 Auto-Negotiation
- 25GBASE-CR/KR 类型支持 AN
- 25GBASE-SR 不支持 AN

**AN 功能**:
- 速率协商: 1G/10G/25G
- FEC 协商: RS-FEC / BASE-R FEC / No-FEC
- 远端故障指示
- 主/从关系建立

**链路配置流程**:
1. 上电后开始 AN 过程
2. 交换能力寄存器
3. 选择最高共同速率
4. 协商 FEC 模式
5. 建立链路

---

## 七、25GAUI 接口

### 7.1 25GAUI 概述

**定义**: 25 Gigabit Attachment Unit Interface

**用途**:
- 芯片到芯片 (C2C) 互联
- 芯片到模块 (C2M) 互联

**特性**:
- 单通道 25.78125 GBd
- 可选的 EEE 支持
- 无指定机械连接器

### 7.2 25GAUI C2C (Annex 109A)

**芯片到芯片应用**:
- 主机芯片到模块芯片的直接连接
- 短距离、高密度互联

### 7.3 25GAUI C2M (Annex 109B)

**芯片到模块应用**:
- 主机到 SFP28/QSFP28 模块
- 传输距离可达数米

---

## 八、Energy Efficient Ethernet (EEE)

### 8.1 EEE 支持

**可选 EEE**:
- 25GBASE-CR: 支持 deep sleep 和 fast wake
- 25GBASE-KR: 支持 deep sleep 和 fast wake
- 25GBASE-SR: 仅支持 fast wake (不支持 deep sleep)

### 8.2 LPI 机制

**Low Power Idle (LPI)**:
- 链路空闲时进入低功耗模式
- 定期发送 Refresh 信号保持链路同步
- 快速 wake 转换到正常工作状态

**功耗节省**:
- 有效降低空闲链路功耗
- 适合数据中心等间歇性流量场景

## 十、延迟约束

### 10.1 延迟预算

**系统延迟要求**:
- MAC 到 MAC 延迟需满足 PAUSE 操作要求
- 25GBASE-SR PMD+PMA 延迟: ≤ 512 bit times (20.48 ns)

### 10.2 帧丢失率

**性能目标**:
- 帧丢失率 < 6.2×10⁻¹⁰ (64字节帧)
- 基于 RS-FEC 处理后的统计

## 十一、应用场景

### 11.1 数据中心

**典型应用**:
- 服务器接入: 25GbE 网卡到 ToR 交换机
- 交换机互联: 25GbE 端口连接
- 存储网络: NVMe over Fabrics

**优势**:
- 相比 10GbE 提供 2.5 倍带宽
- 相比 40GbE 更低成本/端口
- 支持 SFP28/QSFP28 生态

### 11.2 背板应用

**25GBASE-KR**:
- 线路卡到背板连接
- 高密度系统设计
- 支持 FEC 纠错

### 11.3 铜缆互联

**25GBASE-CR**:
- 交换机间短距离互联
- 最高 5 米传输距离
- 成本优化的解决方案

## 十二、与既有标准的关系

### 12.1 IEEE 802.3 系列演进

```
10GbE ──► 40GbE/100GbE ──► 25GbE
           (2010)          (2016)

10BASE-T  → 100BASE-TX → 1000BASE-T → 10GBASE-T
10Mb/s    → 100Mb/s    → 1Gb/s      → 10Gb/s

10GBASE-R → 25GBASE-R → 40GBASE-R/100GBASE-R
10Gb/s    → 25Gb/s    → 40Gb/s/100Gb/s
```

### 12.2 向后兼容性

- 25GbE 不与 10GbE 直接兼容
- 需要独立的物理层
- AN 机制支持多速率协商
