## 1. 224G 核心物理挑战

### 1.1 112G vs 224G 对比

| 参数 | 112G PAM4 | 224G PAM4 | 变化 |
|------|-----------|-----------|------|
| 波特率 | 56 GBaud | **112 GBaud** | 2× |
| Nyquist 频率 | 28 GHz | **56 GHz** | 2× |
| 典型信道损耗 | 25-30 dB | **35-40 dB** | +10 dB |
| **UI (Unit Interval)** | 17.86 ps | **8.93 ps** | **减半** |
| 抖动预算 (TJ) | ~0.3 UI (~5.4 ps) | **~0.25 UI (~2.2 ps)** | 更紧 |
| ADC 采样率 | 56-64 GSa/s | **112-128 GSa/s** | 2× |
| ADC 有效位数 (ENOB) | 5-6 bit | **6-7 bit** | +1 bit |
| CDR 带宽需求 | 10-30 MHz | **30-50 MHz** | 更高 |

### 1.2 关键结论

```
224G 核心挑战:
1. UI 缩短到 8.93 ps
   → CDR 相位分辨率必须进入 sub-100 fs 级别
   
2. 信道损耗 35-40 dB @ 56 GHz
   → 单纯 TX FFE 已无法补偿
   → 必须在 RX 侧引入更强均衡
   
3. ADC 速率翻倍
   → 功耗按 f_s × 2^ENOB 增长
   → ADC 功耗将成为 RX 侧最大功耗源
```

## 2. 为什么必须协同设计

### 2.1 传统设计的失效

在 28G/56G SerDes 中，CDR 和均衡器通常可以**分阶段设计**：
- 先设计 CDR，保证抖动容限和锁定范围
- 再设计均衡器，保证眼图张开度
- 最后拼接做系统仿真

### 2.2 224G 中的四大耦合问题

**（1）CDR 带宽与均衡收敛相互耦合**

```
CDR 带宽 ←→ 均衡器频谱特性

• CDR 带宽决定跟踪输入抖动的频率范围
• 均衡器（CTLE + DFE）改变信号频谱特性
• CDR 带宽过高 → 跟踪均衡器引入的高频噪声
• CDR 带宽过低 → 无法跟踪低频 wander
```

**（2）均衡器放大噪声，影响 CDR 边沿检测**

```
CTLE 噪声增强:
• CTLE 补偿高频损耗时同时放大噪声
• 噪声增大 → 数据边沿抖动增加
• Bang-Bang PD 产生更多错误 Early/Late 判决
• 错误判决调制 VCO/PI → 额外抖动
```

**（3）DFE 反馈延迟限制 CDR 采样精度**

```
224G DFE 延迟问题:
• UI = 8.93 ps
• DFE 1st tap 反馈必须在极短时间内完成
• DFE + CDR 串联 → DFE 延迟压缩 CDR 相位调整窗口
```

**（4）PAM4 CDR 需要多级判决，与均衡器深度绑定**

```
PAM4 CDR 特殊性:
• 3 个眼图，2 个判决电平
• 每个边沿 SNR 比 NRZ 低约 9.5 dB
• CDR 需要从均衡器恢复的"干净眼图"中提取时钟
• 均衡器收敛速度必须快于 CDR 锁定时间
```

## 3. CDR 架构选择与技术细节

### 3.1 224G CDR 架构演进路线

```
224G CDR 架构演进:

56G/112G (当前主流)
├── PLL-Based CDR
├── Dual-Loop CDR (PLL + PI)
└── Bang-Bang CDR

224G (下一代)
├── Sub-Sampling CDR + DSP
├── Dual-Loop CDR (超高频 LC-PLL + 超高分辨率 PI)
├── ADC-Based Digital CDR
└── Baud-Rate CDR (Mueller-Muller + 数字辅助)
```

### 3.2 架构一：Dual-Loop CDR 的极限扩展

**当前最优实践**: Xilinx 16nm Dual-Loop CDR (0.5-32.75 Gb/s)

**224G 扩展关键参数**:
| 模块 | 112G 要求 | 224G 要求 | 技术挑战 |
|------|-----------|-----------|---------|
| 全局 PLL | 8-16.375 GHz | 16-32.75 GHz | LC-VCO 频率上限，Q 值下降 |
| PI 分辨率 | 5-6 bit | **7-8 bit** | 相位步长需 < 1° @ 28 GHz |
| PI 线性度 | ±1 LSB INL | **±0.5 LSB INL** | 非线性转化为确定性抖动 |
| CDR 带宽 | 5-10 MHz | 8-15 MHz | 带宽增加导致参考噪声敏感度上升 |

### 3.3 相位插值器 (PI) 量化分析

```
112G PAM4 vs 224G PAM4 的 PI 步长:

112G PAM4:
  时钟频率 ≈ 28 GHz
  UI = 17.86 ps
  若 PI 为 6 bit (64 阶)
  相位步长 = 17.86 ps / 64 ≈ 279 fs

224G PAM4:
  时钟频率 ≈ 56 GHz  
  UI = 8.93 ps
  若 PI 仍为 6 bit (64 阶)
  相位步长 = 8.93 ps / 64 ≈ 140 fs
  
  但 224G RMS 抖动预算可能只有 200-300 fs
  140 fs 的量化步长会吃掉近一半抖动预算!

结论: 224G 需要 7-8 bit PI，步长进入 35-70 fs 级别
```

### 3.4 架构二：ADC-Based Digital CDR

```
架构示意:
模拟前端: CTLE ──▶ VGA ──▶ 高速 ADC (112 GSa/s, 6-7 bit)
                      │
                      ▼
数字域:   数字插值器 ──▶ 数字 CDR (Baud-rate 或 Fractional-rate)
                      │
                      ▼
            数字 DFE / MLSE ──▶ 判决输出
```

**优势**:
- 完全消除模拟 PI 的非线性和量化误差
- 数字 CDR 带宽和环路参数可灵活编程
- 便于工艺迁移和 PVT 补偿

**劣势**:
| 问题 | 影响 |
|------|------|
| ADC 功耗 | 112 GSa/s × 7 bit Flash ADC 可能 > 500 mW |
| 数字 DSP 延迟 | 数字插值器和 CDR 反馈延迟必须远小于 UI |
| 量化噪声 | ENOB 若只有 6 bit，PAM4 多电平判决 SNR 显著降低 |

**功耗估算**:
```
65nm CMOS 中:
  12.5 GSa/s 4.5-bit Flash ADC + 2-tap FFE & 5-tap DFE
  总功耗: 330 mW (数字部分 85 mW)

按速率比例推算 224G:
  ADC-based 方案总功耗可能达到 2-4 W
  远超功耗预算 (< 1 W/lane)
```

**结论**: ADC-Based CDR 在 224G 中可能是研究热点，但短期内更可能以**混合架构** (模拟 CDR + 数字辅助校准) 形式出现。

### 3.5 架构三：Sub-Sampling CDR

**优势**:
- 大幅降低采样器和时钟树功耗
- 避免全速率时钟分配挑战

**劣势**:
- 收敛速度慢
- 对噪声敏感
- 需要复杂数字后端处理

## 4. 均衡链路设计与 CDR 交互

### 4.1 224G 均衡器配置演进

| 速率 | TX EQ | RX EQ | 备注 |
|------|-------|-------|------|
| 56G | 3-tap FFE | CTLE + 1-tap DFE | 基本够用 |
| 112G | 4-tap FFE | CTLE + 2-tap DFE | 标配 |
| **224G** | 5-tap FFE + 预编码 | CTLE + **4-tap DFE** + 弱 DSP | 或引入 ADC-based |

### 4.2 CTLE 与 CDR 的噪声交互

**CTLE 的噪声增强效应**:
```
CTLE 传递函数:
H(s) = g_m · R_L · (1 + s/ω_z) / (1 + s/ω_p)

其中:
  DC 增益较低
  高频增益被提升以补偿信道损耗
  但同时放大高频噪声
```

**Noise-Induced Jitter**:
```
CTLE 虽然恢复边沿幅度
但如果噪声太大
边沿会在采样时刻附近随机抖动

这种由 CTLE 放大噪声引起的抖动
会直接转化为 CDR 的输入抖动
```

**协同设计策略**:

1. **联合优化 CTLE 峰值与 CDR 带宽**
   - CTLE 高频峰值不应超过信道损耗补偿需求
   - 每多 1 dB 过度峰值，都会引入额外噪声功率
   - CDR 带宽应设置刚好能跟踪信道低频 wander

2. **AGC 位置优化**
   ```
   传统链路: CTLE → AGC → DFE → CDR
   
   优化链路: CTLE → VGA (细粒度控制) → DFE → CDR
   
   VGA 控制信号从 CDR 的 BER 监控或眼图扫描中提取
   实现闭环优化
   ```

### 4.3 DFE 的关键路径与 CDR 采样时序

**DFE 的 1st Tap 反馈是 224G 的最大瓶颈**

```
224G PAM4 延迟预算:
• UI = 8.93 ps
• DFE 需要在当前采样完成后立即计算 ISI 抵消量
• 在当前 UI 内完成减法

典型延迟 (基于 112G 外推):
• 采样器 (StrongARM Latch): ~30 ps
• 1st Tap 反馈路径 (MUX + 电流舵 DAC): ~20-30 ps

总计: ~50-60 ps > 8.93 ps UI !

结论: 必须采用特殊架构才能满足时序要求
```

### 4.4 三大解决方案

**（1）Half-Rate / Quarter-Rate 架构**

```
全速率:  符号周期 = T

半速率:  符号周期 = 2T (可用更慢的电路处理)
Quarter-Rate: 符号周期 = 4T

示例 (Quarter-Rate):
• 4 个采样器交错工作
• 每个只需在 4 UI (35.7 ps) 内完成一次采样
• DFE 反馈延迟要求放宽到 35.7 ps
• 但硬件面积和功耗增加 4 倍
```

**（2）Speculative DFE (预测型 DFE)**

```
工作原理:
1. 在判决结果出来之前，提前做两种假设
   (前一个符号为 0 或 1)
2. 并行计算两种 ISI 抵消值
3. 等判决结果出来后，用 MUX 选择正确的那一路

效果:
• 将"判决器"移出关键路径
• 但硬件复杂度翻倍
```

**（3）Loop-Unrolled DFE**

```
对前 N 个符号的所有可能组合做并行计算

适用于 PAM4 (每个符号 2 bit，组合数为 4^N):
• N=1: speculative DFE
• N=2: 需要 16 路并行

限制: 面积和功耗随 N 指数增长
      通常只展开 1-2 个 tap
```

### 4.5 CDR 与 DFE 的时序协同

```
关键洞察:
CDR 的采样时钟相位
不仅需要考虑眼图中心
还需要考虑 DFE 的延迟

在系统级仿真中，必须将:
• CDR 的相位调整
• DFE 的反馈延迟
• 采样器建立时间
作为一个整体来优化

最佳采样相位:
可能不再严格位于眼图中心
而是偏向某个能最大化 DFE 收敛裕量的位置
```

## 5. 四大维度联合优化

### 5.1 抖动预算协同

**224G 总抖动 (TJ) 预算约为 2.2 ps (0.25 UI)**

```
总抖动预算 (TJ = 2.2 ps)
├── CDR 相关抖动
│   ├── VCO 相位噪声: ~0.3 ps RMS
│   ├── PI 量化抖动: ~0.2 ps RMS
│   ├── Bang-Bang PD 抖动: ~0.25 ps RMS
│   └── 参考时钟抖动 (经 CDR 滤波后): ~0.2 ps RMS
│
├── 均衡相关抖动
│   ├── CTLE 噪声增强: ~0.3 ps RMS
│   ├── DFE 误判传播: ~0.2 ps RMS
│   └── ADC 量化抖动: ~0.15 ps RMS
│
└── 信道相关抖动
    ├── ISI (经均衡后残余): ~0.3 ps RMS
    ├── 串扰 (Crosstalk): ~0.2 ps RMS
    └── 电源/地噪声: ~0.2 ps RMS
```

**协同设计原则**:
- 单独优化 CDR 或均衡器某一项，可能会牺牲另一项
- 例如，提高 CTLE 峰值可以改善 ISI，但会放大噪声
- 需要建立**端到端抖动模型**，通过系统级仿真找到全局最优解

### 5.2 带宽分配协同

**链路中存在多个带宽参数，存在复杂耦合关系**:

| 模块 | 带宽参数 | 影响 |
|------|----------|------|
| TX PLL | ~10 GHz (VCO) | 发射端时钟抖动 |
| CDR | 8-15 MHz (抖动带宽) | 跟踪输入抖动 vs 抑制参考噪声 |
| CTLE | 20-40 GHz (零点) | 补偿信道高频损耗 |
| DFE 自适应 | ~1 MHz (LMS 步长) | 跟踪信道慢变化 |
| AGC | ~10 MHz | 跟踪信号幅度变化 |

**关键协同问题**:

1. **CDR 带宽与 AGC 带宽耦合**
   ```
   若 AGC 响应太快 (带宽 > CDR 带宽)
   → 放大 CDR 无法跟踪的高频幅度调制
   → 引入 AM-to-PM 转换
   → 转化为相位抖动
   
   建议: AGC 带宽 ≤ 0.5 × CDR 带宽
   ```

2. **CTLE 自适应与 CDR 锁定的时序**
   ```
   CTLE 自适应算法需要时间收敛
   CTLE 尚未收敛时，眼图可能完全闭合
   → CDR 无法锁定
   
   解决方案:
   • 上电时先用训练序列快速收敛 CTLE
   • 再启动 CDR
   • 或使用开环 CDR 先粗略锁定，再切换到闭环
   ```

3. **DFE 收敛速度与 CDR 稳定性**
   ```
   DFE 使用 LMS 算法自适应更新抽头系数
   若 LMS 步长太大
   → DFE 抽头会振荡
   → 残余 ISI 周期性变化
   → 落在 CDR 带宽内，被 CDR 跟踪
   → 输出更大抖动
   
   建议:
   • LMS 步长使 DFE 收敛时间 >> CDR 锁定时间
   • DFE 更新频率 > 10 × CDR 带宽
   ```

### 5.3 参考时钟、VCO 与 PI 联合设计

**224G 对参考时钟要求极其苛刻**:

```
参考时钟频率: 100 MHz 或 156.25 MHz
倍频比: 224G / 156.25M ≈ 1433.6×
参考时钟相位噪声经 PLL 倍频后被放大 20×log₁₀(1433.6) ≈ 63 dB

224G 参考时钟要求:
在 10 kHz 偏移处的相位噪声可能需要 <-150 dBc/Hz
```

**VCO 选择**:
| 类型 | 优点 | 缺点 |
|------|------|------|
| LC-VCO | 相位噪声好 | 面积大，调谐范围窄 |
| Ring-VCO | 面积小，调谐范围宽 | 相位噪声差 |

**224G 趋势**: 全局使用 LC-VCO (追求低噪声)，本地使用 Ring-VCO 或 PI (追求宽调谐范围)

**PI 非线性校准**:
```
PI 的 INL (积分非线性) 会直接转化为确定性抖动
224G 需要 7-8 bit 分辨率
INL 必须 < 0.5 LSB

需要: 数字辅助校准 (Digital Background Calibration)
校准算法从 CDR 的抖动监测中提取误差信息
```

### 5.4 功耗协同优化

**224G 功耗目标**: < 5 pJ/b (Linear Drive 架构下可能要求 < 3 pJ/b)

| 模块 | 112G 典型功耗 | 224G 预估功耗 | 优化方向 |
|------|--------------|--------------|---------|
| TX Driver + FFE | 80-120 mW | 150-200 mW | 低摆幅驱动、电流复用 |
| RX CTLE | 30-50 mW | 50-80 mW | 无源/有源混合 CTLE |
| RX DFE | 50-100 mW | 150-250 mW | speculative / 半速率架构 |
| CDR (PLL+PI) | 40-80 mW | 80-150 mW | 共享全局 PLL |
| ADC (若使用) | 150-300 mW | **500-1000 mW** | 低分辨率 SAR/流水线 |
| **总计** | ~400 mW | **~1000 mW** | 协同降功耗 |

**协同降功耗策略**:
```
1. 共享全局 PLL
   多通道 SerDes 中，所有 lane 共享一个高频 LC-PLL
   每个 lane 只保留低功耗的本地 PI

2. 自适应功耗管理
   根据信道质量动态关闭部分 DFE tap
   或降低 ADC 分辨率

3. CTLE + DFE 联合优化
   避免 CTLE 过度补偿
   减少 DFE 的 tap 数需求

4. 数字辅助代替纯模拟
   用低功耗数字校准代替高功耗模拟补偿电路
```

## 6. 仿真验证与测试方法

### 6.1 系统级仿真链路

```
Level 1: 信道级 (ADS/HFSS/Sigrity)
├── S 参数提取
├── 时域脉冲响应
└── 串扰模型

Level 2: 电路级 (Cadence Spectre/HSPICE)
├── TX Driver + FFE
├── CTLE + VGA
├── DFE 电路 (Latch + MUX + DAC)
├── CDR (PLL/PI/PD)
└── 噪声源注入

Level 3: 行为级 (MATLAB/Python/Verilog-A)
├── 自适应算法 (LMS/CMA)
├── CDR 锁定行为
├── 抖动传递函数
└── 联合优化 (眼图 + BER + 抖动)

Level 4: 系统级 (ADS/VPI/OptiSystem)
├── 光电联合仿真
├── 光纤非线性
└── 完整链路 BER 评估
```

### 6.2 关键测试项目

| 测试项 | 目的 | 设备 |
|--------|------|------|
| 抖动传递函数 (Jitter Transfer) | 验证 CDR 带宽和峰值 | BERT + 抖动注入源 |
| 抖动容限 (Jitter Tolerance) | 验证 CDR 最大可接受输入抖动 | BERT |
| 抖动产生 (Jitter Generation) | 测量 CDR 自身引入的抖动 | 高性能采样示波器 |
| 眼图张开度 | 验证均衡器有效性 | 实时示波器 (> 70 GHz) |
| BER 浴盆曲线 | 找到最佳采样相位和裕量 | BERT |
| 功耗效率 (pJ/b) | 验证功耗目标 | 精密电源 |

### 6.3 "CDR + EQ 联合优化"仿真流程

```
1. 固定信道模型
   导入实测的 S 参数或信道脉冲响应

2. 扫描 TX FFE 系数
   找到使接收端眼图最大化的预加重设置

3. 扫描 CTLE 峰值
   在每个 FFE 设置下，找到最优的 CTLE 零点/极点

4. 启动 DFE 自适应
   运行 LMS 算法，观察收敛后的残余 ISI

5. 启动 CDR 闭环
   在均衡收敛后的波形上，注入不同频率和幅度的正弦抖动

6. 测量 CDR 抖动容限和输出抖动
   绘制 Jitter Tolerance 和 Jitter Transfer 曲线

7. 联合优化
   目标: BER < 1e-5 (FEC 前)
   约束: 功耗 < 1 W/lane
   搜索全局最优参数组合
```

### 7.3 CDR 架构对比

| 架构 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|

## 核心结论

```
1. 224G CDR 必须采用 Dual-Loop 架构或混合数字辅助架构
   单环 PLL-Based CDR 难以满足抖动和锁定范围的双重要求

2. 相位插值器需要 7-8 bit 分辨率
   步长进入 35-70 fs 级别
   必须配备数字非线性校准

3. 均衡链路标配 TX 5-tap FFE + RX CTLE + 4-tap DFE
   DFE 的 1st tap 延迟是最大瓶颈
   需要 speculative 或 half-rate 架构

4. CDR 与均衡不再是独立模块
   必须在抖动预算、带宽分配、时序收敛、功耗四个维度上联合优化

5. ADC-Based CDR 是长期研究方向
   短期内功耗过高 (> 500 mW)
   更可能以混合形式 (模拟 CDR + 数字校准) 出现
```
