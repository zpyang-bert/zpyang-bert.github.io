## 执行摘要

Linear Drive（又称 LPO，Linear Pluggable Optics）通过**移除光模块内部的 CDR 和 DSP 芯片**，将信号补偿责任转移到主机侧（Host ASIC）的 SerDes 上，从而显著降低功耗和延迟。然而，这一架构变革彻底改变了通道的特性：传统 Retimed 模块输出的是经过 CDR 重定时后的"干净"数字信号，而 Linear 模块输出的是经过线性放大后的模拟信号，包含**全部的信道失真、噪声和非线性**。这意味着主机 SerDes 必须直接面对一个**电气-光学联合的复杂通道**。

本报告基于资料库中的 IPEC Linear Drive 白皮书、EE720 高速链路课程、OIF CEI-Linear 规范以及 OFC 2026 相关研究，系统构建了 Linear Drive 通道的分层模型，分析了从主机 ASIC 到对端主机 ASIC 的完整信号路径中的关键建模参数、非线性来源和补偿策略，为主机侧 SerDes 的设计和链路预算提供了理论依据。

## 第一章：Linear Drive 架构背景与通道差异

### 1.1 传统 Retimed vs Linear Drive 架构

```
【传统 Retimed 可插拔模块】

主机 ASIC (TX)
    │ 高速串行电信号 (PAM4/NRZ)
    ▼
┌─────────────────────────────┐
│        光模块内部            │
│  ┌─────┐  ┌─────┐ ┌─────┐  │
│  │ CDR │─▶│ DSP │─▶│Driver│─▶ 激光器/调制器
│  └─────┘  └─────┘ └─────┘  │
└─────────────────────────────┘
         │
         ▼ 光纤
┌─────────────────────────────┐
│        光模块内部            │
│  ┌─────┐  ┌─────┐ ┌─────┐  │
│  │ TIA │─▶│ DSP │─▶│ CDR │─▶ 主机 ASIC (RX)
│  └─────┘  └─────┘ └─────┘  │
└─────────────────────────────┘

特点:
- CDR 在模块内重定时，消除输入抖动
- DSP 进行均衡、色散补偿、非线性补偿
- 主机侧 SerDes 看到的电通道是"模块输入/输出"
- 光通道被 DSP "隔离"，主机侧无需关心光纤特性
```

```
【Linear Drive (LPO) 架构】

主机 ASIC (TX)
    │ 高速串行电信号 (带预加重/FFE)
    ▼
┌─────────────────────────────┐
│        光模块内部            │
│  ┌─────────┐ ┌─────────┐   │
│  │ Linear  │─▶│ Laser/  │─▶ 光纤
│  │ Driver  │  │ Modulator│  │
│  └─────────┘  └─────────┘   │
└─────────────────────────────┘
         │
         ▼ 光纤 (无模块内 DSP 补偿)
┌─────────────────────────────┐
│        光模块内部            │
│  ┌─────┐ ┌─────────┐       │
│  │ TIA │─▶│ Linear  │─▶ 主机 ASIC (RX)
│  └─────┘  │ Amplifier│      │
│           └─────────┘       │
└─────────────────────────────┘

特点:
- 模块内无 CDR，无 DSP
- 主机侧 SerDes 直接"看到"完整的光学通道特性
- 所有均衡、补偿、时钟恢复都在主机 ASIC 中完成
- 功耗降低 50%+，延迟接近 0
```

### 1.2 Linear Drive 带来的通道本质变化

| 特性 | 传统 Retimed | Linear Drive |
|------|--------------|--------------|
| 电通道长度 | 主机 → 模块输入 (~5–15 cm PCB) | 主机 → 模块输入 (~5–15 cm PCB) |
| 光通道补偿 | 模块内 DSP 完成 | **主机 SerDes 完成** |
| 输出信号类型 | 数字重定时信号 | **模拟线性放大信号** |
| 抖动传递 | CDR 隔离高频抖动 | **全部传递到主机 RX** |
| 非线性 | DSP 补偿 | **主机 DSP 补偿** |
| 主机 SerDes 复杂度 | 中 | **高** |
| 功耗 | ~15 pJ/b | **~3–5 pJ/b** |

**核心变化**：主机 SerDes 的设计假设从"只需要补偿一段短 PCB 走线"变成了"必须补偿 **PCB + 连接器 + 光模块内部电路 + 光器件 + 光纤 + 对端光器件 + 对端 PCB** 的完整端到端通道"。

## 第二章：Linear Drive 通道的分层模型

我们将 Linear Drive 的完整通道划分为四个层级：**主机侧电通道**、**光模块内部电通道**、**光电转换通道**、**光纤光学通道**。

```
完整 Linear Drive 端到端通道模型:

主机 ASIC (TX) ──▶ [主机侧电通道] ──▶ [光模块 TX 内部] ──▶ [光电转换] 
                                                                   │
                                                                   ▼
                                                            [光纤光学通道]
                                                                   │
                                                                   ▼
主机 ASIC (RX) ◀── [主机侧电通道 RX] ◀── [光模块 RX 内部] ◀── [光电转换]
```

### 2.1 主机侧电通道（Host Electrical Channel）

这是从主机 ASIC 的 TX 焊盘到光模块金手指（或 CPO 中的 interposer/基板）之间的电气路径。

**典型拓扑**：
```
ASIC BGA ──▶ 封装基板 (Package Substrate) ──▶ PCB 过孔 ──▶ PCB 走线 
                                                    │
                                                    ▼
                                              连接器/金手指 (Connector/Finger)
                                                    │
                                                    ▼
                                              光模块 PCB (Module PCB)
                                                    │
                                                    ▼
                                              模块内 TIA/Driver 芯片
```

**关键建模参数**：

| 参数 | 典型值 @ 56 GHz (224G Nyquist) | 建模方法 |
|------|-------------------------------|----------|
| 插入损耗 (IL) | 8–15 dB | S 参数 / RLCG 传输线模型 |
| 回波损耗 (RL) | > 8 dB | S 参数 / TDR 提取 |
| 阻抗不连续性 | ±10% (目标 ±5%) | 3D 电磁仿真 (HFSS/ADS) |
| 串扰 (FEXT/NEXT) | <-30 dB | 耦合传输线模型 |
| 过孔损耗 | 0.5–2 dB | 过孔等效电路模型 |
| 连接器损耗 | 1–3 dB | 厂商 S 参数 + 实测 |

**EE720 Lecture 2 & 3 的启示**：
- 信道损耗主要由 **趋肤效应**（Skin Effect，与 √f 成正比）和 **介质损耗**（Dielectric Loss，与 f 成正比）组成
- 在 56 GHz 时，介质损耗占主导
- 对于 Linear Drive，主机侧电通道的 S 参数必须精确建模到 **2× Nyquist 频率（112 GHz）**，因为光器件的非线性会产生谐波和交调产物

**TDR/S 参数建模要点**：
- 使用 **S 参数级联**方法将封装、PCB、连接器、模块 PCB 的 S 参数串联
- 注意 S 参数的 **参考阻抗一致性**（通常为 100 Ω 差分）
- 对于高频（> 50 GHz），S 参数的 **相位精度** 至关重要，建议使用实测数据而非纯仿真

### 2.2 光模块内部电通道（Module Internal Electrical Channel）

这是光模块内部从金手指到激光器驱动器（或 TIA）之间的短距离走线。

**典型结构**：
```
金手指 ──▶ 模块 PCB 走线 (~2–5 cm) ──▶ Wire Bond / Flip-Chip ──▶ Driver IC / TIA IC
```

**关键特性**：
- 走线短，但存在 **Wire Bond** 引入的寄生电感（~0.5–2 nH）
- 模块 PCB 通常为薄板（2–4 层），走线宽度受限，特征阻抗控制难度大
- 在 224G 下，Wire Bond 的寄生电感与 Driver/TIA 的输入电容形成谐振，可能在 40–60 GHz 产生峰值或凹陷

**建模方法**：
```
Wire Bond 等效电路:

    L_bond (~1 nH)
    ━//\/\/-┬-┬-
            │ │
           ─┴─┴─
           C_pad (~100 fF)
           ─┬─┬─
            │ │
           ─┴─┴─
```

- Wire Bond 电感：可通过 3D 电磁仿真提取，或根据线长/线径估算（~1 nH/mm）
- 焊盘电容：~50–150 fF
- 在 56 GHz 时，1 nH 电感的感抗为 ~350 Ω，与 50 Ω 系统严重失配，必须通过 **阻抗匹配网络** 或 **共封装（CPO）** 来消除 Wire Bond

**CPO 对通道的改善**：
- CPO（Co-Packaged Optics）将光引擎与 ASIC 共封装，电互连距离缩短到 **< 2 mm**
- 消除了连接器、模块 PCB 和 Wire Bond
- 电通道损耗从 8–15 dB 降至 **< 3 dB**
- 这是 224G Linear Drive 最理想的应用场景

### 2.3 光电转换通道（Opto-Electrical Conversion Channel）

这是 Linear Drive 区别于传统 Retimed 的最关键部分。主机 SerDes 必须"看穿"到光电器件层面。

#### 2.3.1 发射端：Driver → Laser/Modulator

**Directly Modulated Laser (DML) / VCSEL**：
- Driver 电流直接调制激光器的输出光功率
- 小信号带宽受激光器的 **弛豫振荡频率** 限制（典型 20–40 GHz）
- **非线性来源**：
  1. **L-I 曲线非线性**：阈值电流以上近似线性，但大信号下饱和
  2. **啁啾（Chirp）**：直接调制引起波长漂移，在光纤色散下导致额外脉冲展宽
  3. **热效应**：调制电流引起结温变化，导致静态工作点漂移
  4. **限幅效应（Clipping）**：驱动电流不能低于阈值（否则消光比恶化）

**Electro-Absorption Modulator (EAM) / Mach-Zehnder Modulator (MZM)**：
- Driver 电压控制光调制器的吸收/相位
- **MZM 的非线性**：
  ```
  P_out = P_in · cos²(π · V / (2 · Vπ))
  ```
  这是典型的 **正弦型非线性**。对于 PAM4 信号，MZM 必须工作在 **线性工作区**（通常为 Null 点到 Quad 点之间的较小范围内），否则高低电平的压缩不同，导致眼图不对称。
- **Vπ 频率依赖性**：高频下 Vπ 会增大（因为电极的微波损耗），导致高频分量的调制深度降低

**OFC 2026 Th4A.1 / Th4B.3 的启示**：
- NTT 的 EA-DFB 在 0.5V 摆幅下实现 448 Gbps PAM4，说明**低摆幅驱动**可以降低非线性
- McGill 的 BTO MZM 实现了 55/105 GHz 的 EO S21，但 Vπ = 6V，需要高增益 RF Driver
- 对于 Linear Drive，Driver 的设计必须在 **带宽、线性度、摆幅** 之间取得平衡

#### 2.3.2 接收端：Photodetector → TIA → Linear Amplifier

**光电探测器（PIN / APD）**：
- 将光功率转换为光电流
- 带宽受 **结电容** 和 **载流子渡越时间** 限制
- 高频响应可建模为一阶或二阶低通滤波器
- 非线性：大光功率下的空间电荷效应（Space Charge Effect）会降低响应度

**TIA（跨阻放大器）**：
- 将光电流转换为电压信号
- 关键参数：跨阻增益、带宽、输入参考噪声电流
- 张震论文中的 TIA 指标：31 GHz 带宽，71 dBΩ 增益，14.54 pA/√Hz 噪声
- **TIA 的带宽限制**会在高频处滚降，这部分滚降会被主机 RX 的 CTLE 补偿

**Linear Amplifier（线性放大器）**：
- LPO 模块中没有 DSP/CDR，只有一级或多级线性放大
- 放大器必须具有极高的 **线性度**（低 IM3），否则会引入新的交调失真
- 增益通常固定或可粗调，不配备复杂的自适应均衡

### 2.4 光纤光学通道（Optical Fiber Channel）

#### 2.4.1 多模光纤（MMF）

- 用于短距离（<100m），典型为数据中心机架内/机架间互联
- **模式色散（Modal Dispersion）**是主要限制：不同模式群速度不同，导致脉冲展宽
- 有效带宽距离积（Bandwidth-Length Product）：
  - OM3: 2000 MHz·km @ 850nm
  - OM4: 4700 MHz·km @ 850nm
  - OM5: 4700 MHz·km @ 850nm, 支持 SWDM
- 对于 100m OM4，模式色散引起的 3-dB 带宽约为 47 GHz，对 224G PAM4（56 GBaud）是足够的
- 但 **VCSEL 的啁啾** 会与模式色散相互作用，产生额外的 ISI

#### 2.4.2 单模光纤（SMF）

- 用于中长距离（>100m 到数 km）
- **色度色散（Chromatic Dispersion, CD）**是主要限制
- 在 1310nm 处，SMF 的 CD 系数约为 0 ps/nm/km（零色散点）
- 在 1550nm 处，CD 系数约为 17 ps/nm/km
- 对于 Linear Drive（无模块内 DSP），如果工作波长在 1550nm 且距离 > 1 km，色散会导致严重的眼图闭合
- **解决方案**：
  1. 使用 **1310nm 光源**（O-band）来避开色散
  2. 在主机 SerDes 中集成 **DSP 进行色散补偿**（但这与 LPO 的低延迟初衷相矛盾）
  3. 使用 **色散补偿光纤（DCF）**或 **啁啾光纤光栅（CFBG）**

**EE720 Lecture 15 的启示**：
- 短距光互连（<100m）中，光纤损耗和色散通常不是瓶颈，**电光器件的带宽和耦合损耗**才是
- 长距光互连（>1km）中，即使使用 Linear Drive，也必须考虑光纤色散和激光器啁啾的联合效应

## 第三章：Linear Drive 联合通道的数学模型

### 3.1 端到端通道的级联模型

将上述四个层级的通道用数学模型串联：

```
Y(f) = H_optical(f) · H_o_e(f) · H_module(f) · H_host(f) · X(f) + N_total(f)
```

其中：
- $X(f)$: 主机 TX 发送的电信号频谱
- $H_{host}(f)$: 主机侧电通道的传递函数（S 参数 S21）
- $H_{module}(f)$: 光模块内部电通道传递函数（含 Wire Bond、模块 PCB）
- $H_{o\_e}(f)$: 光电转换传递函数（Driver → Laser/Modulator → Fiber → PD → TIA → Amp）
- $H_{optical}(f)$: 光纤光学传递函数（色散、模式色散、损耗）
- $N_{total}(f)$: 总噪声（电噪声 + 光噪声）
- $Y(f)$: 主机 RX 接收的电信号频谱

### 3.2 各子通道的详细模型

#### 3.2.1 电气通道模型（Host + Module）

使用 **有损传输线模型**（RLCG model）：
```
∂V/∂z = -(R(f) + jωL) · I
∂I/∂z = -(G(f) + jωC) · V
```

其中：
- $R(f) = R_{DC} + R_s \cdot \sqrt{f}$ （趋肤效应）
- $G(f) = G_0 + G_d \cdot f$ （介质损耗）

对于 PCB 走线，特征阻抗和传播常数为：
```
Z0 = sqrt((R + jωL) / (G + jωC))
γ = sqrt((R + jωL) · (G + jωC))
```

#### 3.2.2 光电转换小信号模型

对于 DML/VCSEL，小信号调制响应可近似为二阶低通：
```
H_laser(f) = f_r² / (f_r² - f² + j · f · γ/2π)
```
其中 $f_r$ 为弛豫振荡频率，$\gamma$ 为阻尼因子。

对于 MZM，频率响应受电极微波损耗限制：
```
H_mzm(f) = exp(-α · f / f_3dB)
```
其中 $f_{3dB}$ 为电光 3-dB 带宽。

对于 TIA，可建模为一阶或二阶低通：
```
H_tia(f) = A_tia / (1 + j · f / f_tia)
```

#### 3.2.3 光纤色散模型（SMF）

色散在频域的传递函数为：
```
H_fiber(f) = exp(j · β₂ · L · (2πf)² / 2)
```
其中：
- $\beta₂ = -D \cdot \lambda² / (2\pi c)$ 为群速度色散参数
- $D$ 为色散系数（ps/nm/km）
- $L$ 为光纤长度
- $\lambda$ 为中心波长
- $c$ 为光速

在时域中，色散导致脉冲展宽：
```
Δτ = D · L · Δλ
```
其中 $\Delta\lambda$ 为光源谱宽（对于直接调制激光器，啁啾会显著增宽谱宽）。

### 3.3 非线性模型

Linear Drive 最大的建模挑战是**光电器件的非线性**，因为模块内没有 DSP 来预补偿这些非线性。

#### 3.3.1 MZM 非线性模型

MZM 的完整大信号模型：
```
P_out(t) = P_in/2 · [1 + cos(π · V(t) / Vπ + φ_bias)]
```

对于 PAM4 信号，$V(t)$ 有 4 个电平。如果偏置点 $\phi_{bias} = \pi/2$（Quad 点），在 $V \ll V\pi$ 时可近似为线性：
```
P_out(t) ≈ P_in/2 · [1 - π · V(t) / Vπ]
```

但实际中，为了保证足够的消光比和眼图张开度，$V(t)$ 的摆幅可能达到 $0.3–0.5 \cdot V\pi$，此时非线性不可忽略。

**对 PAM4 的影响**：
- 4 个电平不再等间距
- 中间两个眼（01 和 10）可能被压缩或扩张
- 需要主机 TX 的 **预失真（Pre-distortion）** 来补偿

#### 3.3.2 激光器 L-I 曲线非线性

DML/VCSEL 的 L-I 曲线：
```
P_opt(t) = η · (I_bias + I_mod(t) - I_th)   (I > I_th)
P_opt(t) = 0                                 (I ≤ I_th)
```

**限幅非线性**：
- 当调制电流 $I_{mod}(t)$ 使总电流低于阈值 $I_{th}$ 时，光输出被钳位到 0
- 对于 PAM4 信号，这会导致最低电平（00）的眼图被"切掉"
- 这种限幅会引入谐波失真，并在接收端产生 ISI

**啁啾模型**：
```
Δν(t) = -α/4π · (d/dt · ln(P_opt(t)) + κ · I_mod(t))
```
其中 $\alpha$ 为线宽增强因子，$\kappa$ 为绝热啁啾系数。

啁啾与色散的联合效应会导致频率相关的群延迟，在时域表现为脉冲的不对称拖尾。

#### 3.3.3 接收端 PD/TIA 非线性

PIN 探测器在大光功率下的空间电荷效应：
- 响应度 $R$ 随光功率增大而轻微下降
- 对于 PAM4 的最高电平（11），响应度可能略低于其他电平
- 这种压缩效应虽然通常较弱（< 1 dB），但在 224G 的紧抖动预算下不可忽略

## 第四章：主机侧 SerDes 的补偿要求

### 4.1 与传统 Retimed 的补偿差异

| 补偿项 | 传统 Retimed | Linear Drive |
|--------|--------------|--------------|
| 电通道 ISI | 主机 TX FFE + RX CTLE/DFE | 相同，但通道更长/更差 |
| 光器件带宽限制 | 模块内 DSP 补偿 | **主机 DSP/DFE 补偿** |
| 光器件非线性 | 模块内 DSP 补偿 | **主机 TX 预失真 + RX 非线性均衡** |
| 光纤色散 | 模块内 DSP 补偿（相干） | **主机 DSP 或 1310nm 规避** |
| 时钟恢复 | 模块内 CDR + 主机 CDR | **主机 CDR 独立承担** |
| 参考时钟传递 | 模块重定时隔离 | **参考时钟抖动直接传递** |

### 4.2 TX 侧补偿策略

**（1）FFE 预加重扩展**
- 传统：3–4 tap FFE 补偿 PCB 损耗
- Linear Drive 224G：需要 **5-tap FFE**，不仅要补偿 PCB，还要预补偿光器件的高频滚降
- FFE 的设计目标：使到达激光器/调制器输入端的信号频谱在 Nyquist 频率处保持平坦

**（2）非线性预失真（Pre-distortion）**
- 针对 MZM 的 $\cos²$ 非线性，TX DSP 可以施加逆非线性映射：
  ```
  V_pre(t) = (Vπ/π) · arccos(1 - 2 · V_target(t)/Vπ_max)
  ```
- 针对 DML 的 L-I 曲线非线性，可以施加逆 L-I 映射，将电平重新调整到线性区间
- 针对 DML 的限幅效应，可以通过 **DC 偏置优化** 和 **信号削波（Clipping）避免** 来缓解

**（3）啁啾 aware 的预编码**
- 如果知道激光器的啁啾参数和光纤色散参数，可以在 TX DSP 中进行 **色散预补偿**
- 这本质上是在电域施加一个与光纤色散相反的相位调制

### 4.3 RX 侧补偿策略

**（1）CTLE 带宽提升**
- 224G Linear Drive 中，CTLE 必须补偿电通道 + 光器件 + TIA 的联合高频滚降
- 总滚降可能达到 30–40 dB @ 56 GHz
- 单级 CTLE 通常只能提供 10–15 dB 的峰值增益，因此可能需要 **多级 CTLE** 或 **CTLE + 有源增益级联**

**（2）DFE Tap 数增加**
- 传统 112G：2-tap DFE
- 224G Linear Drive：可能需要 **4-tap 甚至 6-tap DFE**
- 这是因为光器件的脉冲响应拖尾比纯电气通道更长
- 挑战：tap 数增加会显著增加功耗和反馈延迟

**（3）ADC-Based RX DSP**
- 对于长距离（>1km）的 Linear Drive 链路，DFE 可能不足以补偿光纤色散和激光器啁啾
- 此时需要在主机 RX 中引入 **轻量级 DSP**：
  - 低分辨率 ADC（4–5 bit）@ 112 GSa/s
  - 数字 FIR 均衡器（代替模拟 DFE）
  - 可选的数字色散补偿（用于 1550nm 链路）
- 但这会重新引入功耗问题，与 LPO 的低功耗初衷产生矛盾

**（4）非线性后均衡（Post-equalization）**
- 在 RX 侧可以使用 **Volterra 均衡器** 来补偿光电器件的弱非线性
- Volterra 均衡器的复杂度为 $O(N^k)$，其中 $N$ 为记忆长度，$k$ 为非线性阶数
- 实际中通常只使用 **2 阶 Volterra 核**，并限制记忆长度到 3–5 个符号

### 4.4 CDR 的特殊要求

Linear Drive 没有模块内 CDR，因此主机 CDR 必须直接面对：
1. **更大的输入抖动**：因为光通道的噪声、电源波动、模式噪声等都会传递到电域
2. **非均匀的边沿密度**：某些数据模式可能导致光器件响应变慢，边沿变缓
3. **低频 wander**：激光器的热效应、电源漂移等会在秒级甚至分钟级引起慢变化

**CDR 设计建议**：
- 增加 **CDR 带宽**（从 112G 的 5–10 MHz 提升到 8–15 MHz），以跟踪更多的低频 wander
- 使用 **数字辅助的 Bang-Bang PD 增益控制**，在边沿质量差时降低环路增益，避免过度响应噪声
- 引入 **频率辅助锁定（Frequency Acquisition Aid）**，在启动时通过参考时钟或训练序列快速将 VCO 频率拉到数据率附近

## 第五章：CEI-Linear 规范中的测试点与建模验证

OIF 的 CEI-Linear 规范定义了 4 个关键测试点（TP），这些测试点是通道建模和验证的基准。

```
主机 ASIC (TX)                    光模块 TX                     光纤
     │                                 │                          │
     ▼ TP1                             ▼ TP2                      ▼
┌─────────┐                    ┌─────────────┐              ┌───────────┐
│ 发射端  │────────────────────│ 线性驱动器  │──▶ 激光器 ──▶│   光纤    │
│ 电气规范│                    │   输入端    │              │ 光学规范  │
└─────────┘                    └─────────────┘              └───────────┘
                                                                  │
                                                                  ▼
主机 ASIC (RX)                    光模块 RX                 ┌───────────┐
     ▲                                 ▲                     │   光纤    │
     │ TP4                             │ TP3                 │ 光学规范  │
┌─────────┐                    ┌─────────────┐              └───────────┘
│ 接收端  │────────────────────│ 线性放大器  │◀── 探测器 ──◀│
│ 电气规范│                    │   输出端    │
└─────────┘                    └─────────────┘
```

### 5.1 各测试点的建模要求

| 测试点 | 信号类型 | 关键指标 | 建模重点 |
|--------|----------|----------|----------|
| **TP1** | 电信号 | 眼图、抖动、RL、IL | 主机 TX + 主机 PCB 通道模型 |
| **TP2** | 电信号 | 驱动器输入眼图 | TP1 + 连接器 + 模块 PCB + Wire Bond |
| **TP3** | 光信号 | 光眼图、消光比、OMA | 光纤 + 光器件响应模型 |
| **TP4** | 电信号 | TIA/Amp 输出眼图 | TP3 + 探测器 + TIA + 线性放大器 |

### 5.2 合规性仿真链路

在进行 CEI-Linear 合规性验证时，需要构建以下仿真链路：

**Step 1: 从 TP1 到 TP2**
- 将主机侧电通道 S 参数与模块内部电通道 S 参数级联
- 施加最坏情况（WC）的 PCB 走线长度和过孔数量
- 在 TP2 处测量眼图裕量（Eye Margin）

**Step 2: 从 TP2 到 TP3**
- 将 TP2 的电信号输入到光电器件模型（Driver + Laser/Modulator）
- 考虑光器件的带宽限制、非线性、啁啾
- 通过光纤模型（MMF 或 SMF）
- 在 TP3 处测量光眼图（Optical Eye）和 TDECQ

**Step 3: 从 TP3 到 TP4**
- 将 TP3 的光信号输入到探测器 + TIA + 线性放大器模型
- 考虑探测器响应度、TIA 噪声、放大器增益和带宽
- 在 TP4 处测量输出电眼图

**Step 4: 从 TP4 到主机 RX**
- 将 TP4 信号通过主机 RX 侧 PCB 通道
- 施加主机 RX 的 CTLE + DFE + CDR
- 测量最终 BER 和抖动容限

### 5.3 关键 S 参数与眼图指标

**CEI-112G-Linear 的电气参考要求**（供 224G 外推参考）：

| 指标 | TP1 (主机 TX) | TP4 (模块 RX 输出) |
|------|---------------|-------------------|
| 差分阻抗 | 100 Ω ± 10% | 100 Ω ± 10% |
| 回波损耗 | > 10 dB @ 56 GHz | > 8 dB @ 56 GHz |
| 眼高 | > 80 mV (PAM4 outer) | > 30 mV (PAM4 outer) |
| 眼宽 | > 0.4 UI | > 0.25 UI |
| 抖动 (RMS) | < 0.1 UI | < 0.15 UI |

**注意**：224G 的指标目前尚无正式标准，上述为基于 112G 的合理外推。

## 第六章：224G Linear Drive 的通道建模挑战与解决方案

### 6.1 挑战一：高频电磁建模精度不足

**问题**：
- 224G 的 Nyquist 频率为 56 GHz，需要考虑 2–3 次谐波（112–168 GHz）
- 传统的 2D/2.5D 电磁仿真工具（如 SIwave、ADS Momentum）在 > 50 GHz 时精度下降
- 3D 全波仿真（HFSS）虽然精度高，但计算量巨大，一个连接器模型可能需要数天

**解决方案**：
1. **混合建模**：对关键结构（连接器、过孔、Wire Bond）使用 3D 仿真，对均匀传输线使用 2D RLCG 模型
2. **实测校准**：使用 VNA（矢量网络分析仪）实测 S 参数到 110 GHz，用实测数据修正仿真模型
3. **降阶模型（ROM）**：将 3D 仿真结果提取为 SPICE 等效电路或状态空间模型，加速系统级仿真

### 6.2 挑战二：光电器件模型的参数获取困难

**问题**：
- VCSEL、EAM、MZM 的精确 SPICE 模型通常是厂商专有 IP，难以获取
- 小信号 S 参数无法完整描述大信号非线性
- 温度、老化、批次差异导致模型参数漂移

**解决方案**：
1. **基于测量的行为建模**：
   - 使用 L-I-V 曲线、P-V 曲线、EO S21 等测量数据构建 lookup table 模型
   - 在系统仿真中使用 Verilog-A 或 MATLAB 的行为模型
2. **机器学习辅助建模**：
   - 利用神经网络（如 LSTM、PINN）从测量数据中学习光电器件的输入-输出映射
   - OFC 2026 的 Optics GPT（Th4C.1）展示了 AI 在光学系统建模中的潜力
3. **统计建模**：
   - 对关键参数（如阈值电流、Vπ、响应度）进行蒙特卡洛仿真，评估工艺/温度漂移的影响

### 6.3 挑战三：联合仿真的计算复杂度

**问题**：
- 电气通道是线性（或弱非线性）的，适合频域 S 参数仿真
- 光电器件是强非线性的，适合时域电路仿真
- 光纤色散是线性但具有长记忆（数千个符号），适合频域或分步傅里叶法
- 将这三者联合仿真，时间和内存开销极大

**解决方案**：
1. **分域仿真**：
   - 电气通道：频域 S 参数滤波
   - 光电器件：时域非线性模型
   - 光纤：频域色散滤波器 + 时域非线性分步
   - 在接口处通过 FFT/IFFT 转换
2. **并行化与 GPU 加速**：
   - 光纤传输的分步傅里叶法可以高度并行化
   - 使用 GPU 加速蒙特卡洛 BER 仿真
3. **简化模型**：
   - 对于短距（<100m）链路，可以忽略光纤非线性，只使用线性色散滤波器
   - 对于低啁啾光源，可以简化啁啾模型

### 6.4 挑战四：抖动与噪声的联合统计建模

**问题**：
- Linear Drive 的抖动来源比 Retimed 多得多：电源噪声、参考时钟抖动、激光器 RIN、模式噪声、放大器噪声等
- 这些噪声源具有不同的统计分布（高斯、泊松、1/f 噪声等）
- 传统的正态分布假设可能低估 BER

**解决方案**：
1. **分层噪声预算**：
   - 将总噪声分解为各子系统的贡献
   - 对每种子噪声使用合适的统计模型
2. **瞬态噪声注入**：
   - 在时域仿真中注入实测的电源噪声波形和 RIN 噪声
   - 统计 BER 而非仅使用 Q 因子近似
3. **半解析方法**：
   - 使用 Karhunen-Loève 展开或 Volterra 级数来分析非线性通道中的噪声传播

## 第七章：结论与设计建议

### 7.1 核心结论

1. **Linear Drive 的通道模型必须从"纯电气"扩展为"电气-光学联合通道"**。主机 SerDes 的设计必须同时考虑 PCB 走线、连接器、Wire Bond、激光器/调制器、光纤、探测器、TIA 的级联效应。

2. **光电器件的非线性是 Linear Drive 的最大建模难点**。MZM 的 $\cos²$ 非线性、DML 的 L-I 曲线限幅、激光器啁啾与色散的联合效应，都必须纳入通道模型。

3. **Wire Bond 和模块 PCB 在 224G 下可能成为瓶颈**。1 nH 的 Wire Bond 电感在 56 GHz 时产生 350 Ω 的感抗，严重失配。CPO（共封装光学）是消除这一瓶颈的最有效方案。

4. **主机侧 SerDes 的补偿能力决定了 Linear Drive 的链路距离**。对于短距（<100m），TX FFE + RX CTLE + DFE 通常足够；对于中长距（>1km），可能需要在主机 RX 中引入轻量级 ADC-Based DSP。

5. **CEI-Linear 的 TP1/TP2/TP3/TP4 测试点为通道建模提供了标准化框架**。端到端仿真应逐级验证每个测试点的眼图和抖动指标。

### 7.2 设计建议

**对主机 ASIC 设计师**：
- 在设计 SerDes 时，不要只考虑 PCB 损耗，要为 **光器件带宽限制** 和 **非线性预失真** 预留 DSP 能力
- TX FFE 应支持 **5-tap 或更多**，并具备灵活的预失真查找表（LUT）功能
- RX CTLE 应支持 **多级级联** 或 **可编程高增益峰值**，以补偿 35–40 dB 的总滚降
- CDR 的抖动带宽应适当放宽（8–15 MHz），并引入数字辅助增益控制

**对光模块设计师**：
- 尽量降低模块内部电通道的损耗和不连续性（优化模块 PCB 设计、缩短 Wire Bond）
- Driver 和 TIA 的设计必须优先考虑 **线性度** 而非单纯的带宽
- 对于 MZM，应在模块内集成 **偏置点监控和反馈**，确保长期工作在线性区
- 对于 DML/VCSEL，应选择 **低啁啾** 设计，以减轻光纤色散的影响

**对系统设计师**：
- 短距应用（机架内）优先使用 **850nm/940nm VCSEL + MMF**，可忽略色散
- 中距应用（数据中心楼宇间）优先使用 **1310nm EML/DFB + SMF**，规避 1550nm 的色散问题
- 长距应用（>10km）谨慎使用 Linear Drive，除非主机 RX 具备 DSP 能力

### 7.3 待深入研究的问题

1. **224G CPO 的封装级电磁建模**：芯片与光引擎之间 <2mm 的互连，如何精确提取毫米波频段的 S 参数？
2. **激光器啁啾与 MMF 模式色散的联合效应**：在高速 VCSEL 链路中，啁啾引起的模式耦合如何定量建模？
3. **温度循环下的通道参数漂移**：Linear Drive 没有模块内 DSP 的自动补偿，温度变化如何影响链路裕量？
4. **多通道串扰（Crosstalk）的光学分量**：在密集波分复用（DWDM）或并行光纤中，邻通道串扰如何纳入通道模型？
5. **AI 辅助的光电联合通道建模**：如何利用生成式模型（如 GAN、Diffusion Model）快速构建光电器件的精确行为模型？
