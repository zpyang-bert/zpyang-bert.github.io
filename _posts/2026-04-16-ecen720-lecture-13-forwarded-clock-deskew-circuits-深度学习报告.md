## 一、前向时钟I/O概述

### 1.1 架构特点

```
前向时钟I/O架构:
• 从TX芯片转发高速参考时钟到RX芯片
• Mesochronous系统
• 用于处理器-存储器接口和多处理器通信
• Intel QPI, Hypertransport

"相干时钟":
• 允许低到高频抖动跟踪
• 需要良好的时钟接收放大器(时钟被信道衰减)
```

![](/img/serdes/fundamentals/lectures/lecture10_ee720_jitter_深度学习报告/_images/img-000.jpg)

> 🔍 深度说明：
> 【研究背景】前向时钟架构是高速链路中常用的时钟分发方案，特别是在处理器-存储器接口（如Intel QPI、HyperTransport）中。相比嵌入式时钟，前向时钟将时钟信号单独从TX转发到RX，允许RX进行独立的时钟恢复。
> 【核心结论】Mesochronous系统（共用频率但相位独立）通过前向时钟实现数据与时钟的同步传输，关键在于时钟接收放大器需要具备低噪声和高带宽特性，以应对信道对时钟信号的衰减。
> 【工程价值】在多处理器系统中，前向时钟架构可减少时钟恢复复杂度，但需要精确的deskew电路来补偿不同通道间的相位差异。
> 【落地注意】时钟路径的插入损耗必须仔细建模，接收放大器的带宽需足够高以避免引入额外的抖动。

### 1.2 限制因素

```
前向时钟I/O限制:
• 时钟skew限制性能
• 驱动器强度和负载不匹配
• 互连长度不匹配
• 低通信道导致抖动放大
• 前向时钟占空比变化
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-006.jpg)

> 🔍 深度说明：
> 【研究背景】在CMOS工艺中，驱动器强度的工艺波动可达±20%，负载不匹配来源于各位线的电容和电阻差异。互连长度不匹配在多芯片系统中尤为突出，可能导致数ps到数十ps的skew。
> 【核心结论】前向时钟系统的性能瓶颈主要在于skew管理：包括静态skew（由布局布线决定）和动态skew（由抖动和串扰引起）。低通信道（如长距离背板）会放大时钟抖动，降低时钟质量。
> 【工程价值】理解这些限制因素对于系统架构选择至关重要。在设计阶段需要进行skew预算分配，确保各通道的skew在可接受范围内。
> 【落地注意】占空比校准电路是必需的，因为时钟分配网络中的非对称负载会导致占空比失真，需要在接收端进行补偿。

## 二、数据/时钟Skew对抖动跟踪的影响

### 2.1 时钟到数据Skew效应

```
高速前向时钟允许时钟和数据间的抖动跟踪

时钟到数据skew导致:
• 高频时钟抖动和数据抖动在RX变为异相
• 抖动频率越高，异相越严重
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-007.jpg)

> 🔍 深度说明：
> 【研究背景】在相干时钟系统中，时钟和数据经历相同的信道干扰，但它们之间存在固定的skew。当输入抖动频率增加时，时钟和数据的相位差随时间累积，导致在RX端出现时变相位错位。
> 【核心结论】抖动跟踪能力与skew量直接相关：skew越大，可跟踪的抖动频率上限越低。这是因为高频抖动在时钟和数据之间产生快速相位变化，超出了系统的跟踪带宽。
> 【工程价值】在进行系统设计时，需要根据目标数据率和可接受的抖动水平来确定最大允许skew。对于10Gb/s系统，建议skew控制在10ps以内。
> 【落地注意】skew测量需要在设计阶段和量产阶段都进行，以确保工艺波动不会导致系统性能退化。

### 2.2 抖动频率与Skew关系

```
关键发现:
• 假设无限抖动跟踪带宽(JTB)
• 对于给定skew，抖动频率增加 → 差分抖动增加
• 最大可达2倍

给定抖动频率:
• 在skew = 1/(6f_j)时，系统差分抖动大于1
```

### 2.3 最佳抖动跟踪

```
10Gb/s系统 (UI=100ps):
• 抖动跟踪带宽(JTB)可控范围: 70-800MHz

方法:
• 用低通函数幅度响应衰减时钟抖动
• 极频率 = JTB
```

## 三、抖动Impulse响应与传递函数

### 3.1 JIR与JTF分析

```
JIR (Jitter Impulse Response):
• 测试系统对抖动的响应

JTF (Jitter Transfer Function):
• 输出抖动与输入抖动之比，频率函数
• JIR的DTFT

应用:
• 时钟系统对输入抖动序列的影响
• 可通过将抖动序列与JIR卷积评估
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-177.jpg)

> 🔍 深度说明：
> 【研究背景】JIR和JTF是分析时钟分配网络抖动特性的重要工具。JIR描述系统对单位抖动脉冲的时域响应，JTF是其频域表示。早期时钟分配网络分析主要关注相位噪声传递，但JTF方法更适合数字系统。
> 【核心结论】JTF的幅度响应决定了系统对不同频率抖动的放大或衰减程度。低通型JTF会放大高频抖动（如时钟缓冲器的输入上升沿引起的抖动），这是高速系统中的主要抖动放大机制。
> 【工程价值】通过JTF分析可以预测整个时钟分配链路的抖动性能，指导设计者选择合适的缓冲器类型和级数。在28Gb/s系统中，JTF分析显示存在显著的抖动放大，需要特别注意。
> 【落地注意】JTF测量需要宽带相位检测器和高精度示波器，建议在设计验证阶段进行完整链路的JTF表征。

### 3.2 滤波器/放大器频率响应与抖动传递

```
低通频率响应(缓冲器、分布互连):
• 类似于高通抖动滤波器
• 高频抖动被放大

高通频率响应(AC耦合电容):
• 除Nyquist率抖动(占空比误差)外，类似于全通抖动滤波器

带通频率响应(带通滤波器):
• 类似于低通抖动滤波器
• 中心频率对准基波时钟频率
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-180.jpg)

> 🔍 深度说明：
> 【研究背景】时钟分配网络中的滤波器特性会改变抖动传递行为。低通响应常见于缓冲器链和分布互连，其高频增益大于1，导致高频抖动放大。这是由于CMOS缓冲器的上升/下降时间有限，对窄脉冲产生微分效应。
> 【核心结论】不同频率响应特性的电路元件具有不同的抖动传递特性：低通响应放大高频抖动，高通响应抑制低频抖动，带通响应选择特定频率成分。设计时需要综合考虑这些效应。
> 【工程价值】对于28Gb/s及以上的时钟分配网络，抖动放大效应显著，可能导致系统眼图闭合。112Gb/s系统采用¼速率时钟方案（14GHz）来缓解这一问题。
> 【落地注意】在时钟分布设计中，应避免使用过多数量的缓冲器级联，以减少JTF累积效应导致的抖动放大。

### 3.3 抖动放大

```
低通频率响应:
• 高频抖动在信道传播时被放大

在28Gb/s时钟分配链中存在显著抖动放大
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-186.jpg)

> 🔍 深度说明：
> 【研究背景】抖动放大是高速时钟分配系统中的关键问题。当时钟信号通过具有低通频率响应的电路时，高频抖动成分被放大，导致时钟边沿的时间方差增加。这在28Gb/s时钟分配链中尤为明显。
> 【核心结论】抖动放大倍数与缓冲器的上升/下降时间、信道带宽和时钟频率有关。在极端情况下，抖动放大可达2倍甚至更高，严重影响系统的时序裕量。
> 【工程价值】抖动放大效应限制了时钟分配网络的最大级数和最大速率。对于28Gb/s系统，需要仔细设计缓冲器级数和布局，以控制抖动放大在可接受范围内。
> 【落地注意】采用感应峰化（inductive peaking）可以扩展带宽、减少抖动放大，但其面积和成本代价需要权衡。

## 四、前向时钟去偏斜架构

### 4.1 PLL/PI去偏斜

```
架构:
• PLL产生基准时钟
• 相位插值器(PI)生成精确相位偏移
• 每通道独立调整

优势:
• 细粒度相位控制
• 灵活
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-189.jpg)

> 🔍 深度说明：
> 【研究背景】PLL/PI架构是前向时钟去偏斜的经典方案。PLL提供低抖动的基准时钟，PI生成多相位于插值，实现精细的相位调整。这种架构在Intel的QuickPath Interconnect中得到应用。
> 【核心结论】PLL提供参考相位，PI通过数字控制实现相位粗调（4-8个相位）和细调（插值细分），分辨率可达ps级。多通道独立调整允许补偿工艺差异导致的skew。
> 【工程价值】该架构的优势是灵活性高，适用于多通道并行系统。缺点是每个通道需要独立的PI，增加了面积和功耗。
> 【落地注意】PI的非线性特性和PNM（相位噪声模数）需要仔细校准，否则会导致通道间不匹配。

### 4.2 DLL/PI去偏斜

```
架构:
• DLL锁定延迟线
• 相位插量器提供额外分辨率

特点:
• 无PLL的积分器
• 更快锁定
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-190.jpg)

> 🔍 深度说明：
> 【研究背景】DLL/PI架构相比PLL/PI具有更快的锁定时间和更好的稳定性。DLL通过调整延迟线单元来补偿skew，没有PLL的积分环节，因此不存在稳定性问题。
> 【核心结论】DLL锁定时间通常在数百个时钟周期内完成，而PLL可能需要数千个周期。DLL的相位噪声传递函数在低频段为高通特性，不累积PLL的低频相位噪声。
> 【工程价值】对于需要快速校准的应用（如处理器启动阶段），DLL/PI是更好的选择。其稳定性也更适合工艺波动较大的场景。
> 【落地注意】DLL的延迟单元受工艺和温度影响较大，需要定期重新校准以保持skew校正精度。

### 4.3 ILO去偏斜

```
Injection-Locked Oscillator:
• 注入锁定振荡器用于去偏斜
• 锁定到前向时钟
• 提供抖动过滤
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-191.jpg)

> 🔍 深度说明：
> 【研究背景】ILO是一种利用注入锁定效应的高频振荡器，其输出频率被输入参考时钟锁定。注入锁定提供了极窄的锁定带宽，能够有效过滤输入时钟的高频抖动。
> 【核心结论】ILO的核心优势是固有抖动过滤能力：锁定带宽外的抖动被抑制，锁定带宽内的抖动被跟踪。这使得ILO非常适合用于时钟分配网络的最后一级。
> 【工程价值】相比PLL，ILO不需要外部VCO和环路滤波器，结构更简单。相比DLL，ILO提供更好的抖动过滤性能。适用于对抖动要求极高的系统。
> 【落地注意】ILO的锁定范围有限，需要确保前向时钟的频率在锁定范围内。工艺波动可能导致ILO失锁，需要监控机制。

## 五、相位采集方法

```
两种实现方式:

1. 基于BER
   • 无需额外输入相位采样器
   • 直接从数据眼测量BER

2. 基于相位检测器
   • 需要额外输入相位采样器
   • 周期性上电
```

![](/img/serdes/fundamentals/lectures/lecture13_ee720_fwd_clk_deskew_深度学习报告/_images/img-248.jpg)

> 🔍 深度说明：
> 【研究背景】相位采集是去偏斜系统的关键环节，决定了如何获取相位误差信息并反馈到调整电路。基于BER的方法利用眼图质量作为指标，基于PD的方法利用专用相位检测器直接测量skew。
> 【核心结论】基于BER的方法无需额外硬件，但收敛速度慢，适用于长期校准。基于PD的方法精度高、收敛快，但需要额外的采样器电路，增加了面积和功耗。
> 【工程价值】实际系统通常结合两种方法：启动时使用PD快速收敛，运行期间使用BER进行长期跟踪。
> 【落地注意】相位检测器的静态相位offset需要校准，否则会导致持续的deskew残余误差。
