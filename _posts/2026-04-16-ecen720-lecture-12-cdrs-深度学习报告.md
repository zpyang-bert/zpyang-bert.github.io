## 一、CDR概述

### 1.1 CDR功能

```
CDR = Clock and Data Recovery

核心功能:
• 从输入数据产生采样时钟
• 时钟有效频率必须等于输入数据率
• 10GHz for 10Gb/s
• 或多个间隔100ps的时钟

额外时钟:
• 可用于相位检测

采样时钟要求:
• 足够时序裕量达到目标BER
• 小有效抖动
```

### 1.2 CDR架构类型

```
嵌入式时钟I/O实现:

1. 每通道PLL-based
   • 独立PLL per通道

2. 双环路
   • 全局PLL + 本地DLL/PI

3. 本地相位旋转PLL
   • 局部相位旋转

4. 全局PLL + RX时钟分布
   • 需要时钟分布网络
```

## 二、相位检测器

### 2.1 CDR相位检测器特殊性

```
与PLL的区别:
• CDR输入数据信号不像PLL参考时钟那样周期
• CDR相位检测器必须能处理输入数据序列中缺失的转换边沿

关键: 需要转换边沿才能检测相位误差
```

### 2.2 Hogge相位检测器

```
类型: 线性相位检测器

特性:
• 提供相位误差的符号和幅度信息
• 需要数据转换

工作原理:
• 产生Tb/2参考信号
• 产生"Late"信号，脉宽与相位差成正比
• 如果时钟采样早，Late信号比Tb/2短

相位转移:
• 0rad相对于最佳Tb/2(π)采样时钟与数据间隔
```

### 2.3 Alexander (2x Oversampled)相位检测器

![](/img/serdes/fundamentals/lectures/lecture10_ee720_jitter_深度学习报告/_images/img-000.jpg)

> 🔍 深度说明：
> 【研究背景】Alexander相位检测器是高速SerDes中最广泛使用的相位检测器类型。它于1970年代由James Alexander提出，是一种bang-bang（二进制）相位检测器，仅提供相位误差的符号而非幅度信息。这种简化使其在高速应用中更容易实现。
> 【核心结论】Alexander PD使用2x过采样：每个bit周期采样两次（data0, data1）和一次边沿（edge）。判决逻辑基于：如果边沿采样与第二比特相同，则时钟"晚"；如果边沿采样与第一比特相同，则时钟"早"。这种二进制判决虽然丢失了幅度信息，但避免了线性相位检测器的高速实现难题。
> 【工程价值】Bang-bang PD的主要优势是(1)高速实现简单（只需比较器）；(2)对相位误差的幅度不敏感；(3)不存在线性PD的鉴相死区问题。代价是引入了pattern-dependent抖动，因为当数据流缺少转换边沿时，相位信息无法更新。
> 【落地注意】在实际系统中，Alexander PD需要足够的数据转换密度来保持锁定。随机数据的TD=0.5（50%转换率）是典型假设，但对于某些数据模式（如长0或长1序列），CDR可能会出现相位滑动。设计中需要考虑这些极端情况。

```
类型: 非线性(二进制)"Bang-Bang" PD

特性:
• 仅提供相位误差符号信息(非幅度)
• 最常用的CDR相位检测器

结构:
• 2个数据采样 + 1个边沿采样

判决逻辑:
• 如果边沿采样与第二比特相同(或与第一比特不同) → 时钟采样"Late"
• 如果边沿采样与第一比特相同(或与第二比特不同) → 时钟采样"Early"

转换密度:
• 无转换则无信息
• 随机数据假设TD=0.5
```

### 2.4 Mueller-Muller Baud-Rate相位检测器

```
类型: Baud-rate相位检测器

特点:
• 每符号(每比特)仅需一个采样时钟
• 比较脉冲前后采样幅度

判决原理:
• 尝试均衡脉冲前后采样幅度

如果:
• 正值 → post-cursor ISI太高，采样太早
• 负值 → pre-cursor ISI太高，采样太晚
```

## 三、模拟PLL-based CDR

### 3.1 基本架构

```
线性PD的PLL-based CDR:
鉴相器 → 电荷泵 → 环路滤波器 → VCO → 时钟

特性:
• XOR输出直接驱动电荷泵
• 需要相对高速的电荷泵
```

### 3.2 CDR带宽特性

```
非线性能的影响:
• CDR"带宽"随输入相位变化幅度变化
• 使用非线性相位检测器时

最终性能验证:
• 需要时域非线性模型
```

### 3.3 56Gb/s PAM4模拟PLL CDR

**ISSCC 2019 (Roshan-Zamir)**：

![](/img/serdes/fundamentals/lectures/lecture12_ee720_cdrs_深度学习报告/_images/img-085.jpg)

> 🔍 深度说明：
> 【研究背景】这是ISSCC 2019发表的56Gb/s PAM4模拟PLL CDR，代表了当时最先进的CDR设计水平。相比数字CDR方案，模拟PLL CDR在功耗和延迟上具有优势，特别是在PAM4系统中需要处理更复杂的信号电平。
> 【核心结论】该架构采用¼速率设计，降低了硬件速度压力。关键创新包括：(1)3个数据采样器用于PAM4检测；(2)1个边沿采样器用于CDR和均衡自适应；(3)1个误差采样器用于阈值自适应；(4)CTLE + 2-bit Flash ADC的混合信号前端；(5)Bang-Bang PD用于相位跟踪；(6)LC-VCO用于低相位噪声时钟。
> 【工程价值】该设计展示了先进SerDes收发器的完整信号链路：模拟前端（CTLE）提供连续时间均衡，ADC提供数字域的灵活性，DFE（1-tap FIR + 1-tap IIR）消除残余ISI。PAM4的对称转换检测方案最小化了检测误差。
> 【落地注意】对于56Gb/s PAM4系统，CDR带宽的选择需要在抖动跟踪能力和PAM4信号处理复杂度之间取得平衡。BBPD在PAM4信号上的工作需要特别设计，因为PAM4有多个转换电平。

```
架构:
• ¼速率
• 3个数据采样器(PAM4检测)
• 1个边沿采样器(CDR和均衡自适应)
• 1个误差采样器(阈值自适应)

组成:
• CTLE + 2-bit Flash ADC
• 1-tap FIR + 1-tap IIR DFE
• PAM4均衡器
• Bang-Bang PD
• LC-VCO

BBPD工作原理:
• 对称PAM4转换上工作
• 最小化检测误差
```

## 四、双环路CDR

### 4.1 双环路架构

```
结构:
• 全局PLL (频率合成)
• 本地环路 (相位恢复)

全局PLL:
• 产生高频参考时钟
• 分配到各本地环路

本地环路:
• 相位插值器(PI)
• DLL
• 相位旋转PLL

优势:
• 频率和相位分离控制
• 更灵活
```

### 4.2 相位插值器

```
功能:
• 基于全局时钟生成精确相位偏移
• 可编程相位分辨率

应用:
• 每通道相位对齐
• 去偏斜
```

## 五、相位插值器

### 5.1 相位插值器原理

```
基于两个正交时钟(I和Q):
• 通过权重控制合成任意相位
• Σ (I·sin(θ) + Q·cos(θ))

实现:
• 电流加权
• 数字控制权重
```

### 5.2 相位插值器在CDR中的应用

```
双环路CDR中的角色:
• 提供细粒度相位调整
• 响应相位检测器输出
• 更新采样时钟相位
```

## 六、CDR抖动特性

### 6.1 抖动跟踪

```
CDR抖动跟踪能力受限于CDR带宽

低频抖动:
• 被CDR跟踪
• 跟踪带宽内

高频抖动:
• 无法跟踪
• 直接影响采样误差

CDR抖动传递函数:
• 低频: 跟踪(增益~1)
• 高频: 不跟踪(衰减)
```

### 6.2 技术缩放影响

```
技术缩放允许更高带宽CDR

优势:
• 更接近源同步系统的抖动跟踪优势
• 但: PAM4和ADC前端给CDR带宽带来挑战
```
